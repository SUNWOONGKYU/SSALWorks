# P2-2-2 사용자 플로우

## 단계 개요

**사용자가 서비스를 이용하는 흐름**을 설계합니다.

회원가입부터 결제까지, 사용자가 어떤 순서로 화면을 이동하는지 그려보는 작업입니다.

## 필요성

- **사용자 경험 설계**: 자연스러운 이용 흐름을 만듭니다
- **화면 구성 파악**: 어떤 화면이 필요한지 알 수 있습니다
- **예외 처리 계획**: 오류 상황을 미리 대비합니다

## 설계 내용

### 주요 플로우

- 회원가입: 처음 방문 → 가입 완료
- 로그인: 비회원 → 로그인 → 메인
- 핵심 기능: 서비스의 주요 기능 흐름
- 결제: 상품 선택 → 결제 완료

### 플로우 요소

- 시작점: 어디서 시작하는가
- 단계: 어떤 순서로 진행하는가
- 분기: 조건에 따라 다른 경로
- 종료점: 어디서 끝나는가

## PO가 결정할 것

- 필요한 플로우: 어떤 흐름을 설계할지
- 사용자 유형: 일반 사용자, 관리자 등
- 특별한 조건: 특정 조건에서의 처리
