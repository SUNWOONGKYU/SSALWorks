# P2-1 프로젝트 개요

## 단계 개요

프로젝트의 **전체 범위와 목표**를 정리합니다.

무엇을 만들 것인지, 무엇을 포함하고 무엇을 제외할 것인지, 누가 관련되어 있는지를 명확히 하는 기획의 첫 단계입니다.

## 필요성

- **목표 명확화**: 무엇을 만들 것인지 정확히 정의합니다
- **범위 설정**: 포함/제외할 것을 미리 정해 혼란을 방지합니다
- **책임 명확화**: 누가 무엇을 담당하는지 정합니다

## 정리 내용

### 프로젝트 개요

- 프로젝트명: 공식 명칭
- 한 줄 요약: 무엇을 하는 서비스인지
- 핵심 기능: 3~5개의 주요 기능

### 범위 정의

- 포함 (In Scope): 이번에 만들 것
- 제외 (Out of Scope): 이번에 안 만들 것

### 이해관계자

- Project Owner: 최종 결정권자
- 개발자/AI: 구현 담당
- 사용자: 피드백 제공

## PO가 결정할 것

- 프로젝트 유형: 웹앱, 모바일, API 등
- 핵심 기능: 반드시 필요한 기능 목록
- 제외 범위: 이번에 안 만들 것
