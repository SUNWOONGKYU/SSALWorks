# P2-2-1 요구사항 정의

---

## 이 단계는 무엇인가요?

서비스에 **어떤 기능이 필요한지** 구체적으로 정의합니다.

"이런 기능이 필요해"라는 아이디어를 체계적인 요구사항 목록으로 정리하는 단계입니다.

---

## 왜 필요한가요?

- **개발 범위 확정**: 무엇을 만들어야 하는지 명확해집니다
- **우선순위 설정**: 먼저 만들 것과 나중에 만들 것을 구분합니다
- **MVP 정의**: 최소한으로 필요한 기능을 정합니다

---

## 무엇을 정의하나요?

### 기능 요구사항

| 분류 | 예시 |
|------|------|
| 사용자 기능 | 회원가입, 로그인, 검색 |
| 관리자 기능 | 사용자 관리, 통계 |
| 시스템 기능 | 알림, 백업, 동기화 |

### 비기능 요구사항

| 분류 | 예시 |
|------|------|
| 성능 | 페이지 로딩 2초 이내 |
| 보안 | HTTPS, 비밀번호 암호화 |
| 사용성 | 모바일 대응 |

### 우선순위

| 등급 | 의미 |
|------|------|
| Must | 반드시 필요 (MVP) |
| Should | 있으면 좋음 |
| Could | 나중에 추가 |

---

## PO가 준비할 것

| 항목 | 설명 |
|------|------|
| 핵심 기능 목록 | 반드시 있어야 하는 기능 |
| 사용자 유형 | 일반 사용자, 관리자 등 |
| 성능 기대치 | 속도, 동시 접속 등 |
