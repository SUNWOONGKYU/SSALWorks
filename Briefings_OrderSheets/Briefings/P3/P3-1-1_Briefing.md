# P3-1-1 프론트엔드 프로토타입

## 이 단계는 무엇인가요?

**실제로 동작하는 화면**을 만듭니다.

디자인 가이드라인을 바탕으로 클릭하고 입력할 수 있는 화면을 구현하는 단계입니다.

## 왜 필요한가요?

- **동작 검증**: 실제로 써보면서 문제를 발견합니다
- **디자인 확인**: 화면이 의도대로 보이는지 확인합니다
- **개발 기반**: 본개발의 기초가 됩니다

## 무엇을 만드나요?

### 개발 범위

- 공통 레이아웃: 헤더, 푸터, 사이드바
- 공통 컴포넌트: 버튼, 입력폼, 카드
- 반응형: PC, 모바일 대응

### 기술 선택

- 마크업: HTML5
- 스타일: CSS 또는 Tailwind
- 스크립트: JavaScript

## PO가 결정할 것

- 구현할 화면: 어떤 화면을 먼저 만들지
- 기술 스택: 사용할 기술 선택
- 참고 디자인: 참고할 서비스
