# 단계별로 요청하기

## 💡 Tip

한 번에 큰 작업을 요청하기보다 **작은 단계로 나눠서** 요청하면 더 정확한 결과를 얻습니다.

## 왜 단계별로?

- AI가 한 번에 처리할 수 있는 양에 한계가 있음
- 중간에 방향 수정이 쉬움
- 각 단계별로 검증 가능
- 오류 발생 시 원인 파악이 쉬움

## 예시 비교

### ❌ 한 번에 큰 요청

```
"회원 인증 시스템 전체를 구현해줘.
회원가입, 로그인, 비밀번호 재설정,
Google OAuth, 이메일 인증까지 다 포함해서."
```

### ✅ 단계별 요청

```
1단계: "먼저 회원가입 API 구조를 설계해줘"
  → 검토 후 승인

2단계: "회원가입 API를 구현해줘"
  → 테스트 후 확인

3단계: "로그인 API를 구현해줘"
  → 테스트 후 확인

4단계: "Google OAuth를 연동해줘"
  → 테스트 후 확인
```

## 실제 대화 흐름

```
👤: "회원가입 기능을 만들어야 해"

🤖: "회원가입 기능을 구현하겠습니다.
    먼저 어떤 정보를 받을지 정해야 합니다.
    - 이메일
    - 비밀번호
    - 닉네임
    이 정도면 될까요?"

👤: "응, 그거면 돼"

🤖: "API 먼저 만들까요, 아니면 프론트엔드 폼부터?"

👤: "API 먼저"

🤖: "회원가입 API를 구현했습니다.
    [코드 표시]
    테스트해볼까요?"

👤: "응"

🤖: "테스트 통과했습니다. 다음으로 프론트엔드 폼을 만들까요?"
```

## 중간 확인 포인트

각 단계 후 확인할 것:
- [ ] 코드가 의도대로 작동하는가?
- [ ] 다음 단계로 넘어가도 되는가?
- [ ] 수정이 필요한 부분이 있는가?
