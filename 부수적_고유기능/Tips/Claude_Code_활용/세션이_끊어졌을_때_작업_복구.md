# 세션이 끊어졌을 때 작업 복구

## 질문
Claude Code 세션이 끊어졌는데 이전 작업을 어떻게 이어서 하나요?

## 답변

### 자동 복구 방법

Claude Code는 이전 대화를 기억합니다. 새 세션에서:

```
"이전 작업 이어서 해줘"
```

또는

```
"마지막 작업이 뭐였지?"
```

### work_logs 활용

SSAL Works 프로젝트에서는 `.claude/work_logs/current.md`를 사용합니다:

```
"work_logs 확인하고 이전 작업 이어서 진행해줘"
```

### 수동 복구 방법

**1. 최근 파일 변경 확인**
```bash
git status
git diff
```

**2. 작업 로그 확인**
```bash
cat .claude/work_logs/current.md
```

**3. 구체적으로 지시**
```
"index.html에서 FAQ 섹션 작업하다 끊겼어.
마지막으로 searchFaqLocal 함수까지 추가했고,
이제 showFaqAnswer 함수를 추가해야 해."
```

### 세션 끊김 방지

**1. 작업 단위를 작게**
- 한 번에 너무 큰 작업 요청 X
- 작은 단위로 나눠서 요청

**2. 중간 저장 요청**
```
"여기까지 저장하고, 다음 단계 진행해줘"
```

**3. 정기적 커밋**
```
"지금까지 작업 커밋해줘"
```

### 컨텍스트 제한

Claude Code에는 컨텍스트 제한이 있습니다:
- 대화가 길어지면 자동 요약됨
- 중요한 정보는 파일로 저장해두면 안전

### 관련 파일
- `.claude/work_logs/current.md`
- `CLAUDE.md` - 작업 기록 확인 섹션
