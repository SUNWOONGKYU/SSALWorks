# 검증 완료 후 문서화하기

## 💡 Tip

작업이 끝나면 **검증 → 문서화 → 보고** 순서로 마무리해야 완료입니다.

## SSAL Works 완료 프로세스

```
작업 완료
    ↓
검증 수행 (테스트, 코드 리뷰)
    ↓
문서화 (outbox에 보고서 저장)
    ↓
최종 보고
```

## 필수 산출물

### 1. 작업 결과 보고서
```
위치: Web_ClaudeCode_Bridge/outbox/

파일명: {task_id}_completed.json
또는: agenda{n}_{기능명}_completed.json
```

**내용:**
```json
{
    "task_id": "S2BA1",
    "task_name": "회원가입 API",
    "status": "완료",
    "files": [
        "S2_개발-1차/Backend_API/auth/signup.ts"
    ],
    "completed_at": "2024-12-16T10:00:00Z"
}
```

### 2. 검증 결과 보고서
```
파일명: {task_id}_verification.json
```

**내용:**
```json
{
    "task_id": "S2BA1",
    "verification_result": "통과",
    "tests": {
        "unit": "24/24 통과",
        "e2e": "5/5 통과"
    },
    "issues_found": [],
    "verified_at": "2024-12-16T10:30:00Z"
}
```

### 3. 프로젝트 문서 업데이트

- API 문서 업데이트
- README 업데이트
- 변경 이력 기록

## 완료 보고 형식

```markdown
✅ 작업 완료 및 검증 완료

**작업 내용:**
- 회원가입 API 구현
- 입력 검증 추가

**생성된 파일:**
- S2_개발-1차/Backend_API/auth/signup.ts

**검증 결과:**
- ✅ Unit Tests: 24/24 통과
- ✅ Lint: 오류 없음

**생성된 문서:**
- ✅ Web_ClaudeCode_Bridge/outbox/S2BA1_completed.json
- ✅ Web_ClaudeCode_Bridge/outbox/S2BA1_verification.json
```

## 왜 문서화가 중요한가?

1. **추적성**: 누가 언제 무엇을 했는지 기록
2. **품질 관리**: 검증 결과 증빙
3. **협업**: 팀원과 진행 상황 공유
4. **감사**: 나중에 문제 발생 시 원인 추적
