# 작업 규칙 CLAUDE.md에 업데이트하기

## 요약
새로운 규칙이 생기면 `CLAUDE.md`에 추가. 문서화된 규칙만 Claude Code가 일관되게 따른다.

## 상세

### 왜 필요한가

```
❌ 규칙 안 적음:
"Production에 직접 저장하지 마" → 다음 세션 → 규칙 모름 → 또 실수

✅ CLAUDE.md에 추가:
규칙 추가 → 다음 세션 → 규칙 확인 → 올바르게 작업
```

### 업데이트할 규칙 종류

| 종류 | 예시 |
|------|------|
| 금지 규칙 | 새 폴더 생성 전 승인 필수 |
| 저장 위치 | Frontend → S?_*/Frontend/ + Production/Frontend/ |
| 작업 절차 | 완료 시 work_logs 업데이트 필수 |

### 업데이트 지시 방법

```
"CLAUDE.md에 '테스트 없이 완료 보고 금지' 규칙 추가해줘"
"CLAUDE.md 절대 규칙 섹션에 이 내용 추가해줘"
```

### 규칙 작성 팁
- 명확하고 구체적으로 (모호한 표현 피하기)
- 예시 포함
- 중요한 규칙은 강조 표시 (🛑, ⚠️)
