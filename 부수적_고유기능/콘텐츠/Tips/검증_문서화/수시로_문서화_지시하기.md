# 수시로 문서화 지시하기

## 요약
작업 중 중요한 내용은 바로 문서화 지시. 설정 방법, 트러블슈팅 과정, 결정 사항 등 나중에 잊어버리기 전에 기록.

## 상세

### 문서화 대상

| 종류 | 예시 |
|------|------|
| 설정 방법 | Supabase, Vercel, API 키 설정 과정 |
| 트러블슈팅 | 오류 발생 → 해결 과정 |
| 중요 결정 | 기술 선택 이유, 아키텍처 결정 |

### 문서화 지시 방법

```
"방금 한 Supabase OAuth 설정 과정 문서화해줘"
"CORS 오류 해결 과정을 Documentation/troubleshooting.md에 추가해줘"
```

### 저장 위치

| 위치 | 내용 |
|------|------|
| `Documentation/` | 기능 명세, API 문서, 설정 가이드 |
| `work_logs/` | 작업 기록, 일일 로그 |

### 주의
- 작업 직후 바로 문서화 (나중에 미루면 세부사항 잊음)
- 민감정보(API 키)는 문서에 직접 포함 금지
