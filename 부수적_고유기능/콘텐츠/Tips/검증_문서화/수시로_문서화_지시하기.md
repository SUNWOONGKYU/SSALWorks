# 수시로 문서화 지시하기

## 핵심 요약

작업 중에 나중에 참고할 내용이 있으면 수시로 문서화하도록 Claude Code에게 지시하세요. 설정 방법, 트러블슈팅 과정, 중요한 결정 사항 등을 기록해두면 나중에 유용합니다.

## 왜 수시로 문서화해야 하나?

### 문서화 없이 작업하면

```
❌ 문서화 안 함:
API 설정 완료 → 1주 후 → "어떻게 설정했더라?"
버그 해결 완료 → 같은 버그 발생 → "어떻게 고쳤더라?"
중요 결정 → 나중에 → "왜 이렇게 했지?"
```

### 수시로 문서화하면

```
✅ 문서화 함:
API 설정 완료 → 설정 방법 기록 → 나중에 바로 참고
버그 해결 완료 → 해결 과정 기록 → 같은 버그 즉시 해결
중요 결정 → 이유 기록 → 나중에 맥락 파악
```

## 문서화할 내용

### 1. 설정 방법

```
문서화 대상:
- 외부 서비스 설정 (Supabase, Vercel, API 등)
- 환경변수 설정
- 개발 환경 구성
- 배포 설정
```

### 2. 트러블슈팅

```
문서화 대상:
- 발생한 오류와 해결 방법
- 시도했던 방법들
- 최종 해결책
- 재발 방지 방법
```

### 3. 중요 결정

```
문서화 대상:
- 기술 선택 이유
- 아키텍처 결정
- 라이브러리 선택 근거
- 트레이드오프 고려사항
```

### 4. 작업 과정

```
문서화 대상:
- 복잡한 기능 구현 과정
- 특별한 처리 방법
- 주의해야 할 사항
- 연관된 파일들
```

## 문서화 지시 방법

### 간단한 지시

```
"방금 한 설정 과정 문서화해줘"
```

### 상세한 지시

```
"Supabase OAuth 설정 과정을 문서화해줘:
- 어떤 단계로 진행했는지
- 어떤 값을 어디에 설정했는지
- 주의할 점은 무엇인지"
```

### 저장 위치 지정

```
"방금 해결한 CORS 오류 해결 과정을
S2_개발-1차/Documentation/troubleshooting.md에 추가해줘"
```

## 문서화 시점

### 즉시 문서화해야 하는 경우

| 상황 | 이유 |
|------|------|
| 설정 완료 직후 | 단계를 기억할 때 |
| 버그 해결 직후 | 과정이 생생할 때 |
| 중요 결정 직후 | 이유를 잊기 전에 |
| 복잡한 작업 완료 | 세부사항 기억할 때 |

### 문서화 미루면 안 되는 이유

```
❌ 나중에 하려고 미룸:
세부사항 잊어버림 → 불완전한 문서 → 나중에 참고 불가

✅ 바로 문서화:
세부사항 생생함 → 완전한 문서 → 나중에 바로 참고
```

## 문서 저장 위치

### 위치별 용도

| 위치 | 내용 |
|------|------|
| `Documentation/` | 기능 명세, API 문서 |
| `work_logs/` | 작업 기록, 일일 로그 |
| `outbox/` | 완료 보고서, 검증 결과 |
| 해당 기능 폴더 | 해당 기능 관련 문서 |

### 예시

```
S2_개발-1차/
├── Documentation/
│   ├── api_spec.md          # API 명세
│   ├── setup_guide.md       # 설정 가이드
│   └── troubleshooting.md   # 문제 해결 기록
├── Frontend/
│   └── README.md            # 프론트엔드 설명
└── Backend_API/
    └── README.md            # 백엔드 설명
```

## 문서 형식

### 설정 가이드 형식

```markdown
# [서비스명] 설정 가이드

## 사전 요구사항
- 필요한 계정/권한

## 설정 단계
1. 첫 번째 단계
2. 두 번째 단계
...

## 설정 값
- 환경변수: VARIABLE_NAME=설명

## 주의사항
- 주의할 점

## 확인 방법
- 설정이 잘 됐는지 확인하는 방법
```

### 트러블슈팅 형식

```markdown
# [오류명] 해결

## 증상
- 오류 메시지
- 발생 상황

## 원인
- 왜 발생했는지

## 해결 방법
1. 해결 단계

## 예방 방법
- 재발 방지 방법
```

## 효과적인 문서화 팁

### 1. 스크린샷/코드 포함

```
"설정 화면 캡처해서 문서에 포함해줘"
(또는)
"실제 설정 코드 예시도 넣어줘"
```

### 2. 체크리스트 형태

```markdown
## 설정 체크리스트
- [ ] API 키 발급
- [ ] 환경변수 설정
- [ ] 테스트 확인
```

### 3. 버전 정보 포함

```markdown
> 작성일: 2025-01-15
> 적용 버전: Supabase v2.x
```

## 주의사항

- 민감정보(API 키 등)는 문서에 직접 포함 금지
- 문서는 간결하고 명확하게
- 문서 위치는 일관되게 유지
- 오래된 문서는 주기적으로 업데이트
- 너무 세세한 것까지 문서화할 필요는 없음
