# 개발 완료 후 서비스 개시 전 - 디자인/완성도 체크

## 요약
개발 완료 후 서비스 개시 전까지 오류 수정/보완 작업을 하다 보면 디자인이 깨지거나 기능이 빠질 수 있다. 수시로 디자인 체크 + 완성도 체크 필요.

## 상세

### 두 가지 체크

| 체크 | 확인 내용 |
|------|----------|
| 디자인 체크 | 레이아웃 깨짐, 반응형 틀어짐, 스타일 날아감 |
| 완성도 체크 | 기능 누락, 빠진 화면, 미완성 부분 |

### 프로세스

```
서브 에이전트 체크 → Main Agent 검토 → PO 최종 확인 → 수정
```

### Claude Code에게 요청

```
"디자인 깨진 곳 없는지 체크해줘"

"원래 계획이랑 비교해서 빠진 기능 있는지 확인해줘"
```
