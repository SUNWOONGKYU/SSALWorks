# .gitignore로 API 키, 비밀번호 같은 민감정보가 GitHub에 올라가지 않게 보호하기

> 이 문서는 .gitignore를 활용하여 민감한 정보가 GitHub에 올라가지 않도록 보호하는 방법을 설명합니다.

---

## 왜 중요한가

API 키, 비밀번호 등이 GitHub에 올라가면 악의적인 사용자가 발견할 수 있습니다. 한 번 노출된 키는 즉시 폐기하고 재발급해야 합니다.

---

## 반드시 .gitignore에 추가할 것

| 유형 | 파일 예시 |
|------|----------|
| 환경 변수 | `.env`, `.env.local`, `config.json` |
| 인증 정보 | `credentials.json`, `service-account.json` |
| 개인 키 | `*.pem`, `*.key`, `id_rsa` |
| 데이터베이스 | `*.db`, `*.sqlite` |
| 빌드 결과물 | `node_modules/`, `dist/`, `.next/` |

---

## 기본 .gitignore 예시

```gitignore
# 환경 변수
.env
.env.local
.env.production

# 인증/키
*.pem
*.key
credentials.json
service-account.json

# 의존성
node_modules/

# 빌드
dist/
.next/

# IDE
.vscode/
.idea/
```

---

## 이미 커밋된 파일 제거

```bash
# 1. 파일을 Git 추적에서 제거 (파일은 유지)
git rm --cached .env

# 2. .gitignore에 추가
echo ".env" >> .gitignore

# 3. 커밋
git commit -m "chore: 민감정보 파일 추적 제거"
```

---

## .env.example 활용

```
.env (gitignore에 추가, 실제 값)
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_KEY=eyJhbGc...

.env.example (커밋됨, 빈 값 템플릿)
SUPABASE_URL=
SUPABASE_KEY=
```

팀원은 `.env.example`을 복사해서 `.env`를 만들고 실제 값을 입력합니다.
