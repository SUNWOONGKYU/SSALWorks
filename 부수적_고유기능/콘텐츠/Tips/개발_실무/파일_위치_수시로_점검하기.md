# 파일 위치 수시로 점검하기

## 핵심 요약

작업 디렉토리 내 파일들이 적절한 폴더에 있는지 수시로 검토하고, 잘못된 위치의 파일은 올바른 폴더로 이동해야 합니다.

## 왜 파일 위치 점검이 중요한가?

### 1. 관리 용이성

파일이 제자리에 있으면:
- 프로젝트 구조가 깔끔하게 유지됨
- 팀원들과 협업할 때 혼란이 없음
- 나중에 유지보수할 때 편리함

### 2. 파일 찾기 쉬움

올바른 폴더에 파일이 있으면:
- 필요한 파일을 빠르게 찾을 수 있음
- "어디에 저장했더라?" 고민 불필요
- 작업 시간 단축

## 점검 방법

### 1. 정기적으로 디렉토리 구조 확인

```
# 프로젝트 루트에서 폴더 구조 확인
ls -la
tree (또는 dir /s)
```

### 2. 점검 체크리스트

- [ ] 루트 디렉토리에 임시 파일이 쌓여있지 않은가?
- [ ] 코드 파일이 올바른 Stage 폴더에 있는가?
- [ ] 문서 파일이 Documentation 폴더에 있는가?
- [ ] 테스트 파일이 Testing 폴더에 있는가?
- [ ] Production 폴더와 Stage 폴더가 동기화되어 있는가?

### 3. 잘못된 위치 발견 시

1. 파일의 용도 확인
2. `Project_Directory_Structure.md` 참고하여 올바른 위치 확인
3. 파일 이동
4. 관련 참조(import, 링크 등) 수정

## 자주 발생하는 문제

| 문제 | 올바른 위치 |
|------|------------|
| 루트에 임시 파일 방치 | 해당 용도의 폴더로 이동 또는 삭제 |
| SQL 파일이 Frontend에 | Database 폴더로 이동 |
| 테스트 파일이 코드와 섞임 | Testing 폴더로 이동 |
| 문서가 여기저기 흩어짐 | Documentation 폴더로 통합 |

## 습관 만들기

- **작업 시작 전**: 현재 폴더 구조 빠르게 확인
- **작업 완료 후**: 새로 생성한 파일 위치 확인
- **커밋 전**: git status로 변경된 파일 경로 점검
- **주 1회**: 전체 프로젝트 구조 리뷰
