# 최적화 성능 개선 작업 시키기

## 핵심 요약

프로젝트가 어느 정도 완성된 시점에 최적화 작업을 요청하세요. 코드 정리, 불필요한 파일 제거, 로딩 속도 개선, 번들 크기 축소 등을 Claude Code에게 시킬 수 있습니다.

## 언제 최적화를 요청하나?

### 최적화 적합 시점

| 시점 | 설명 |
|------|------|
| 기능 완성 후 | 핵심 기능이 모두 동작할 때 |
| 배포 전 | 프로덕션 환경에 올리기 전 |
| 성능 문제 발견 시 | 느림/버벅거림 발생 시 |
| 정기 점검 | 월 1회 등 주기적으로 |

### 너무 이른 최적화의 문제

```
❌ 기능 개발 중 최적화
→ 코드 변경 시 최적화도 다시 해야 함
→ 시간 낭비

✅ 기능 완성 후 최적화
→ 안정된 코드에 최적화 적용
→ 효율적
```

## 최적화 요청 방법

### 1. 전체 최적화 요청

```
"프로젝트 전체 최적화해줘:
- 사용하지 않는 코드 제거
- 성능 개선
- 번들 크기 축소"
```

### 2. 특정 영역 최적화

```
"프론트엔드 성능 최적화해줘"

"데이터베이스 쿼리 최적화해줘"

"이미지 로딩 속도 개선해줘"
```

### 3. 구체적인 문제 해결

```
"메인 페이지 로딩이 5초 이상 걸려. 3초 이내로 줄여줘"

"번들 크기가 2MB야. 500KB 이하로 줄여줘"
```

## 최적화 영역별 요청

### 코드 정리

```
"다음 최적화 작업해줘:
- 사용하지 않는 함수/변수 제거
- 중복 코드 통합
- 주석 정리
- 불필요한 console.log 제거"
```

### 파일 정리

```
"프로젝트 파일 정리해줘:
- 사용하지 않는 파일 찾아서 보고
- 임시 파일 삭제
- 중복 파일 확인"
```

### 프론트엔드 최적화

```
"프론트엔드 성능 최적화해줘:
- 이미지 lazy loading 적용
- CSS/JS 압축
- 코드 스플리팅
- 캐싱 설정"
```

### 백엔드 최적화

```
"백엔드 성능 최적화해줘:
- N+1 쿼리 문제 해결
- 인덱스 추가
- 응답 캐싱
- 불필요한 데이터 제거"
```

## 주요 최적화 항목

### 프론트엔드

| 항목 | 설명 | 요청 예시 |
|------|------|----------|
| 이미지 최적화 | 크기 축소, 포맷 변환 | "이미지 WebP로 변환해줘" |
| Lazy Loading | 필요할 때만 로드 | "이미지 lazy loading 적용해줘" |
| 코드 스플리팅 | 필요한 코드만 로드 | "라우트별 코드 스플리팅해줘" |
| Tree Shaking | 미사용 코드 제거 | "빌드 시 dead code 제거해줘" |
| 압축 | CSS/JS minify | "프로덕션 빌드 압축 설정해줘" |

### 백엔드

| 항목 | 설명 | 요청 예시 |
|------|------|----------|
| 쿼리 최적화 | 효율적인 DB 조회 | "느린 쿼리 찾아서 최적화해줘" |
| 인덱스 | DB 검색 속도 향상 | "자주 조회하는 컬럼에 인덱스 추가해줘" |
| 캐싱 | 반복 요청 최소화 | "API 응답 캐싱 적용해줘" |
| 페이지네이션 | 대량 데이터 분할 | "목록 API에 페이지네이션 추가해줘" |
| N+1 해결 | 불필요한 쿼리 제거 | "N+1 쿼리 문제 확인하고 해결해줘" |

## 성능 측정 요청

### 현재 상태 파악

```
"현재 성능 상태 분석해줘:
- 페이지 로딩 시간
- 번들 크기
- API 응답 시간"
```

### 개선 전후 비교

```
"최적화 전후 성능 비교 리포트 만들어줘"
```

## 최적화 체크리스트

### 배포 전 필수 최적화

- [ ] console.log 제거 또는 조건부 처리
- [ ] 개발용 코드/주석 제거
- [ ] 환경변수 프로덕션 설정 확인
- [ ] 에러 핸들링 완료
- [ ] 이미지 최적화
- [ ] CSS/JS 압축 설정

### 성능 점검 항목

- [ ] 첫 페이지 로딩 3초 이내
- [ ] 번들 크기 적정 수준
- [ ] API 응답 1초 이내
- [ ] 메모리 누수 없음
- [ ] 불필요한 리렌더링 없음

## 최적화 요청 예시

### 종합 최적화 요청

```
"프로젝트 최적화 작업해줘:

1. 코드 정리
   - 사용하지 않는 import 제거
   - 중복 코드 통합
   - console.log 제거

2. 성능 개선
   - 이미지 lazy loading
   - API 응답 캐싱
   - 불필요한 리렌더링 방지

3. 빌드 최적화
   - 번들 크기 분석 및 축소
   - 코드 스플리팅

완료 후 개선 결과 보고해줘"
```

## 주의사항

- 기능 개발이 끝난 후에 최적화 진행
- 최적화 전에 현재 상태 백업/커밋
- 최적화 후 기능 테스트 필수 (동작 확인)
- 과도한 최적화는 오히려 복잡도 증가
- 성능 측정 도구로 실제 개선 확인
- 사용자 체감 성능이 가장 중요
