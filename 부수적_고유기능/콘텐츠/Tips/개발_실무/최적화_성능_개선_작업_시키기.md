# 최적화 성능 개선 작업 시키기

> 이 문서는 프로젝트 최적화와 성능 개선 작업을 요청하는 방법을 설명합니다.

---

## 최적화 적합 시점

| 시점 | 설명 | 요청 예시 |
|------|------|----------|
| **기능 완성 후** | 핵심 기능이 모두 동작할 때 | "핵심 기능 완료됐으니 최적화해줘" |
| **배포 전** | 프로덕션에 올리기 전 | "배포 전에 성능 점검해줘" |
| **성능 문제 발견 시** | 느림/버벅거림 발생 시 | "로딩이 느려, 원인 파악해줘" |

**최적화하면 안 되는 시점**: 기능 개발 중, 요구사항 미확정

---

## 최적화 요청 방법

### 전체 최적화

```
"프로젝트 전체 최적화해줘:
- 사용하지 않는 코드 제거
- 성능 개선
- 번들 크기 축소"
```

### 영역별 요청

| 영역 | 요청 예시 |
|------|----------|
| 코드 정리 | "사용하지 않는 함수/변수 제거해줘" |
| 프론트엔드 | "이미지 lazy loading 적용해줘" |
| 백엔드 | "N+1 쿼리 문제 해결해줘" |
| 빌드 | "번들 크기 분석하고 축소해줘" |
| 데이터베이스 | "느린 쿼리 찾아서 최적화해줘" |

---

## 최적화 전 필수 작업

### 백업/커밋

```bash
git add . && git commit -m "chore: 최적화 전 상태 저장"
```

### 현재 상태 측정

```
"최적화 전에 현재 성능 측정해줘:
- 페이지 로딩 시간
- 번들 크기
- Lighthouse 점수"
```

---

## 최적화 후 확인

```
"최적화 전후 성능 비교해줘"
"최적화 후 기능 테스트해줘"
```

---

## 주의사항

- 기능 개발이 끝난 후에 최적화 진행
- 최적화 전에 현재 상태 백업/커밋
- 최적화 후 기능 테스트 필수
- 과도한 최적화 주의 (실제로 문제가 되는 부분만)

---

## 체크리스트

### 최적화 전
- [ ] 기능 개발이 완료되었는가?
- [ ] 현재 상태를 커밋했는가?
- [ ] 현재 성능을 측정했는가?

### 최적화 후
- [ ] 성능이 개선되었는가?
- [ ] 모든 기능이 정상 작동하는가?
- [ ] 최적화 결과를 문서화했는가?
