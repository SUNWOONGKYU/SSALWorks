# 폴더 정리 및 임의 생성 금지 규칙

## 요약
디렉토리 폴더는 수시로 정리하고, Claude Code가 임의로 폴더를 만들지 못하도록 CLAUDE.md에 규칙을 정해둔다. 무분별한 폴더 생성은 프로젝트 구조를 망가뜨린다.

## 상세

### CLAUDE.md에 추가할 규칙

```markdown
## 🛑 절대 규칙: 새 폴더 생성 금지

⛔ 새 폴더를 만들기 전에 반드시 사용자에게 물어야 합니다!

프로세스:
1. 해당 폴더가 존재하는가? → YES → 저장
2. 해당 폴더가 존재하는가? → NO → 사용자에게 질문
3. 사용자 승인 후에만 폴더 생성
```

### 폴더 정리 요청

```
"프로젝트 폴더 정리해줘:
- 빈 폴더 확인
- 임시 파일 정리
- 잘못된 위치의 파일 보고"
```

### 정리 주기

| 시점 | 정리 내용 |
|------|----------|
| 작업 완료 후 | 임시 파일 정리 |
| 커밋 전 | 불필요한 파일 확인 |
| Stage 완료 시 | 해당 Stage 폴더 정리 |
| 월 1회 | 전체 프로젝트 정리 |

### 주의
- CLAUDE.md에 규칙을 명확히 작성해야 Claude Code가 따름
- 한 번 어지러워진 구조는 정리하기 어려움
- 예방이 치료보다 쉬움
