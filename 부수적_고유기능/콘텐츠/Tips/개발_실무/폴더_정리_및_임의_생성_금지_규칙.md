# 폴더 정리 및 임의 생성 금지 규칙

> 이 문서는 프로젝트 폴더 구조를 유지하고 임의 폴더 생성을 방지하기 위한 규칙을 설명합니다.

---

## 왜 이 규칙이 필요한가

`temp/`, `backup/`, `test_files/` 같은 폴더가 하나둘 생기면 프로젝트 구조가 무너집니다. SAL Grid에서는 **폴더 구조 동결** 규칙으로 이를 방지합니다.

---

## CLAUDE.md에 규칙 명시

```markdown
🛑 절대 규칙: 새 폴더 생성 금지

⛔ 새 폴더를 만들기 전에 반드시 사용자에게 물어야 합니다!

프로세스:
1. 폴더가 존재하는가? → YES → 저장
2. 폴더가 존재하는가? → NO → ⚠️ 중단!
3. 사용자에게 승인 요청
4. 승인 후에만 폴더 생성
```

---

## 폴더 정리 요청

```
"프로젝트 폴더 정리해줘:
- 빈 폴더 확인
- 임시 파일 정리
- 잘못된 위치의 파일 보고"
```

---

## 정리 주기

| 시점 | 요청 예시 |
|------|----------|
| 작업 완료 후 | "임시 파일 정리해줘" |
| 커밋 전 | "git status 확인해줘" |
| Stage 완료 시 | "S2 폴더 정리 상태 확인해줘" |
| 월 1회 | "전체 프로젝트 폴더 구조 리뷰해줘" |

---

## 자주 발생하는 문제

**루트에 임시 파일**: 삭제하거나 적절한 폴더로 이동

**빈 폴더**: 삭제 (Git에서 빈 폴더는 추적되지 않음)

**중복 파일** (login_backup.js, login_v2.js): Git이 버전 관리 → 중복 파일 삭제

---

## 금지 행동

- ❌ 사용자 승인 없이 `mkdir` 실행
- ❌ "나중에 정리하면 되겠지" 생각
- ❌ Production, Stage에 임의 하위 폴더 추가
- ❌ 임시/백업 파일 방치

---
