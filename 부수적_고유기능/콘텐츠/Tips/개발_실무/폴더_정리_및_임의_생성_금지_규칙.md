# 폴더 정리 및 임의 생성 금지 규칙

## 핵심 요약

디렉토리 폴더는 수시로 정리하고, Claude Code가 임의로 폴더를 만들지 못하도록 CLAUDE.md에 규칙을 정해두세요. 무분별한 폴더 생성은 프로젝트 구조를 망가뜨립니다.

## 왜 폴더 생성을 제한해야 하나?

### 임의 폴더 생성의 문제

```
❌ 제한 없이 폴더 생성:
Claude Code가 "편의상" 새 폴더 생성
→ 폴더가 여기저기 생김
→ 기존 구조와 불일치
→ 프로젝트 구조 혼란
→ 파일 찾기 어려움

✅ 폴더 생성 제한:
새 폴더 생성 전 승인 필수
→ 기존 구조 유지
→ 일관된 프로젝트 구조
→ 관리 용이
```

## CLAUDE.md에 규칙 추가하기

### 절대 규칙 설정

```markdown
## 🛑 절대 규칙: 새 폴더 생성 금지

⛔ 새 폴더를 만들기 전에 반드시 사용자에게 물어야 합니다!
⛔ 절대 임의로 폴더를 생성하지 마세요!

프로세스:
1. 파일 저장이 필요함
2. 해당 폴더가 존재하는가? → YES → 저장
3. 해당 폴더가 존재하는가? → NO → 사용자에게 질문
4. 사용자 승인 후에만 폴더 생성
```

### 금지 행동 명시

```markdown
❌ 금지 행동:
- mkdir 명령어를 사용자 승인 없이 실행
- "편의상" 새 폴더 생성
- "나중에 정리하면 되겠지" 생각
- Production, Stage 폴더에 임의로 하위 폴더 추가
```

## 폴더 정리 방법

### 정기 정리 요청

```
"프로젝트 폴더 정리해줘:
- 빈 폴더 확인
- 임시 파일 정리
- 잘못된 위치의 파일 보고"
```

### 정리 체크리스트

| 확인 항목 | 조치 |
|----------|------|
| 빈 폴더 | 삭제 또는 유지 결정 |
| 임시 파일 | 삭제 또는 적절한 위치로 이동 |
| 중복 폴더 | 통합 |
| 용도 불명 폴더 | 내용 확인 후 정리 |
| 네이밍 불일치 | 규칙에 맞게 수정 |

### Claude Code에게 정리 지시

```
"다음 정리 작업해줘:
1. 루트 디렉토리의 불필요한 파일 찾기
2. 빈 폴더 목록 만들기
3. 임시/테스트 폴더 확인
4. 정리 필요한 항목 보고"
```

## 폴더 생성이 정말 필요할 때

### 승인 요청 프로세스

```
Claude Code가 해야 할 것:

"폴더 [폴더명]이 존재하지 않습니다.
새 폴더를 생성해도 될까요?

경로: [전체 경로]
용도: [왜 필요한지]

승인하시겠습니까?"
```

### 사용자 확인 후 생성

```
사용자: "네, 만들어도 돼"
→ Claude Code가 폴더 생성

사용자: "아니, 기존 폴더 사용해"
→ Claude Code가 기존 폴더에 저장
```

## 폴더 구조 원칙

### 이미 정해진 구조 유지

```
기존 구조:
P0_작업_디렉토리_구조_생성/
S0_Project-SSAL-Grid_생성/
S1_개발_준비/
S2_개발-1차/
S3_개발-2차/
S4_개발-3차/
S5_운영/
Production/

→ 이 구조 안에서 작업
→ 새 최상위 폴더 생성 금지
```

### 하위 폴더 규칙

```
각 Stage 폴더 하위:
├── Frontend/       ← 이미 존재
├── Backend_API/    ← 이미 존재
├── Database/       ← 이미 존재
├── Documentation/  ← 이미 존재
├── Testing/        ← 이미 존재
└── Security/       ← 이미 존재

→ 이 안에서 작업
→ 새 영역 폴더 추가 시 승인 필수
```

## 폴더 정리 주기

### 권장 정리 시점

| 시점 | 정리 내용 |
|------|----------|
| 작업 완료 후 | 임시 파일 정리 |
| 커밋 전 | 불필요한 파일 확인 |
| Stage 완료 시 | 해당 Stage 폴더 정리 |
| 월 1회 | 전체 프로젝트 정리 |

### 정리 명령

```
"이번 작업에서 생성된 임시 파일 정리해줘"

"S2 폴더 정리하고 구조 보고해줘"

"전체 프로젝트 폴더 상태 점검해줘"
```

## 문제 발생 시 대응

### 임의 폴더가 생성된 경우

```
1. 폴더 내용 확인
2. 파일들을 올바른 위치로 이동
3. 빈 폴더 삭제
4. CLAUDE.md 규칙 강화
```

### Claude Code에게 지시

```
"임의로 생성된 [폴더명] 폴더의 파일들을
올바른 위치로 이동하고 빈 폴더 삭제해줘"
```

## 주의사항

- CLAUDE.md에 규칙을 명확히 작성해야 Claude Code가 따름
- 규칙 없이 작업하면 폴더가 무분별하게 생성됨
- 한 번 어지러워진 구조는 정리하기 어려움
- 예방이 치료보다 쉬움 - 처음부터 규칙 설정
- 팀 협업 시 폴더 규칙 공유 필수
