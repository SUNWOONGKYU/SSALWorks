# 반응형 디자인은 별도로 요청하기

## 핵심 요약

반응형 디자인과 모바일 최적화는 자동으로 되지 않습니다. "반응형으로 만들어줘", "모바일 최적화해줘"라고 명시적으로 요청해야 합니다.

## 왜 별도 요청이 필요한가?

### 기본 작업과 반응형 작업의 차이

```
❌ "로그인 페이지 만들어줘"
→ 데스크탑 크기에 맞춰 제작됨
→ 모바일에서 레이아웃 깨짐

✅ "로그인 페이지 반응형으로 만들어줘"
→ 모든 화면 크기에 대응
→ 모바일/태블릿/데스크탑 모두 정상 표시
```

### 반응형이 자동 적용되지 않는 이유

| 이유 | 설명 |
|------|------|
| 작업량 증가 | 반응형은 추가 CSS/레이아웃 작업 필요 |
| 디자인 결정 | 모바일용 레이아웃 변경 여부 결정 필요 |
| 우선순위 | 기능 구현과 반응형은 별개 작업 |
| 명시적 요구 | 사용자 요구사항에 따라 적용 여부 결정 |

## 반응형 요청 방법

### 1. 새 페이지 제작 시

```
"회원가입 페이지를 반응형으로 만들어줘"

"대시보드 페이지 만들어줘. 모바일/태블릿/데스크탑 반응형으로"
```

### 2. 기존 페이지 반응형 적용

```
"기존 로그인 페이지를 반응형으로 수정해줘"

"메인 페이지 모바일 최적화해줘"
```

### 3. 구체적인 브레이크포인트 지정

```
"768px 이하에서 사이드바 숨김 처리해줘"

"모바일(480px), 태블릿(768px), 데스크탑(1024px) 기준으로 반응형 적용해줘"
```

## 반응형 요청 시 포함할 내용

### 상세 요청 예시

```
"상품 목록 페이지를 반응형으로 만들어줘:

1. 데스크탑: 4열 그리드
2. 태블릿: 2열 그리드
3. 모바일: 1열 리스트

네비게이션은 모바일에서 햄버거 메뉴로 바꿔줘"
```

### 반응형 요소 지정

| 요소 | 모바일 처리 옵션 |
|------|------------------|
| 네비게이션 | 햄버거 메뉴, 하단 탭 바 |
| 사이드바 | 숨김, 오버레이, 하단 이동 |
| 테이블 | 카드형, 가로 스크롤, 축소 |
| 이미지 | 비율 유지, 숨김, 작게 |
| 폰트 | 크기 조정, vw 단위 사용 |

## 일반적인 브레이크포인트

```css
/* 모바일 */
@media (max-width: 480px) { }

/* 태블릿 */
@media (max-width: 768px) { }

/* 작은 데스크탑 */
@media (max-width: 1024px) { }

/* 큰 데스크탑 */
@media (min-width: 1200px) { }
```

### Claude Code에게 지시

```
"다음 브레이크포인트로 반응형 적용해줘:
- 모바일: 480px 이하
- 태블릿: 481px ~ 768px
- 데스크탑: 769px 이상"
```

## 모바일 우선 vs 데스크탑 우선

### 모바일 우선 (Mobile First)

```
"모바일 우선으로 반응형 만들어줘"

특징:
- 기본 CSS가 모바일용
- min-width 미디어쿼리 사용
- 점점 커지는 방향으로 스타일 추가
```

### 데스크탑 우선 (Desktop First)

```
"데스크탑 우선으로 반응형 만들어줘"

특징:
- 기본 CSS가 데스크탑용
- max-width 미디어쿼리 사용
- 점점 작아지는 방향으로 스타일 수정
```

## 반응형 검증 요청

```
"반응형이 잘 적용됐는지 확인해줘"

"모바일 크기에서 레이아웃 깨지는 부분 있어?"

"375px, 768px, 1440px에서 각각 어떻게 보이는지 설명해줘"
```

## 체크리스트

### 반응형 요청 시 확인사항

- [ ] 대상 기기 명시 (모바일/태블릿/데스크탑)
- [ ] 브레이크포인트 기준 결정
- [ ] 모바일용 네비게이션 방식 결정
- [ ] 사이드바 처리 방식 결정
- [ ] 이미지/미디어 처리 방식 결정
- [ ] 폰트 크기 조정 여부 결정

## 주의사항

- 반응형은 "기능"이 아니라 "디자인" 작업이므로 별도 요청 필수
- 처음부터 반응형으로 만드는 게 나중에 수정하는 것보다 효율적
- 모바일 사용자 비율이 높다면 모바일 우선 개발 고려
- 테스트는 실제 기기 또는 브라우저 개발자 도구로 확인
- Tailwind CSS 사용 시 반응형 클래스(sm:, md:, lg:) 활용 가능
