# 반응형 디자인은 별도로 요청하기

> 이 문서는 반응형 디자인 작업을 효과적으로 요청하는 방법을 설명합니다.

---

## 왜 별도로 요청해야 하는가

"로그인 페이지 만들어줘"라고만 요청하면 데스크톱 크기에 맞춰 제작됩니다. 모바일에서는 가로 스크롤이 생기고 버튼이 너무 작아집니다. 반응형은 별도 작업이므로 명시적으로 요청해야 합니다.

---

## 요청 방법

### 기본 요청

```
"로그인 페이지 반응형으로 만들어줘"
```

### 상세 요청

```
"상품 목록 페이지를 반응형으로 만들어줘:
- 데스크탑: 4열 그리드
- 태블릿: 2열 그리드
- 모바일: 1열 리스트
- 네비게이션은 모바일에서 햄버거 메뉴로"
```

---

## 주요 브레이크포인트

| 디바이스 | 화면 너비 | 특징 |
|----------|----------|------|
| 모바일 | ~ 480px | 세로 스크롤, 터치 |
| 태블릿 | 481px ~ 768px | 터치 + 넓은 화면 |
| 데스크톱 | 769px ~ | 마우스 인터페이스 |

---

## 흔한 실수

### viewport 메타 태그 누락

```html
<!-- 반드시 포함해야 함 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### 고정 너비 사용

```css
/* ❌ 잘못된 예 */
.container { width: 1200px; }

/* ✅ 올바른 예 */
.container { width: 100%; max-width: 1200px; }
```

### 작은 터치 영역

```css
/* ✅ 최소 44px 권장 */
.button { padding: 12px 24px; min-height: 44px; }
```

---

## 테스트 방법

1. F12 키로 개발자 도구 열기
2. 디바이스 툴바 버튼 클릭 (또는 Ctrl+Shift+M)
3. 다양한 디바이스 선택하여 확인

```
"로그인 페이지가 모바일에서 어떻게 보이는지 확인해줘"
```

---

## 체크리스트

### 요청 전
- [ ] 반응형이 필요한 페이지인가?
- [ ] 각 화면 크기에서 원하는 레이아웃을 정했는가?

### 구현 후
- [ ] 모바일에서 가로 스크롤이 없는가?
- [ ] 버튼이 터치하기 쉬운 크기인가?
- [ ] 텍스트가 읽기 쉬운 크기인가?
- [ ] viewport 메타 태그가 있는가?

---

*반응형 디자인은 "기능"이 아니라 "디자인" 작업입니다. 별도로 요청하세요.*
