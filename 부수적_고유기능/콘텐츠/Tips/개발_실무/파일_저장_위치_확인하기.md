# 파일 저장 위치 확인하기

> 이 문서는 SAL Grid에서 파일 저장 위치를 올바르게 결정하는 방법을 설명합니다.

---

## 기본 규칙: Task ID → 저장 위치

Task ID `S2BA1`에서:
- S2 → `S2_개발-1차/`
- BA → `Backend_APIs/`
- 저장 위치: `S2_개발-1차/Backend_APIs/`

---

## Stage 폴더

| Stage | 폴더명 | 설명 |
|-------|--------|------|
| S1 | `S1_개발_준비/` | 환경 설정, DB |
| S2 | `S2_개발-1차/` | 핵심 기능 |
| S3 | `S3_개발-2차/` | 추가 기능 |
| S4 | `S4_개발-3차/` | QA, 최적화 |
| S5 | `S5_운영/` | 배포 |

---

## Area 폴더

| Area | Stage 저장 | Production 저장 |
|------|:----------:|:---------------:|
| F (Frontend) | ✅ | ✅ |
| BA (Backend APIs) | ✅ | ✅ |
| S (Security) | ✅ | ✅ |
| BI (Backend Infra) | ✅ | ✅ |
| E (External) | ✅ | ✅ |
| D, M, U, T, O, C | ✅ | ❌ |

---

## 이중 저장 규칙

5개 Area(F, BA, S, BI, E)는 Stage + Production 양쪽에 저장:

```
Task: S2BA1

저장 위치:
1. S2_개발-1차/Backend_APIs/subscription-cancel.js    ← Stage
2. Production/api/Backend_APIs/subscription-cancel.js  ← Production
```

---

## S?_*/ 표기법

`S?_*/Frontend/` = 해당 Stage의 Frontend 폴더

```
S1F1 → S1_개발_준비/Frontend/
S2F1 → S2_개발-1차/Frontend/
```

---

## Claude Code에게 요청

```
"S2BA1 Task 파일 저장 위치가 어디야?"
"이 파일 Stage와 Production 양쪽에 저장해줘"
```

---

## 주의사항

- 임의 폴더 생성 금지 (사용자 승인 필수)
- Stage → Production 복사 시 파일명 동일하게 유지
- 저장 위치 불확실하면 저장 전에 확인
