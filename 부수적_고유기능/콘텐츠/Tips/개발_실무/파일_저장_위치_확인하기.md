# 코드 파일이 올바른 위치에 저장되었는지 확인하기

> 이 문서는 Claude Code가 생성한 파일이 올바른 위치에 저장되었는지 확인하는 방법을 설명합니다.

---

## 왜 위치 확인이 중요한가

| 문제 | 결과 |
|------|------|
| 잘못된 위치 | import 실패, 빌드 오류 |
| 중복 파일 | 어떤 게 최신인지 혼란 |
| 규칙 위반 | 프로젝트 구조 붕괴 |

---

## 저장 후 확인 방법

```
"방금 생성한 파일 위치 알려줘"
"Production 폴더에도 복사했어?"
"파일 경로 전체로 보여줘"
```

---

## 이중 저장 규칙 (5개 Area)

| Area | Stage 폴더 | Production 폴더 |
|------|-----------|-----------------|
| Frontend | S?_*/Frontend/ | Production/Frontend/ |
| Backend APIs | S?_*/Backend_APIs/ | Production/api/Backend_APIs/ |
| Security | S?_*/Security/ | Production/api/Security/ |
| Backend Infra | S?_*/Backend_Infra/ | Production/api/Backend_Infra/ |
| External | S?_*/External/ | Production/api/External/ |

---

## 확인 명령어

```
"git status 보여줘"  → 변경된 파일 경로 확인
"ls Production/Frontend/"  → 파일 존재 확인
```

---

## 흔한 실수

| 실수 | 올바른 방법 |
|------|------------|
| 루트에 저장 | 적절한 하위 폴더에 |
| Stage만 저장 | Production에도 복사 |
| 오타 폴더 생성 | 기존 폴더 확인 후 저장 |

---

## 저장 위치 요청

```
"login.html을 Production/Frontend/pages/auth/에 저장해줘"
"api/Backend_APIs/에 저장해줘"
```

---

## 위치 수정 요청

```
"방금 만든 파일이 잘못된 위치야. 올바른 위치로 이동해줘"
"루트에 있는 파일들 정리해줘"
```

---

## 자동 확인 요청

```
"파일 저장할 때마다 위치 알려줘"
"이중 저장이 필요한 파일이면 양쪽 다 저장해줘"
```

---

## 확인 습관

매 작업 후:
1. 파일 경로 확인
2. 규칙에 맞는지 검토
3. Production 복사 여부 확인
