# 파일 저장 위치 확인하기

## 핵심 요약

파일을 저장하기 전에 `Project_Directory_Structure.md`를 확인하고, 지정된 폴더에만 저장하세요. 임의 위치에 저장하면 나중에 찾기 어렵고 프로젝트 구조가 엉망이 됩니다.

## 왜 저장 위치가 중요한가?

### 잘못된 저장의 결과

```
❌ 임의 위치 저장:
파일 여기저기 흩어짐
→ "이 파일 어디 있지?" 반복
→ 프로젝트 구조 파악 어려움
→ 팀원과 협업 혼란
→ 유지보수 악몽

✅ 정해진 위치 저장:
파일이 예측 가능한 위치에
→ 파일 찾기 쉬움
→ 프로젝트 구조 깔끔
→ 협업 원활
→ 유지보수 용이
```

## 파일 유형별 저장 위치

### SSALWorks 프로젝트 기준

| 파일 유형 | 저장 위치 |
|----------|----------|
| 프론트엔드 코드 | `S?_*/Frontend/` + `Production/Frontend/` |
| 백엔드 API | `S?_*/Backend_API/` + `Production/Backend_API/` |
| 데이터베이스 | `S?_*/Database/` + `Production/Database/` |
| 문서 | `S?_*/Documentation/` |
| 테스트 | `S?_*/Testing/` |
| 보안 관련 | `S?_*/Security/` |
| 설정 파일 | 해당 Stage 폴더 내 |

### Stage 폴더 매핑

| Stage | 폴더명 | 설명 |
|-------|--------|------|
| S1 | S1_개발_준비 | 개발 환경 설정 |
| S2 | S2_개발-1차 | 핵심 기능 개발 |
| S3 | S3_개발-2차 | 고급 기능 개발 |
| S4 | S4_개발-3차 | QA 및 최적화 |
| S5 | S5_운영 | 배포 및 운영 |

## 저장 전 확인 절차

### 1. 파일 용도 파악

```
이 파일은 무엇인가?
- 프론트엔드 코드? → Frontend/
- API 코드? → Backend_API/
- DB 스키마? → Database/
- 문서? → Documentation/
```

### 2. 해당 Stage 확인

```
현재 어떤 Stage 작업인가?
- S1 작업? → S1_개발_준비/
- S2 작업? → S2_개발-1차/
```

### 3. 이중 저장 여부 확인

```
코드 파일(Frontend, Backend_API, Database)은
Stage 폴더 + Production 폴더 둘 다 저장
```


### 확인 요청

```
"이 파일 저장하기 전에 올바른 위치인지 확인해줘"

"Project_Directory_Structure.md 기준으로 저장 위치 맞는지 검토해줘"
```

## 자주 하는 실수

| 실수 | 올바른 위치 |
|------|------------|
| 루트에 파일 저장 | 해당 용도 폴더로 이동 |
| SQL을 Frontend에 | Database/ 폴더로 |
| 테스트를 코드와 함께 | Testing/ 폴더로 |
| 문서를 여기저기 | Documentation/ 폴더로 |
| Production만 저장 | Stage 폴더에도 저장 |

## 잘못 저장된 파일 발견 시

### 1. 위치 확인

```
"이 파일 현재 위치가 맞는지 확인해줘"
```

### 2. 이동 요청

```
"잘못된 위치의 파일들 올바른 폴더로 이동해줘"

"루트에 있는 임시 파일들 정리해줘"
```

### 3. 참조 수정

```
"파일 이동 후 import 경로도 수정해줘"
```

## 주의사항

- 루트 디렉토리에 파일 저장 금지
- 새 폴더 생성은 사용자 승인 후에만
- 문서와 실제 구조 일치 유지
- 파일 이동 시 참조(import) 수정 필수
- Production 폴더는 배포용 최신 코드만 유지
