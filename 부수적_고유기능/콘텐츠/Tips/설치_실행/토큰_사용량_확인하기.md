# 토큰 사용량 확인하기

## 핵심 요약

토큰 사용량과 리셋 시점이 궁금하면 `/status` 명령어를 입력하세요. Claude Code CLI 화면에서 현재 사용량과 리셋까지 남은 시간을 확인할 수 있습니다.

## 토큰이란?

Claude Code는 입력(프롬프트)과 출력(응답)에 사용되는 **토큰** 기준으로 사용량을 측정합니다.

### 토큰 계산 방식

- 영어: 약 4글자 = 1토큰
- 한글: 약 1-2글자 = 1토큰 (한글이 더 많은 토큰 소비)
- 코드: 약 3-4글자 = 1토큰

### 예시

```
"Hello, World!" ≈ 4 토큰
"안녕하세요" ≈ 5-6 토큰
```

## 사용량 확인 방법

### /status 명령어

Claude Code 실행 중에 입력:

```
/status
```

### 표시되는 정보

```
📊 사용량 정보
━━━━━━━━━━━━━━━━━━━━━━━━
• 현재 사용량: 150,000 / 500,000 토큰
• 사용률: 30%
• 리셋까지: 4시간 23분
• 구독: Claude Max
```

## 토큰 절약 팁

### 1. 불필요한 출력 줄이기

```
❌ "자세하게 설명해줘"
✅ "핵심만 간단히"
```

### 2. 컨텍스트 관리

- 긴 대화는 새 세션으로
- 필요 없는 파일 읽기 최소화
- 큰 파일은 필요한 부분만 참조

### 3. CLI 여러 개 사용

한 CLI에서 모든 작업을 하면 컨텍스트가 빠르게 쌓입니다.
- CLI 1: 프론트엔드
- CLI 2: 백엔드
- CLI 3: 문서화

작업 분리로 토큰 효율 향상!

### 4. 명확하고 구체적인 요청

```
❌ "이거 고쳐줘"
✅ "login.js의 validateEmail 함수에서 @ 검증 로직 수정해줘"
```

모호한 요청 → 추가 질문 → 토큰 낭비

## 토큰 한도에 도달했을 때

### 증상

```
⚠️ 토큰 한도에 도달했습니다.
리셋까지 3시간 42분 남았습니다.
```

### 대처 방법

1. **기다리기**: 리셋 시간까지 대기
2. **새 세션**: 긴 컨텍스트가 원인이면 새 세션 시작
3. **다른 CLI**: 여러 CLI를 번갈아 사용
4. **작업 분할**: 큰 작업을 작은 단위로 나누기

## 구독 플랜별 한도

| 플랜 | 토큰 한도 | 리셋 주기 |
|------|----------|----------|
| Claude Pro | 기본 | 매일 |
| Claude Max | 5x | 매일 |

정확한 한도는 Anthropic 공식 문서를 확인하세요.

## 주의사항

- 토큰 한도는 구독 플랜에 따라 다름
- 긴 코드 파일을 읽으면 토큰이 많이 소비됨
- 이미지 분석도 토큰을 사용함
- 리셋 시간은 UTC 기준일 수 있음
