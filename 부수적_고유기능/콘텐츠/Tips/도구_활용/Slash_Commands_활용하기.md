# Slash Commands 활용하기

## 핵심 요약

`/명령어` 형식으로 미리 정의된 작업을 실행할 수 있습니다. `.claude/commands/` 폴더에 마크다운 파일로 작성하세요. `/commit`, `/review`, `/test` 등 반복 작업을 자동화하면 일관성과 효율이 높아집니다.

## Slash Commands란?

Claude Code에서 자주 사용하는 작업을 **한 단어로 실행**할 수 있게 해주는 단축 명령어입니다. 마크다운 파일에 정의해두면 `/명령어`만 입력해서 복잡한 지시를 한 번에 전달할 수 있습니다.

### 비유로 이해하기

```
Slash Command = 매크로 키보드

매번 "Ctrl+C, Ctrl+V, Alt+Tab, Ctrl+V" 누르는 대신
"매크로 키 1번" 하나로 처리하는 것과 같음
```

## 왜 Slash Commands를 사용하나?

### 1. 반복 입력 제거

```
매번 이렇게 입력하는 대신:
"변경사항을 확인하고 의미 있는 커밋 메시지를 작성해서
커밋해줘. 커밋 메시지는 conventional commits 형식으로"

이것만 입력:
/commit
```

### 2. 일관성 유지

모든 커밋, 리뷰, 테스트가 동일한 기준으로 수행됩니다.

### 3. 실수 방지

정의된 절차대로 실행되므로 단계 누락이 없습니다.

## 명령어 정의 방법

### 파일 위치

```
.claude/commands/
├── commit.md
├── review.md
├── test.md
├── deploy.md
└── ...
```

### 기본 구조

```markdown
# commit

변경사항을 확인하고 커밋을 생성합니다.

## 절차

1. `git status`로 변경 파일 확인
2. `git diff`로 변경 내용 분석
3. Conventional Commits 형식으로 메시지 작성
4. `git add .` 후 커밋 실행

## 커밋 메시지 형식

- feat: 새 기능
- fix: 버그 수정
- docs: 문서 변경
- style: 코드 스타일
- refactor: 리팩토링
- test: 테스트
- chore: 기타 작업
```

## 실제 사용 예시

### /commit 사용

```
입력: /commit

실행되는 내용:
1. git status 확인
2. 변경 내용 분석
3. "feat: 회원가입 폼 유효성 검사 추가" 같은 메시지 생성
4. 커밋 실행
```

### /review 사용

```
입력: /review

실행되는 내용:
1. 최근 변경사항 분석
2. 코드 품질 검토
3. 잠재적 버그 확인
4. 개선 제안 제공
```

### /test 사용

```
입력: /test

실행되는 내용:
1. 테스트 파일 확인
2. npm test 또는 해당 테스트 명령 실행
3. 결과 분석 및 보고
```

## 자주 사용하는 명령어 예시

| 명령어 | 용도 | 설명 |
|--------|------|------|
| `/commit` | Git 커밋 | 변경사항 분석 후 커밋 |
| `/review` | 코드 리뷰 | 최근 변경 코드 검토 |
| `/test` | 테스트 실행 | 테스트 수행 및 결과 분석 |
| `/deploy` | 배포 | 빌드 및 배포 절차 실행 |
| `/status` | 상태 확인 | 프로젝트 현황 파악 |
| `/init` | 초기 설정 | 새 세션 시작 시 컨텍스트 로드 |

## 명령어 작성 팁

### 1. 명확한 절차 정의

```markdown
## 절차

1. 먼저 A를 확인한다
2. 그 다음 B를 실행한다
3. 결과를 C 형식으로 보고한다

(모호하지 않게 단계별로 명시)
```

### 2. 조건부 동작 정의

```markdown
## 조건

- 변경사항이 없으면: "변경사항 없음" 보고
- 테스트 실패 시: 실패 원인 분석 후 수정 제안
- 충돌 발생 시: 충돌 파일 목록과 해결 방법 제시
```

### 3. 출력 형식 지정

```markdown
## 출력 형식

```
[커밋 완료]
- 커밋 해시: abc1234
- 메시지: feat: 로그인 기능 추가
- 변경 파일: 3개
```
```

## 고급 활용: 인자 전달

### 인자가 있는 명령어

```markdown
# review-file

특정 파일을 집중 리뷰합니다.

## 사용법
/review-file [파일경로]

## 절차
1. 지정된 파일 읽기
2. 코드 품질 분석
3. 개선점 제안
```

### 사용 예시

```
/review-file src/components/LoginForm.js
```

## 프로젝트별 명령어 설정

### SSALWorks 프로젝트 예시

```
.claude/commands/
├── commit.md          # 커밋 자동화
├── review.md          # 코드 리뷰
├── test.md            # 테스트 실행
├── deploy.md          # Vercel 배포
├── status.md          # 프로젝트 상태 확인
├── worklog.md         # 작업 로그 업데이트
└── verify.md          # Task 검증 실행
```

## 주의사항

- 명령어 파일은 `.md` 확장자 필수
- 파일명이 곧 명령어 이름 (`commit.md` → `/commit`)
- 너무 많은 명령어는 오히려 혼란 (핵심만 정의)
- 명령어 내용은 명확하고 구체적으로 작성
- 팀 프로젝트에서는 명령어 목록 공유 필요
