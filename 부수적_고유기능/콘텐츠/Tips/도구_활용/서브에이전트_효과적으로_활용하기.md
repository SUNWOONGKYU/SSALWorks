# 서브에이전트 효과적으로 활용하기

> 이 문서는 서브에이전트를 활용하여 복잡한 작업을 분담하는 방법을 설명합니다.

---

## 서브에이전트란 무엇인가

복잡한 작업을 전문 역할별로 분담하는 가상의 팀원입니다. 작업자(Task Agent)가 코드를 작성하고, 검증자(Verification Agent)가 검토합니다.

---

## 주요 서브에이전트

| Task Agent | 역할 | Verification Agent |
|------------|------|-------------------|
| `frontend-developer` | UI/UX 개발 | `code-reviewer` |
| `backend-developer` | API 개발 | `code-reviewer` |
| `database-specialist` | DB 설계 | `qa-specialist` |
| `security-specialist` | 보안 설정 | `security-auditor` |
| `test-engineer` | 테스트 작성 | `qa-specialist` |
| `documentation-specialist` | 문서 작성 | `code-reviewer` |

---

## 작업자-검증자 분리 (필수)

```
❌ 잘못된 예:
backend-developer가 작성 → backend-developer가 검증

✅ 올바른 예:
backend-developer가 작성 → code-reviewer가 검증
```

**작업자와 검증자는 반드시 달라야 합니다.** 자기가 만든 코드를 자기가 검증하면 문제를 놓칩니다.

---

## 요청 방법

### 단일 작업

```
"backend-developer로 로그인 API 만들어줘"
```

### 작업 + 검증

```
"backend-developer로 로그인 API 만들고
code-reviewer로 검증해줘"
```

### 연속 작업

```
"다음 순서로 작업해줘:
1. database-specialist: users 테이블 설계
2. backend-developer: 회원가입 API 구현
3. test-engineer: 테스트 코드 작성
4. code-reviewer: 전체 검증"
```

---

## 주의사항

- 간단한 작업에 과도한 분업 피하기
- 복잡한 작업은 검증 단계 필수
- 작업 유형에 맞는 에이전트 선택
