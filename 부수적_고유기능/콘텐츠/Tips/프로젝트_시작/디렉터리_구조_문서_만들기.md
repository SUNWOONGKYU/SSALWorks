# 디렉터리 구조 문서 만들기

## 핵심 요약

`P0_프로젝트_구조/` 폴더에 `Project_Directory_Structure.md` 파일을 만들어 전체 폴더 구조를 문서화하세요. 파일을 어디에 저장해야 하는지 명확해집니다.

## 왜 디렉터리 구조 문서가 필요한가?

### 1. 파일 저장 위치 명확화

프로젝트가 커지면 "이 파일 어디에 저장하지?" 고민이 생깁니다. 디렉터리 구조 문서가 있으면:
- 파일 유형별 저장 위치가 명확함
- 팀원 간 일관된 파일 관리 가능
- 새로운 참여자도 빠르게 구조 파악

### 2. Claude Code와의 협업 효율 향상

Claude Code가 이 문서를 참조하면:
- 파일 생성 시 올바른 위치에 저장
- 불필요한 폴더 생성 방지
- 프로젝트 구조 일관성 유지

### 3. 프로젝트 유지보수 용이

시간이 지나도:
- 프로젝트 구조를 빠르게 파악
- 리팩토링 시 참고 가능
- 문서와 실제 구조 비교로 정리 필요 여부 판단

## 문서 작성 방법

### 기본 구조

```markdown
# 프로젝트 디렉토리 구조 가이드

> **버전**: v1.0
> **최종 업데이트**: 2025-01-15

---

## 전체 디렉토리 구조

project/
│
# ========== 예비단계 ==========
├── P0_프로젝트_구조/
├── P1_사업계획/
├── P2_프로젝트_기획/
├── P3_프로토타입/
│
# ========== 실행단계 ==========
├── S1_개발_준비/
├── S2_개발_1차/
├── S3_개발_2차/
├── S4_개발_3차/
├── S5_운영/
│
# ========== 배포용 ==========
├── Production/
│
# ========== 설정 ==========
└── .claude/

---

## 각 폴더 설명

### P0_프로젝트_구조/
프로젝트의 기초 문서를 관리합니다.
- Project_Directory_Structure.md: 이 문서
- Project_Status.md: 진행 상황

### S2_개발_1차/
핵심 기능 개발 단계의 산출물입니다.
- Frontend/: 프론트엔드 코드
- Backend_API/: API 코드
- Database/: DB 스키마
- Testing/: 테스트 코드

---

## 파일 저장 규칙

| 파일 유형 | 저장 위치 |
|----------|----------|
| HTML 페이지 | S?_*/Frontend/ + Production/Frontend/ |
| API 코드 | S?_*/Backend_API/ + Production/Backend_API/ |
| SQL 파일 | S?_*/Database/ + Production/Database/ |
| 테스트 코드 | S?_*/Testing/ |
```

### 포함해야 할 내용

| 섹션 | 내용 |
|------|------|
| 전체 폴더 트리 | 한눈에 보이는 구조도 |
| 각 폴더 설명 | 용도와 저장할 파일 유형 |
| 네이밍 규칙 | 파일/폴더 명명 규칙 |
| 파일 저장 규칙 | 어떤 파일을 어디에 저장하는지 |
| 주의사항 | 특별히 주의할 점 |

## 효과적인 활용 팁

### 1. CLAUDE.md에서 참조

```markdown
# CLAUDE.md
파일 저장 전 반드시 P0_프로젝트_구조/Project_Directory_Structure.md를 확인하세요.
```

### 2. 변경 시 즉시 업데이트

폴더 구조가 바뀌면 문서도 함께 수정하세요. 문서와 실제 구조가 다르면 혼란이 생깁니다.

### 3. 버전 관리

문서 상단에 버전과 업데이트 날짜를 기록해서 언제 수정됐는지 추적하세요.

## 주의사항

- 너무 깊은 중첩은 피하기 (3-4단계 권장)
- 폴더명은 용도가 명확하게
- 실제 구조와 문서 동기화 유지
- Production 폴더는 이중 저장 규칙 명시
