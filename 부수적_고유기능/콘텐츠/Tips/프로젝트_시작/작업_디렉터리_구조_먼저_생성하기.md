# 새 프로젝트 시작 시 코드 작성 전에 폴더 구조부터 만들기

> 이 문서는 코드 작성 전에 프로젝트 폴더 구조를 먼저 설계하고 생성하는 방법을 설명합니다.

---

## 왜 폴더 구조가 먼저인가

코드부터 작성하면 파일이 여기저기 흩어집니다. 나중에 정리하려면 더 많은 시간이 필요합니다. 처음부터 구조를 잡으면 파일 관리가 쉬워집니다.

---

## 올바른 순서

```
❌ 잘못된 순서
코드 작성 → 파일 여기저기 → 나중에 정리 → 혼란

✅ 올바른 순서
폴더 구조 설계 → 문서화 → 코드 작성 → 제자리 저장
```

---

## 기본 폴더 구조

```
프로젝트/
├── P0_프로젝트_구조/      # 기초 문서
├── P1_시장조사/           # 예비단계 1
├── P2_기획/               # 예비단계 2
├── P3_프로토타입/         # 예비단계 3
├── S1_개발_준비/          # 실행단계 1
├── S2_개발-1차/           # 실행단계 2
├── S3_개발-2차/           # 실행단계 3
├── S4_개발-3차/           # 실행단계 4
├── S5_운영/               # 실행단계 5
├── Production/            # 배포용 코드
└── .claude/               # Claude Code 설정
```

---

## 영역별 하위 폴더

| 영역 | 폴더명 | 설명 |
|------|--------|------|
| 프론트엔드 | Frontend/ | HTML, CSS, JS |
| 백엔드 API | Backend_API/ | API 코드 |
| 데이터베이스 | Database/ | SQL 파일 |
| 보안 | Security/ | 인증/인가 |
| 테스트 | Testing/ | 테스트 코드 |
| 문서 | Documentation/ | 기획/설계 문서 |
