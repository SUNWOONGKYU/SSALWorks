# 작업 디렉터리 구조 먼저 생성하기

## 요약
코드 작성 전에 폴더 구조부터 생성. 체계적인 구조 → 파일 관리 쉬움. 단계(P1~S5)별 + 영역(Frontend, Backend_API 등)별 분류.

## 상세

### 올바른 순서

```
X 코드 작성 → 파일 여기저기 → 나중에 정리 → 혼란
O 폴더 구조 설계 → 문서화 → 코드 작성 → 제자리 저장
```

### 기본 폴더 구조

```
프로젝트/
├── P0_프로젝트_구조/      # 기초 문서
├── P1~P3_*/               # 예비단계 (기획)
├── S1~S5_*/               # 실행단계 (개발)
├── Production/            # 배포용 코드
└── .claude/               # Claude Code 설정
```

### 영역별 하위 폴더

| 영역 | 폴더명 |
|------|--------|
| 프론트엔드 | Frontend/ |
| 백엔드 API | Backend_API/ |
| 데이터베이스 | Database/ |
| 보안 | Security/ |
| 테스트 | Testing/ |

### Claude Code에게 요청

```
"프로젝트를 시작합니다. 다음 구조로 폴더를 생성해주세요:
- P0_프로젝트_구조/
- S1_개발_준비/
- Production/
- .claude/
```
