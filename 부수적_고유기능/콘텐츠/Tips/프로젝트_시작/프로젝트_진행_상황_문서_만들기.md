# 프로젝트 진행 상황 문서 만들기

## 핵심 요약

`P0_프로젝트_구조/` 폴더에 `Project_Status.md` 파일을 만들어 현재 진행 상황을 기록하세요. 어디까지 완료됐고 다음에 무엇을 해야 하는지 파악할 수 있습니다.

## 왜 진행 상황 문서가 필요한가?

### 1. 현재 위치 파악

프로젝트가 길어지면 "지금 어디까지 했지?" 혼란이 생깁니다.
- 완료된 단계와 작업 목록
- 현재 진행 중인 작업
- 다음에 해야 할 작업

### 2. 세션 간 연속성 유지

Claude Code 세션이 바뀌어도:
- 이전 작업 내역 파악
- 중단된 지점부터 재개
- 불필요한 중복 작업 방지

### 3. 팀 협업 시 소통

여러 사람이 작업할 때:
- 누가 무엇을 했는지 파악
- 충돌 방지
- 전체 진행률 공유

## work_logs와의 차이점

| 항목 | Project_Status.md | work_logs/current.md |
|------|-------------------|---------------------|
| 목적 | 전체 진행 현황 | 상세 작업 기록 |
| 범위 | 프로젝트 전체 | 최근 세션 |
| 상세도 | 요약 | 상세 (코드, 파일 목록) |
| 업데이트 | 주요 마일스톤 | 매 작업 |
| 대상 | 전체 팀원/PO | 개발자/AI |

**둘 다 함께 사용하면 가장 효과적입니다.**

## 효과적인 관리 방법

### 1. 단계 완료 시마다 업데이트

```
S1 완료 → Project_Status.md 업데이트
- S1 진행률 100%로 변경
- S1 완료 사항 요약 추가
- S2 진행중으로 변경
```

### 2. CLAUDE.md에서 참조

```markdown
# CLAUDE.md
## 세션 시작 시 필수 확인
1. P0_프로젝트_구조/Project_Status.md 읽고 현재 상황 파악
```

### 3. 버전 관리

문서 상단에 버전과 업데이트 날짜를 기록해서 추적하세요.

## 주의사항

- 너무 상세하면 유지가 어려움 (요약 수준 유지)
- 완료된 작업도 기록 유지 (히스토리 추적)
- 날짜 기록 필수 (언제 업데이트했는지)
- 블로커는 명확하게 표시 (누가 해결해야 하는지)
