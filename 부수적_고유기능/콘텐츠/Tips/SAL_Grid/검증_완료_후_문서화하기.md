# 검증 완료 후 문서화하기

## 요약
작업 완료 → 검증 → 문서화 → 보고 순서로 마무리. outbox에 작업 결과와 검증 결과 JSON을 저장하고, Grid에 검증 정보를 기록한다.

## 상세

### 문서화 프로세스

```
작업 완료
    ↓
검증 수행 (Verification Agent)
    ↓
문서화
├── 작업 결과 보고서 (outbox)
├── 검증 결과 보고서 (outbox)
└── Grid 검증 필드 업데이트 (#16-21)
    ↓
사용자에게 완료 보고
```

### 파일명 규칙

```
작업 결과: {task_id}_{task_name}_completed.json
검증 결과: {task_id}_{task_name}_verification.json

예시:
- S2BA1_signup_api_completed.json
- S2BA1_signup_api_verification.json
```

### Grid 검증 필드

| 필드 | 내용 |
|------|------|
| #16 Test | 테스트 결과 (unit/integration/e2e) |
| #17 Build | 빌드 결과 (compile/lint/type_check) |
| #18 Integration | 의존성 연동 확인 |
| #19 Blockers | 차단 요소 |
| #20 Comprehensive | 종합 검증 결과 |
| #21 Note | 검증자 의견 |

### 주의
- 검증 없이 문서화 금지 (검증 → 문서화 순서)
- PO 승인 전까지 Stage Gate 통과 선언 금지
