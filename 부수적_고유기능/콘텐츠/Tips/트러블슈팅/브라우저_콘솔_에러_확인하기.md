# 브라우저 콘솔 에러 확인하기

## 핵심 요약

Claude Code가 에러를 보여달라고 할 때 브라우저 개발자 도구(F12 또는 Ctrl+Shift+J)를 열고, Console 탭에서 빨간색 에러 메시지를 복사해서 전달하세요. 정확한 에러 메시지가 문제 해결의 핵심입니다.

## 개발자 도구 여는 방법

### 단축키

| 브라우저 | Windows/Linux | Mac |
|----------|---------------|-----|
| Chrome | F12 또는 Ctrl+Shift+J | Cmd+Option+J |
| Firefox | F12 또는 Ctrl+Shift+K | Cmd+Option+K |
| Edge | F12 또는 Ctrl+Shift+J | Cmd+Option+J |
| Safari | - | Cmd+Option+C |

### 메뉴로 열기

```
Chrome:
1. 우측 상단 ⋮ 클릭
2. 도구 더보기 → 개발자 도구

또는:
1. 페이지에서 우클릭
2. "검사" 클릭
```

## Console 탭 이해하기

### 메시지 유형

| 색상/아이콘 | 유형 | 의미 |
|------------|------|------|
| 🔴 빨간색 | Error | 실행 중단되는 심각한 오류 |
| 🟡 노란색 | Warning | 경고 (실행은 됨) |
| 🔵 파란색 | Info | 정보성 메시지 |
| ⚫ 검은색 | Log | 일반 로그 (console.log) |

### 에러 메시지 구조

```
Uncaught TypeError: Cannot read property 'email' of undefined
    at handleSubmit (login.js:45:23)
    at HTMLButtonElement.<anonymous> (login.js:12:5)
```

```
구성 요소:
1. 에러 유형: TypeError
2. 에러 메시지: Cannot read property 'email' of undefined
3. 발생 위치: login.js 파일 45번째 줄
4. 호출 스택: 어디서 호출되었는지 추적
```

## 에러 복사 방법

### 기본 복사

```
1. Console 탭 열기
2. 빨간색 에러 메시지 클릭
3. 텍스트 전체 선택 (Ctrl+A 또는 드래그)
4. 복사 (Ctrl+C)
```

### 우클릭 복사

```
1. 에러 메시지 우클릭
2. "Copy" 또는 "메시지 복사" 선택
3. Claude Code에 붙여넣기
```

### 스택 트레이스 포함 복사

```
1. 에러 메시지 옆 ▶ 화살표 클릭 (펼치기)
2. 전체 스택 트레이스 보이면
3. 우클릭 → "Save as..." 또는 전체 복사
```

## 자주 보는 에러 유형

### 1. TypeError

```javascript
// 에러
Uncaught TypeError: Cannot read property 'X' of undefined

// 원인: 존재하지 않는 객체의 속성 접근
// 예: user가 undefined인데 user.email 접근

// 해결: null/undefined 체크
if (user && user.email) {
  // 안전하게 접근
}
```

### 2. ReferenceError

```javascript
// 에러
Uncaught ReferenceError: supabase is not defined

// 원인: 변수/함수가 정의되지 않음
// 예: Supabase SDK 로드 전에 사용

// 해결: 스크립트 로드 순서 확인
```

### 3. SyntaxError

```javascript
// 에러
Uncaught SyntaxError: Unexpected token '<'

// 원인: JavaScript 파일에 HTML 반환됨
// 예: 잘못된 파일 경로로 404 HTML 반환

// 해결: 스크립트 경로 확인
```

### 4. CORS 에러

```javascript
// 에러
Access to fetch at 'https://...' from origin 'http://...'
has been blocked by CORS policy

// 원인: 다른 도메인 API 호출 시 CORS 미설정
// 해결: 서버에서 CORS 헤더 추가 또는 같은 도메인 사용
```

### 5. Network 에러

```javascript
// 에러
Failed to fetch
// 또는
net::ERR_CONNECTION_REFUSED

// 원인: 서버 연결 실패
// 해결: 서버 실행 상태, URL, 네트워크 확인
```

## Claude Code에게 에러 전달하기

### 좋은 에러 보고 예시

```
"로그인 버튼 클릭하니까 이런 에러가 나와:

Uncaught TypeError: Cannot read property 'user' of null
    at handleLogin (login.js:34:15)
    at HTMLButtonElement.<anonymous> (login.js:8:3)

콘솔에 이것도 나와:
POST https://xxx.supabase.co/auth/v1/token 400 (Bad Request)"
```

### 필수 포함 정보

```
1. 에러 메시지 전체 (빨간색 텍스트)
2. 발생 시점 (어떤 동작 후?)
3. 파일명과 줄 번호
4. Network 탭 에러 (있다면)
```

### 나쁜 에러 보고 예시

```
❌ "에러 나요"
❌ "안 돼요"
❌ "빨간 글씨가 나와요"
```

## Network 탭 활용

### Network 탭 확인 방법

```
1. 개발자 도구 → Network 탭
2. 페이지 새로고침
3. 빨간색 요청 확인 (실패한 요청)
4. 요청 클릭 → Response 탭에서 에러 내용 확인
```

### API 에러 확인

```
1. Network 탭에서 실패한 요청 클릭
2. Headers 탭: 요청 정보 확인
3. Response 탭: 서버 응답 확인
4. 응답 내용 복사하여 Claude Code에 전달
```

### 상태 코드 의미

| 코드 | 의미 | 일반적 원인 |
|------|------|-------------|
| 400 | Bad Request | 잘못된 요청 데이터 |
| 401 | Unauthorized | 인증 필요/실패 |
| 403 | Forbidden | 권한 없음 |
| 404 | Not Found | 잘못된 URL |
| 500 | Server Error | 서버 오류 |

## 에러 필터링

### 특정 에러만 보기

```
Console 탭 상단 필터:
- "Errors" 클릭: 에러만 표시
- "Warnings" 클릭: 경고만 표시
- 검색창: 특정 키워드로 필터
```

### 소스별 필터

```
Console 탭 우측:
- "user messages" 체크: console.log만
- 파일명 필터: 특정 파일의 로그만
```

## 디버깅 팁

### Sources 탭에서 중단점 설정

```
1. Sources 탭 → 파일 선택
2. 줄 번호 클릭 (파란 마커 표시)
3. 페이지 새로고침
4. 해당 줄에서 실행 멈춤
5. 변수 값 확인 가능
```

### 에러 발생 시 자동 중단

```
1. Sources 탭 → 우측 패널
2. "Pause on exceptions" 클릭 (⏸ 아이콘)
3. 에러 발생 시 자동으로 해당 줄에서 멈춤
```

## 체크리스트

### 에러 발생 시 확인

- [ ] Console 탭에서 빨간색 에러 확인
- [ ] 에러 메시지 전체 복사
- [ ] 파일명과 줄 번호 확인
- [ ] Network 탭에서 실패한 요청 확인
- [ ] 상태 코드와 응답 내용 확인

### Claude Code에 전달

- [ ] 에러 메시지 전체 복사하여 전달
- [ ] 어떤 동작에서 발생했는지 설명
- [ ] Network 에러가 있다면 함께 전달
- [ ] 스크린샷 필요 시 첨부

## 주의사항

- 에러를 무시하고 넘어가지 않기
- 경고(Warning)도 가능하면 해결하기
- 콘솔 로그가 너무 많으면 필터 사용
- 민감한 정보가 콘솔에 노출되지 않도록 주의
- 프로덕션에서는 console.log 제거 또는 최소화
