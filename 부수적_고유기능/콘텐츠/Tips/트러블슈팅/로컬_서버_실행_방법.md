# 로컬 서버 실행 방법

## 요약
HTML 테스트 시 로컬 서버 필요. `npx serve` (권장), `python -m http.server`, VS Code Live Server 중 선택. 포트 충돌 시 다른 번호 지정.

## 상세

### 왜 서버가 필요한가?

파일을 더블클릭해서 열면 주소가 `file://`로 시작한다. 이 방식은 보안 제한 때문에 많은 기능이 작동하지 않는다.

| 기능 | file:// | http:// (서버) |
|------|:-------:|:--------------:|
| CORS 요청 | X | O |
| ES 모듈 import | X | O |
| fetch API | X | O |
| Supabase SDK | X | O |

### 실행 방법 비교

| 방법 | 설치 | 자동 새로고침 | 명령어 |
|------|:----:|:------------:|--------|
| npx serve | 불필요 | X | `npx serve 폴더명` |
| Python | 불필요 | X | `python -m http.server` |
| Live Server | 확장 설치 | O | VS Code에서 클릭 |

### 기본 명령어

```bash
# npx serve (권장) - 폴더와 포트 지정
npx serve Production/Frontend -l 3000

# Python - 현재 폴더를 8000번 포트로
python -m http.server 8000
```

### 포트 충돌이란?

포트 번호는 한 번에 하나의 프로그램만 사용할 수 있다. 이미 3000번을 다른 프로그램이 쓰고 있으면 에러가 발생한다.

```
Error: listen EADDRINUSE: address already in use :::3000
```

### 포트 충돌 해결

```bash
# 방법 1: 다른 포트 사용 (가장 간단)
npx serve -l 8080

# 방법 2: 사용 중인 프로세스 찾아서 종료 (Windows)
netstat -ano | findstr :3000    # PID 확인
taskkill /PID 12345 /F          # 해당 PID 종료

# 방법 3: Mac/Linux
lsof -i :3000                   # PID 확인
kill -9 12345                   # 해당 PID 종료
```
