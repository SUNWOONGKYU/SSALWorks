# 환경변수 관리 베스트 프랙티스

## 요약
.env.example은 Git에 커밋(빈 값), .env.local은 gitignore에 추가. 설정: .env.example 복사 → .env.local 생성 → 실제 키 입력 → 서버 재시작.

## 상세

### 파일 구조

| 파일 | Git 커밋 | 용도 |
|------|----------|------|
| `.env.example` | O | 필요한 변수 목록 (빈 값) |
| `.env.local` | X | 로컬 개발용 실제 값 |

### .env.example 예시

```bash
# Supabase
SUPABASE_URL=
SUPABASE_ANON_KEY=

# OpenAI
OPENAI_API_KEY=

# 앱 설정
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### .gitignore 설정

```gitignore
.env
.env.*
!.env.example
```

### 설정 순서

```bash
# 1. 복사
cp .env.example .env.local

# 2. 편집
code .env.local

# 3. 서버 재시작 (필수!)
npm run dev
```

### 브라우저 접두사

| 프레임워크 | 접두사 |
|-----------|--------|
| Next.js | `NEXT_PUBLIC_` |
| Vite | `VITE_` |
| CRA | `REACT_APP_` |
