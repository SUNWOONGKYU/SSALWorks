# 중요한 파일 주기적으로 백업하기

## 핵심 요약

혹시 모를 상황에 대비해 중요한 파일은 별도로 백업해두세요. 클라우드 저장소나 외부 드라이브에 복사해두면 안전합니다. Git과 별개로 정기 백업 습관을 들이세요.

## 왜 백업이 필요한가?

### Git만으로는 부족한 경우

```
Git이 있어도 위험한 상황:
- 하드디스크 고장 → 로컬 .git 폴더 손실
- 랜섬웨어 감염 → 모든 파일 암호화
- 실수로 전체 폴더 삭제
- GitHub 계정 해킹/정지
- 인터넷 없이 작업해야 할 때
```

### 백업의 목적

```
Git = 버전 관리 (변경 이력)
백업 = 재난 복구 (데이터 보존)

둘 다 필요합니다!
```

## 백업 대상

### 필수 백업 파일

| 파일/폴더 | 중요도 | 이유 |
|----------|--------|------|
| `.claude/` | 매우 높음 | 작업 로그, 설정 |
| `CLAUDE.md` | 매우 높음 | 프로젝트 규칙 |
| `Production/` | 매우 높음 | 배포용 코드 |
| `P0_*/` 문서 | 높음 | 프로젝트 구조/상태 |
| `.env.example` | 높음 | 환경변수 템플릿 |
| `Database/*.sql` | 높음 | DB 스키마 |

### 백업 불필요한 것

```
❌ 백업 불필요:
- node_modules/ (npm install로 복구)
- dist/, build/ (빌드로 재생성)
- .cache/ (캐시 파일)
- 로그 파일 (*.log)
```

## 백업 방법

### 방법 1: 클라우드 동기화 (가장 쉬움)

#### Google Drive / OneDrive / Dropbox

```
1. 클라우드 앱 설치
2. 동기화 폴더 설정
3. 프로젝트 폴더를 동기화 대상에 포함
4. 자동 백업 (실시간)
```

**장점**: 설정 후 자동, 버전 이력 제공
**단점**: 용량 제한, 인터넷 필요

### 방법 2: 수동 복사

#### 정기적으로 복사

```bash
# Windows
xcopy /E /I "C:\Projects\my-project" "D:\Backup\my-project_%date%"

# Mac/Linux
cp -r ~/Projects/my-project ~/Backup/my-project_$(date +%Y%m%d)
```

**장점**: 인터넷 불필요, 완전한 통제
**단점**: 수동 작업 필요

### 방법 3: 외부 드라이브

```
1. USB 드라이브 또는 외장 HDD 준비
2. 주기적으로 프로젝트 폴더 복사
3. 물리적으로 분리 보관
```

**장점**: 랜섬웨어로부터 안전 (분리 보관 시)
**단점**: 물리적 분실/고장 위험

### 방법 4: NAS (네트워크 저장소)

```
1. NAS 장비 설정
2. 백업 스케줄 설정
3. 자동으로 정기 백업
```

**장점**: 대용량, 자동화, 로컬 네트워크
**단점**: 초기 비용, 설정 필요

## 백업 주기

### 권장 주기

| 백업 유형 | 주기 | 대상 |
|----------|------|------|
| 실시간 동기화 | 상시 | 클라우드 |
| 일일 백업 | 매일 | 작업 로그, 코드 |
| 주간 백업 | 매주 | 전체 프로젝트 |
| 월간 백업 | 매월 | 압축 아카이브 |

### 백업 타이밍

```
✅ 백업하기 좋은 시점:
- 퇴근 전 / 작업 종료 시
- 중요한 기능 완료 후
- 배포 전
- 주말 시작 전

❌ 피해야 할 시점:
- 작업 중간 (불완전한 상태)
- 빌드 에러가 있을 때
```

## 백업 검증

### 백업 테스트

```
정기적으로 복구 테스트:

1. 백업 파일에서 새 폴더로 복구
2. 프로젝트가 정상 동작하는지 확인
3. 빌드/테스트 실행
4. 문제 없으면 OK
```

### 체크리스트

- [ ] 백업 파일이 최신인가?
- [ ] 모든 중요 파일이 포함되어 있는가?
- [ ] 복구 테스트를 해봤는가?
- [ ] 백업 위치가 안전한가?

## 3-2-1 백업 원칙

### 원칙

```
3: 데이터 3개 복사본 유지
2: 2개 이상의 다른 저장 매체 사용
1: 1개는 물리적으로 다른 장소에 보관
```

### 실천 예시

```
복사본 1: 로컬 PC (작업 중인 파일)
복사본 2: 클라우드 (Google Drive 동기화)
복사본 3: 외장 HDD (주간 백업, 별도 장소 보관)
```

## 백업 자동화

### Windows 작업 스케줄러

```
1. 작업 스케줄러 열기
2. 새 작업 만들기
3. 트리거: 매일 저녁 6시
4. 동작: 백업 스크립트 실행
```

### 백업 스크립트 예시

```batch
@echo off
set BACKUP_DIR=D:\Backup\project_%date:~0,4%%date:~5,2%%date:~8,2%
xcopy /E /I /Y "C:\Projects\my-project" "%BACKUP_DIR%"
echo 백업 완료: %BACKUP_DIR%
```

## 복구 절차

### 로컬 파일 손실 시

```
1. 최신 백업 위치 확인
2. 백업에서 프로젝트 폴더로 복사
3. npm install (의존성 설치)
4. 동작 확인
```

### 전체 복구 시

```
1. Git clone (GitHub에서)
2. 환경변수 설정 (.env)
3. npm install
4. 백업에서 추가 파일 복사 (로그, 설정 등)
5. 빌드 및 테스트
```

## 주의사항

- 백업에도 민감정보 포함 주의 (.env 등)
- 클라우드 백업 시 암호화 권장
- 외장 드라이브는 정기적으로 상태 확인
- 백업 위치를 팀원과 공유
- 복구 절차를 문서화해두기
- 백업은 "했다"가 아니라 "복구할 수 있다"가 중요
