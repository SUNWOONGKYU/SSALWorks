# 중요한 파일 주기적으로 백업하기

## 요약
Git과 별개로 중요 파일을 클라우드나 외부 드라이브에 백업. 3-2-1 원칙: 3개 복사본, 2개 다른 매체, 1개는 다른 장소.

## 상세

### 필수 백업 대상

| 파일/폴더 | 중요도 |
|----------|--------|
| `.claude/` | 매우 높음 |
| `CLAUDE.md` | 매우 높음 |
| `Production/` | 매우 높음 |
| `Database/*.sql` | 높음 |

### 백업 불필요

```
node_modules/  (npm install로 복구)
dist/, build/  (빌드로 재생성)
.cache/        (캐시 파일)
```

### 백업 방법

| 방법 | 장점 | 단점 |
|------|------|------|
| 클라우드 동기화 | 자동, 실시간 | 용량 제한 |
| 외부 드라이브 | 랜섬웨어 안전 | 수동 필요 |
| NAS | 대용량, 자동화 | 초기 비용 |

### 권장 백업 주기

| 주기 | 대상 |
|------|------|
| 실시간 | 클라우드 동기화 |
| 매일 | 작업 로그, 코드 |
| 매주 | 전체 프로젝트 |

### 3-2-1 원칙 실천

```
복사본 1: 로컬 PC
복사본 2: 클라우드 (Google Drive)
복사본 3: 외장 HDD (별도 보관)
```
