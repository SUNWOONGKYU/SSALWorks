# work_logs로 작업 기록 관리하기

## 핵심 요약

`.claude/work_logs/current.md` 파일에 작업 내역을 기록하세요. 무엇을 했는지, 다음에 무엇을 해야 하는지, 중요한 참고사항을 적어두면 세션이 끊어져도 다음 세션에서 바로 이어서 작업할 수 있습니다.

## work_logs란?

Claude Code 작업의 **연속성을 보장하는 기록 시스템**입니다. 세션이 바뀌어도 이전 작업 내역을 파악하고 이어서 작업할 수 있게 해줍니다.

### 왜 필요한가?

```
work_logs 없이:
세션 1: 로그인 기능 완료
세션 2: "뭘 했더라...? 어디까지 했더라...?"

work_logs 사용:
세션 1: 로그인 기능 완료 → work_logs에 기록
세션 2: work_logs 읽기 → "로그인 완료, 다음은 회원가입" → 바로 시작
```

## 폴더 구조

```
.claude/
└── work_logs/
    ├── current.md        # 현재 활성 로그 (필수)
    └── archive/          # 과거 로그 보관
        ├── 2025-01-10.md
        ├── 2025-01-11.md
        └── ...
```

## current.md 작성 방법

### 기본 구조

```markdown
# Work Log - Current

> 최종 업데이트: 2025-01-15 14:30

## 현재 상태
- 현재 단계: S2 개발-1차
- 진행 중인 Task: S2F1 (로그인 UI)
- 다음 예정: S2BA1 (로그인 API)

## 최근 작업 내역

### 2025-01-15

#### 완료된 작업
- [x] 로그인 페이지 UI 구현
- [x] 이메일/비밀번호 입력 폼
- [x] 로그인 버튼 스타일링

#### 생성/수정된 파일
- `Production/Frontend/pages/auth/login.html`
- `Production/Frontend/css/auth.css`

#### 다음 작업
- [ ] 로그인 API 연동 (S2BA1)
- [ ] 에러 메시지 표시

#### 참고사항
- Supabase Auth 사용 예정
- Google OAuth는 별도 Task에서 처리

---

### 2025-01-14
(이전 작업 내역...)
```

### 포함해야 할 내용

| 섹션 | 내용 | 중요도 |
|------|------|--------|
| 현재 상태 | 단계, 진행 중 Task | 필수 |
| 완료된 작업 | 오늘 끝낸 것들 | 필수 |
| 생성/수정된 파일 | 파일 경로 목록 | 필수 |
| 다음 작업 | 앞으로 해야 할 것 | 필수 |
| 참고사항 | 중요한 결정사항, 주의점 | 권장 |
| 블로커 | 막힌 부분, 필요한 것 | 상황별 |

## 작업 기록 시점

### 언제 기록하나?

```
✅ 기록해야 하는 시점:
- Task 완료 시
- 파일 생성/수정 시
- 중요한 결정을 내렸을 때
- 세션 종료 전
- 문제 발생 시

❌ 기록 불필요:
- 간단한 오타 수정
- 임시 테스트 파일
- 바로 되돌릴 변경
```

### 기록 예시

**Task 완료 시:**
```markdown
#### 완료된 작업
- [x] S2F1: 로그인 UI 구현 완료
  - 이메일/비밀번호 폼
  - 유효성 검사 UI
  - 반응형 레이아웃
```

**문제 발생 시:**
```markdown
#### 블로커
- ⚠️ Google OAuth: Supabase Provider 설정 필요 (PO 작업)
- ⚠️ 환경변수 SUPABASE_URL 미설정
```

## 자동 순환 규칙

### 파일 크기 관리

```
current.md가 50KB 초과 시:
1. current.md → 2025-01-15.md로 이름 변경
2. 새로운 current.md 생성
3. 이전 로그 링크 추가
```

### 아카이브 규칙

```
30일 이상 된 로그:
work_logs/2025-01-10.md → work_logs/archive/2025-01-10.md
```

## 세션 시작/종료 체크리스트

### 세션 시작 시

```
1. .claude/work_logs/current.md 읽기
2. "현재 상태" 섹션 확인
3. "다음 작업" 섹션 확인
4. 이어서 작업 시작
```

### 세션 종료 시

```
1. 오늘 완료한 작업 기록
2. 생성/수정된 파일 목록 추가
3. 다음 작업 예정 기록
4. 중요한 참고사항 메모
5. 최종 업데이트 시간 갱신
```

## work_logs vs Project_Status

| 항목 | work_logs/current.md | Project_Status.md |
|------|---------------------|-------------------|
| 목적 | 상세 작업 기록 | 전체 진행 현황 |
| 범위 | 최근 세션 | 프로젝트 전체 |
| 상세도 | 상세 (파일, 코드) | 요약 |
| 업데이트 | 매 작업 | 주요 마일스톤 |
| 대상 | AI/개발자 | 전체 팀 |

**둘 다 함께 사용하면 가장 효과적입니다.**

## CLAUDE.md와 연계

```markdown
# CLAUDE.md

## 세션 시작 시 필수 확인
1. `.claude/work_logs/current.md` 읽기 ← 작업 기록
2. `P0_프로젝트_구조/Project_Status.md` 확인 ← 전체 현황
```

## 주의사항

- 너무 상세하게 적으면 읽기 어려움 (적정 분량 유지)
- 파일 경로는 정확하게 기록
- 완료/미완료 구분 명확히 (체크박스 활용)
- 민감정보(API 키 등)는 기록하지 않기
- 정기적으로 오래된 로그 아카이브
