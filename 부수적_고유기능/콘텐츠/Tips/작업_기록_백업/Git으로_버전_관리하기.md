# git add, commit, push로 코드 변경 이력 관리하고 실수 복구하기

> 이 문서는 Git을 활용하여 코드 변경 이력을 관리하고 실수를 복구하는 방법을 설명합니다.

---

## Git이란

파일 변경 이력을 기록하는 버전 관리 시스템입니다. 실수해도 이전 버전으로 복구할 수 있고, 여러 사람이 동시에 작업할 수 있습니다.

---

## 자주 사용하는 명령어

| 명령어 | 설명 |
|--------|------|
| `git status` | 현재 상태 확인 |
| `git add .` | 모든 변경 파일 스테이징 |
| `git commit -m "메시지"` | 변경사항 저장 |
| `git log --oneline` | 커밋 이력 보기 |
| `git diff` | 변경 내용 보기 |

---

## 커밋 메시지 형식

```
feat: 새 기능 추가
fix: 버그 수정
docs: 문서 수정
refactor: 코드 구조 개선
chore: 빌드/설정 변경
```

예시: `git commit -m "feat: 로그인 API 구현"`

---

## 실수 복구

```bash
# 파일 하나 복구 (마지막 커밋 상태로)
git checkout -- login.js

# 마지막 커밋 취소 (파일 유지)
git reset --soft HEAD~1

# 완전히 되돌리기 (주의: 변경사항 삭제됨)
git reset --hard HEAD~1
```

---

## 커밋 시점

| 좋은 시점 | 피할 시점 |
|----------|----------|
| 기능 완료 시 | 동작 안 하는 코드 |
| 버그 수정 후 | 민감정보 포함 |
| 작업 중단 전 | 임시 디버깅 코드 |
