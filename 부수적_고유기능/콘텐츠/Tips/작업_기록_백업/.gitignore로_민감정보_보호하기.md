# .gitignore로 민감정보 보호하기

## 요약
`.gitignore`에 민감정보 파일 등록 → 커밋에서 제외. `.env`, `credentials.json` 등은 절대 GitHub에 올리지 마세요. 한 번 노출된 키는 즉시 폐기 후 재발급.

## 상세

### 반드시 .gitignore에 추가할 것

| 유형 | 파일 예시 |
|------|----------|
| API 키 | `.env`, `config.json` |
| 인증 정보 | `credentials.json`, `*.pem` |
| 개인 키 | `id_rsa`, `*.key` |
| 데이터베이스 | `*.db`, `*.sqlite` |

### 기본 .gitignore 예시

```gitignore
# 환경 변수
.env
.env.local

# 인증/키
*.pem
*.key
credentials.json

# 의존성/빌드
node_modules/
dist/
```

### 이미 커밋된 파일 제거

```bash
git rm --cached .env
git commit -m "chore: 민감정보 파일 추적 제거"
```

### .env.example 활용

```
.env (gitignore에 추가, 실제 값)
.env.example (커밋됨, 빈 값 템플릿)
```
