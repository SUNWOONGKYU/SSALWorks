# 단계별로 요청하기

## 핵심 요약

큰 작업은 한 번에 다 시키지 말고 단계별로 나눠서 요청하세요. 각 단계 완료를 확인하고 다음으로 진행하면 품질이 높아지고 문제 발생 시 빠르게 대응할 수 있습니다.

## 왜 단계별로 요청해야 하나?

### 한 번에 요청 vs 단계별 요청

```
❌ 한 번에 요청:
"회원가입, 로그인, 마이페이지, 결제 기능 다 만들어줘"
→ 작업량 과다
→ 중간에 문제 발생 시 어디서 잘못됐는지 파악 어려움
→ 품질 저하 가능

✅ 단계별 요청:
"1단계: 회원가입 기능 만들어줘"
→ 완료 확인
"2단계: 로그인 기능 만들어줘"
→ 완료 확인
→ 각 단계별 품질 보장
→ 문제 발생 시 즉시 대응 가능
```

### 단계별 요청의 장점

| 장점 | 설명 |
|------|------|
| 품질 향상 | 각 단계를 꼼꼼하게 처리 |
| 빠른 피드백 | 문제 즉시 발견 및 수정 |
| 방향 조정 | 진행 중 요구사항 변경 가능 |
| 진행 파악 | 어디까지 완료됐는지 명확 |
| 컨텍스트 유지 | 복잡도 관리 용이 |

## 작업 분할 방법

### 기능 단위 분할

```
전체 작업: 쇼핑몰 만들기

1단계: 상품 목록 페이지
2단계: 상품 상세 페이지
3단계: 장바구니 기능
4단계: 결제 기능
5단계: 주문 내역 페이지
```

### 개발 흐름 분할

```
전체 작업: 로그인 기능 구현

1단계: UI 구현 (폼, 버튼)
2단계: API 연동
3단계: 상태 관리
4단계: 에러 핸들링
5단계: 테스트
```

### 복잡도 분할

```
전체 작업: 검색 기능

1단계: 기본 검색 (키워드 매칭)
2단계: 필터 추가 (카테고리, 가격)
3단계: 정렬 기능
4단계: 검색어 자동완성
5단계: 검색 히스토리
```

## 요청 방법

### 전체 계획 먼저

```
"회원 관리 기능을 구현하려고 해.
다음 순서로 진행하자:

1. 회원가입 폼 UI
2. 회원가입 API 연동
3. 로그인 기능
4. 비밀번호 찾기
5. 프로필 수정

우선 1단계부터 시작해줘"
```

### 단계별 지시

```
[1단계 완료 후]
"잘 됐어. 2단계 API 연동 진행해줘"

[2단계 완료 후]
"좋아. 근데 에러 메시지 한글로 바꿔줘.
그리고 3단계 로그인 기능 시작해줘"
```

### 검증 포함

```
"1단계 완료 후 다음 확인해줘:
- 폼 유효성 검사 동작하는지
- 필수 필드 표시되는지
- 에러 메시지 표시되는지

확인되면 2단계 진행해줘"
```

## 단계 관리 방법

### 체크리스트 활용

```
"다음 체크리스트로 진행해줘:

□ 1단계: DB 스키마 설계
□ 2단계: API 엔드포인트 구현
□ 3단계: 프론트엔드 연동
□ 4단계: 테스트 작성
□ 5단계: 문서화

각 단계 완료 시 알려줘"
```

### 마일스톤 설정

```
"다음 마일스톤으로 나눠서 진행해줘:

마일스톤 1: 기본 CRUD (생성, 읽기, 수정, 삭제)
마일스톤 2: 검색 및 필터
마일스톤 3: 권한 관리
마일스톤 4: 최적화

마일스톤 1부터 시작해줘"
```

## 단계 완료 확인

### 확인 요청

```
"1단계 완료된 것 같아. 확인해줘:
- 모든 파일 저장됐어?
- 테스트 통과해?
- 다음 단계로 넘어가도 돼?"
```

### 중간 점검

```
"지금까지 진행 상황 정리해줘:
- 완료된 단계
- 현재 진행 중인 단계
- 남은 단계
- 발견된 이슈"
```

## 방향 수정 방법

### 요구사항 변경

```
"2단계까지 잘 했어.
근데 기획이 바뀌어서 3단계 전에 알림 기능 먼저 해야 해.

수정된 순서:
3단계: 알림 기능 (새로 추가)
4단계: 기존 3단계 (결제)
5단계: 기존 4단계 (주문내역)

3단계 알림 기능부터 진행해줘"
```

### 문제 발생 시

```
"2단계에서 문제가 생겼어.
API 응답이 예상과 다르네.

2단계 잠시 멈추고,
원인 분석부터 해줘.
해결되면 다시 2단계 계속하자"
```

## 실제 예시

### 예시: 블로그 기능 구현

```
[요청]
"블로그 기능을 단계별로 구현하자:

1단계: 글 목록 페이지 (UI만)
2단계: 글 목록 API 연동
3단계: 글 상세 페이지
4단계: 글 작성 기능
5단계: 글 수정/삭제
6단계: 댓글 기능

1단계부터 시작해줘"

[진행]
→ 1단계 완료 → 확인 → 승인
→ 2단계 완료 → 페이지네이션 추가 요청
→ 2단계 수정 완료 → 확인 → 승인
→ 3단계 진행...
```

## 단계 분할 체크리스트

### 작업 분할 시 확인사항

- [ ] 각 단계가 독립적으로 테스트 가능한가?
- [ ] 단계별 목표가 명확한가?
- [ ] 단계 순서가 논리적인가?
- [ ] 한 단계가 너무 크지 않은가?
- [ ] 의존성이 명확한가?

## 주의사항

- 단계를 너무 잘게 나누면 오히려 비효율
- 한 단계는 의미 있는 결과물이 나와야 함
- 단계 간 의존성 고려
- 완료 기준을 명확히
- 중간에 방향 수정 가능함을 인지
