# 단계별로 요청하기

## 요약
큰 작업은 단계별로 나눠서 요청. 각 단계 완료 확인 후 다음 진행. 품질 향상 + 문제 즉시 대응 가능.

## 상세

### 한 번에 vs 단계별

```
X "회원가입, 로그인, 결제 다 만들어줘"
  → 중간에 문제 발생 시 어디서 잘못됐는지 파악 어려움

O "1단계: 회원가입 만들어줘" → 완료 확인
  "2단계: 로그인 만들어줘" → 완료 확인
  → 각 단계별 품질 보장
```

### 작업 분할 방법

```
전체: 로그인 기능

1단계: UI 구현 (폼, 버튼)
2단계: API 연동
3단계: 에러 핸들링
4단계: 테스트
```

### 요청 방법

```
"회원 관리 기능을 구현하려고 해.
다음 순서로 진행하자:
1. 회원가입 폼 UI
2. 회원가입 API 연동
3. 로그인 기능

우선 1단계부터 시작해줘"
```

### 단계 완료 확인

```
"1단계 완료된 것 같아. 확인해줘:
- 모든 파일 저장됐어?
- 테스트 통과해?
- 다음 단계로 넘어가도 돼?"
```
