# 이해될 때까지 질문하기

## 핵심 요약

Claude Code가 제시한 내용 중 이해가 안 되는 것이 있으면 이해될 때까지 계속 질문하세요. "왜 그렇게 하는 거야?", "더 쉽게 설명해줘"라고 물어보면 됩니다.

## 왜 이해될 때까지 질문해야 하나?

### 이해 없이 진행 vs 이해 후 진행

```
❌ 이해 없이 진행:
Claude Code: "useCallback으로 감싸주세요"
사용자: (이해 안 되지만) "알겠어 해줘"
→ 비슷한 상황에서 스스로 해결 못함
→ 문제 발생 시 원인 파악 못함

✅ 이해 후 진행:
Claude Code: "useCallback으로 감싸주세요"
사용자: "useCallback이 뭐야? 왜 필요해?"
Claude Code: [상세 설명]
→ 개념 습득
→ 비슷한 상황에서 스스로 판단 가능
```

### 질문의 가치

| 질문하면 | 질문 안 하면 |
|----------|-------------|
| 개념 습득 | 복붙만 반복 |
| 응용 가능 | 같은 실수 반복 |
| 자신감 향상 | 의존도 높음 |
| 더 좋은 질문 가능 | 막연한 질문 반복 |

## 질문 유형

### 1. "왜" 질문

```
"왜 이렇게 해야 해?"

"왜 이 방식이 더 좋아?"

"왜 그 라이브러리를 추천하는 거야?"

"왜 여기에 에러 처리가 필요해?"
```

### 2. "어떻게" 질문

```
"이게 어떻게 동작하는 거야?"

"어떻게 이 결과가 나오는 거야?"

"어떻게 응용할 수 있어?"

"어떻게 테스트해볼 수 있어?"
```

### 3. "뭐야/뭐지" 질문

```
"이 용어가 뭐야?"

"이 문법이 뭐지?"

"이 패턴이 뭐야?"

"이 오류가 뭐지?"
```

### 4. 재설명 요청

```
"더 쉽게 설명해줘"

"예시로 설명해줘"

"비유로 설명해줘"

"단계별로 설명해줘"
```

## 이해 확인 방법

### 내가 이해한 것 확인

```
"내가 이해한 게 맞는지 확인해줘:
[내가 이해한 내용]

맞아?"
```

### 요약 요청

```
"방금 설명한 거 핵심만 세 줄로 요약해줘"

"결론만 말해줘"

"한 문장으로 정리하면?"
```

### 비교 요청

```
"기존 방식이랑 뭐가 다른 거야?"

"A랑 B 차이점이 뭐야?"

"이거 안 쓰면 어떻게 돼?"
```

## 구체적인 질문 예시

### 코드 관련

```
[상황] Claude Code가 async/await 코드 제시

"이 async/await가 뭐야?
왜 그냥 함수로 안 쓰고 이렇게 써?"

"Promise랑 뭐가 달라?"

"try-catch가 왜 필요해?"
```

### 아키텍처 관련

```
[상황] Claude Code가 컴포넌트 분리 제안

"왜 이렇게 나눠야 해?
한 파일에 다 있으면 안 돼?"

"이렇게 나누면 뭐가 좋아?"

"실제로 어떤 효과가 있어?"
```

### 에러 관련

```
[상황] Claude Code가 에러 원인 설명

"이 에러가 왜 발생한 거야?"

"어떻게 하면 이런 에러를 미리 방지해?"

"비슷한 에러 또 있어?"
```

## 깊이 있는 이해를 위한 질문

### 레벨 1: 기초

```
"이게 뭐야?"
"어떻게 동작해?"
```

### 레벨 2: 이유

```
"왜 이렇게 해?"
"다른 방법은 없어?"
```

### 레벨 3: 응용

```
"다른 상황에서는 어떻게 적용해?"
"언제 이걸 쓰면 안 돼?"
```

### 레벨 4: 연결

```
"이거랑 관련된 개념 뭐가 있어?"
"이걸 더 깊이 공부하려면 뭘 봐야 해?"
```

## 질문 연속 예시

```
[1차 질문]
사용자: "useEffect가 뭐야?"
Claude Code: [기본 설명]

[2차 질문 - 더 깊게]
사용자: "의존성 배열이 뭐야?"
Claude Code: [의존성 배열 설명]

[3차 질문 - 구체적으로]
사용자: "빈 배열이랑 배열 없는 거 뭐가 달라?"
Claude Code: [차이점 설명]

[4차 질문 - 실용적으로]
사용자: "언제 어떤 걸 써야 해? 예시 보여줘"
Claude Code: [예시 코드와 상황별 사용법]

[이해 확인]
사용자: "그러면 API 호출은 빈 배열로 하면 되는 거지?"
Claude Code: [확인 및 추가 팁]
```

## 효과적인 질문 방법

### 구체적으로 물어보기

```
❌ "이해 안 돼"
✅ "두 번째 줄에서 map이 뭘 하는 건지 이해가 안 돼"

❌ "왜?"
✅ "왜 여기서 async를 붙여야 해?"
```

### 어디가 모르는지 명시

```
✅ "전체적인 흐름은 이해했는데,
3번째 단계에서 데이터가 어떻게 변환되는지 모르겠어"

✅ "이 코드는 이해했는데,
이걸 언제 써야 하는지 모르겠어"
```

## 질문하기 어려울 때

### 뭘 물어야 할지 모를 때

```
"솔직히 뭘 모르는지도 모르겠어.
기초부터 다시 설명해줘"

"이해하려면 어떤 배경 지식이 필요해?"

"이거 이해하려면 뭘 먼저 알아야 해?"
```

### 질문이 바보 같다고 느껴질 때

```
기억하세요:
- 바보 같은 질문은 없음
- 모르는 게 당연함
- 질문해야 배움
- Claude Code는 판단하지 않음
```

## 체크리스트

### 이해도 확인 체크리스트

- [ ] 이 코드가 무엇을 하는지 설명할 수 있는가?
- [ ] 왜 이 방식을 사용하는지 아는가?
- [ ] 비슷한 상황에서 스스로 적용할 수 있는가?
- [ ] 다른 사람에게 설명할 수 있는가?
- [ ] 이것과 관련된 다른 개념을 아는가?

## 주의사항

- 이해 안 되는 채로 넘어가지 않기
- 질문에 부끄러워하지 않기
- "당연히 알겠지"라고 가정하지 않기
- 모르면 솔직하게 모른다고 하기
- 이해될 때까지 계속 질문하기
