# 구체적으로 질문하는 방법

## 요약
"안 돼요" 대신 상황, 시도한 것, 오류 메시지 포함하여 질문. 구체적 질문 = 정확한 답변 = 빠른 해결.

## 상세

### 좋은 질문의 구성 요소

| 요소 | 예시 |
|------|------|
| 상황 | "로그인 기능 구현하려고 했습니다" |
| 시도 | "onClick 이벤트 추가했습니다" |
| 결과 | "클릭해도 반응 없습니다" |
| 오류 | "콘솔에 TypeError 표시됩니다" |

### 질문 개선 예시

```
X "로그인 안 돼요"

O "로그인 테스트 중입니다.
   - 이메일: test@example.com 입력
   - 버튼 클릭 시 '401 Unauthorized' 응답
   - 콘솔에 'Invalid credentials' 메시지
   원인이 무엇일까요?"
```

### 오류 메시지 공유

```
"다음 오류가 발생합니다:

TypeError: Cannot read property 'map' of undefined
    at ProductList (ProductList.js:15:23)"
```

### 질문 개선 공식

```
"안 돼요" → "A를 하려고 B를 시도했는데 C 오류 발생"
"이상해요" → "예상 결과는 A인데 실제로는 B가 나옵니다"
```
