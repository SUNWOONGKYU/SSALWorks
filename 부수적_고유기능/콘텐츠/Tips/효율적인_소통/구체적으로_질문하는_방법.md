# 구체적으로 질문하는 방법

## 핵심 요약

"안 돼요" 대신 구체적으로 질문하세요. 상황, 시도한 것, 오류 메시지를 포함하면 정확한 답변을 빠르게 받을 수 있습니다.

## 왜 구체적으로 질문해야 하나?

### 막연한 질문 vs 구체적 질문

```
❌ 막연한 질문:
"안 돼요"
→ Claude Code: "무엇이 안 되나요?"
→ 추가 질문 필요 → 시간 낭비

✅ 구체적 질문:
"로그인 버튼 클릭 시 'undefined is not a function' 오류가 발생합니다.
Chrome 브라우저에서 테스트했고, 콘솔에 이 오류가 뜹니다."
→ Claude Code: 바로 원인 분석 및 해결책 제시
→ 빠른 문제 해결
```

## 좋은 질문의 구성 요소

### 필수 요소

| 요소 | 설명 | 예시 |
|------|------|------|
| 상황 | 무엇을 하려고 했는가 | "로그인 기능을 구현하려고 했습니다" |
| 시도 | 무엇을 했는가 | "버튼에 onClick 이벤트를 추가했습니다" |
| 결과 | 무엇이 발생했는가 | "클릭해도 아무 반응이 없습니다" |
| 오류 | 오류 메시지가 있다면 | "콘솔에 TypeError가 표시됩니다" |
| 환경 | 어떤 환경인가 | "Chrome 120, Windows 11" |

## 질문 템플릿

### 오류 해결 질문

```
[상황]
~를 하려고 했습니다.

[시도한 것]
다음과 같이 코드를 작성했습니다:
[코드 또는 설명]

[발생한 오류]
오류 메시지: [오류 내용]

[환경]
- 브라우저/OS: [환경 정보]
- 관련 파일: [파일 경로]

이 문제를 어떻게 해결할 수 있을까요?
```

### 기능 구현 질문

```
[목표]
~한 기능을 구현하고 싶습니다.

[현재 상태]
현재 [상황 설명]입니다.

[제약 조건]
- [조건 1]
- [조건 2]

[질문]
어떻게 구현하면 좋을까요?
```

## 질문 변환 예시

### 예시 1: 오류 해결

```
❌ 나쁜 질문:
"로그인 안 돼요"

✅ 좋은 질문:
"로그인 기능 테스트 중입니다.

시도한 것:
- 이메일: test@example.com
- 비밀번호: 올바른 비밀번호 입력

발생한 오류:
- 로그인 버튼 클릭 시 '401 Unauthorized' 응답
- 콘솔에 'Invalid credentials' 메시지

확인한 것:
- 데이터베이스에 해당 사용자 존재 확인함
- 비밀번호 해싱 확인함

원인이 무엇일까요?"
```

### 예시 2: 기능 구현

```
❌ 나쁜 질문:
"페이지네이션 만들어줘"

✅ 좋은 질문:
"상품 목록 페이지에 페이지네이션을 추가하고 싶습니다.

현재 상태:
- 전체 상품 500개
- 한 페이지에 전부 로딩 중 (느림)

원하는 것:
- 한 페이지에 20개씩 표시
- 이전/다음 버튼
- 페이지 번호 표시
- URL에 페이지 번호 포함 (/products?page=2)

기술 스택:
- React, TypeScript
- API는 이미 페이지네이션 지원 (?page=&limit=)

어떻게 구현하면 좋을까요?"
```

### 예시 3: 성능 문제

```
❌ 나쁜 질문:
"느려요"

✅ 좋은 질문:
"메인 페이지 로딩이 느립니다.

측정 결과:
- 첫 로딩: 8초
- 번들 크기: 3.2MB
- Network 탭에서 큰 파일: main.js (2.5MB)

시도한 것:
- 이미지 최적화 (효과 미미)
- 코드 스플리팅 시도 (설정 오류로 실패)

목표:
- 로딩 시간 3초 이내
- 번들 크기 1MB 이하

어떻게 개선할 수 있을까요?"
```

## 오류 메시지 공유 방법

### 오류 전체 복사

```
✅ 좋은 예:
"다음 오류가 발생합니다:

TypeError: Cannot read property 'map' of undefined
    at ProductList (ProductList.js:15:23)
    at renderWithHooks (react-dom.development.js:14985:18)
    at mountIndeterminateComponent (react-dom.development.js:17811:13)"
```

### 스크린샷 설명

```
✅ 좋은 예:
"스크린샷을 보면:
- 빨간 오류 메시지가 화면 상단에 표시
- 'Network Error' 라고 적혀있음
- F12 콘솔에 'CORS policy' 관련 메시지"
```

## 맥락 제공하기

### 프로젝트 맥락

```
"현재 S2 단계 개발 중입니다.
로그인/회원가입 기능을 구현하고 있고,
Supabase Auth를 사용합니다."
```

### 이전 시도 맥락

```
"어제 세션에서 이 문제를 시도했고,
JWT 토큰 만료 시간을 늘려봤지만 해결되지 않았습니다."
```

### 관련 파일 맥락

```
"관련 파일들:
- src/pages/Login.js (로그인 페이지)
- src/api/auth.js (인증 API)
- src/hooks/useAuth.js (인증 훅)"
```

## 체크리스트

### 질문 전 확인사항

- [ ] 무엇을 하려고 했는지 명시했는가?
- [ ] 무엇을 시도했는지 포함했는가?
- [ ] 오류 메시지 전체를 복사했는가?
- [ ] 환경 정보를 포함했는가?
- [ ] 관련 코드/파일을 언급했는가?

## 질문 개선 공식

```
[기존 질문] → [구체적 질문]

"안 돼요"
→ "A를 하려고 B를 시도했는데 C 오류가 발생합니다"

"이상해요"
→ "예상 결과는 A인데 실제로는 B가 나옵니다"

"왜 그래요?"
→ "X 상황에서 Y 동작이 발생하는 이유가 뭔가요?"
```

## 주의사항

- 막연한 질문은 막연한 답변으로 이어짐
- 오류 메시지는 생략하지 말고 전체 복사
- 코드 공유 시 관련 부분만 추출
- 민감정보(비밀번호, API 키)는 제외
- 너무 긴 질문도 좋지 않음 - 핵심만 간결하게
