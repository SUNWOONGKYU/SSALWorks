# Claude Code에게 사용법 물어보기

## 핵심 요약

Claude Code 사용법을 모르겠으면 Claude Code에게 직접 물어보세요. "이 기능 어떻게 써?", "이거 가능해?" 등 질문하면 바로 답을 얻을 수 있습니다.

## 왜 Claude Code에게 직접 물어보나?

### 장점

```
✅ 직접 질문의 장점:
- 즉시 답변 받을 수 있음
- 현재 맥락에 맞는 설명
- 따로 문서 찾을 필요 없음
- 예시 코드까지 받을 수 있음
- 추가 질문도 바로 가능
```

### 어떤 것을 물어볼 수 있나?

| 카테고리 | 질문 예시 |
|----------|----------|
| 기능 확인 | "Claude Code로 파일 생성할 수 있어?" |
| 사용법 | "MCP 서버 어떻게 설정해?" |
| 가능 여부 | "Git 커밋 자동화 가능해?" |
| 비교 | "Skills와 Slash Commands 차이가 뭐야?" |
| 문제 해결 | "이 오류 어떻게 해결해?" |

## 질문 예시

### 기능 사용법

```
"Claude Code에서 파일 검색하는 방법 알려줘"

"여러 파일 동시에 수정하는 방법이 뭐야?"

"프로젝트 전체 구조 보는 명령어 있어?"
```

### 도구 사용법

```
"Slash Command 만드는 방법 설명해줘"

"MCP 서버 추가하려면 어떻게 해?"

"Skills 파일 작성 방법 알려줘"
```

### 가능 여부 확인

```
"Claude Code로 이미지 생성 가능해?"

"웹페이지 스크래핑 할 수 있어?"

"데이터베이스에 직접 연결할 수 있어?"
```

### 설정 관련

```
"CLAUDE.md 파일 역할이 뭐야?"

"세션 간에 정보 공유하는 방법 있어?"

"work_logs 어떻게 설정해?"
```

## 질문 유형별 예시

### 1. "어떻게" 질문

```
"어떻게 Git 커밋 메시지를 자동으로 생성해?"

"어떻게 여러 파일에서 특정 문자열을 찾아?"

"어떻게 프로젝트 의존성을 업데이트해?"
```

### 2. "가능해?" 질문

```
"터미널 명령어 실행 가능해?"

"웹 검색해서 정보 가져올 수 있어?"

"코드 리뷰 자동화 가능해?"
```

### 3. "뭐야/뭐지" 질문

```
"Task Agent가 뭐야?"

"서브에이전트가 뭐지?"

"Project Grid가 뭐야?"
```

### 4. "왜" 질문

```
"왜 새 폴더 만들 때 승인이 필요해?"

"왜 Production 폴더와 Stage 폴더에 둘 다 저장해?"

"왜 검증 에이전트를 별도로 써야 해?"
```

## 심화 질문

### 비교 질문

```
"Slash Commands와 Skills 뭐가 다른 거야?
각각 언제 쓰는 게 좋아?"

"Task Agent와 Verification Agent 차이점 설명해줘"
```

### 추천 질문

```
"이 프로젝트에 적합한 MCP 서버 추천해줘"

"초보자에게 먼저 배우면 좋을 기능 뭐야?"

"생산성 높이는 Claude Code 팁 알려줘"
```

### 문제 해결 질문

```
"세션이 끊어졌을 때 어떻게 복구해?"

"파일이 잘못된 위치에 저장됐어. 어떻게 정리해?"

"Claude Code가 자꾸 같은 실수를 반복해. 어떻게 해?"
```

## 좋은 질문 vs 나쁜 질문

### 나쁜 질문

```
❌ "안 돼"
→ 뭐가 안 되는지 모름

❌ "이상해"
→ 뭐가 이상한지 모름

❌ "도와줘"
→ 뭘 도와야 하는지 모름
```

### 좋은 질문

```
✅ "파일 저장이 안 돼. 권한 오류가 나는데 어떻게 해결해?"
→ 문제, 오류 내용 명시

✅ "MCP 서버 설정했는데 연결 안 돼.
설정 파일 보여줄게, 뭐가 잘못됐어?"
→ 상황, 시도한 것 포함

✅ "Slash Command로 자동 커밋 기능 만들고 싶어.
어떻게 시작하면 돼?"
→ 목표 명확
```

## 질문 후 후속 활동

### 답변 이해 안 되면

```
"더 쉽게 설명해줘"

"예시 코드 보여줘"

"단계별로 알려줘"
```

### 답변 확인하려면

```
"내가 이해한 게 맞아? [이해한 내용]"

"이렇게 하면 되는 거야? [시도할 방법]"
```

### 추가 질문

```
"그러면 [관련 내용]은 어떻게 돼?"

"이것과 비슷한 기능 또 있어?"

"실제 사용 예시 보여줘"
```

## 유용한 메타 질문

```
"Claude Code로 할 수 있는 것들 목록으로 알려줘"

"이 프로젝트에서 Claude Code 100% 활용하는 방법 알려줘"

"내가 모르고 있을 수 있는 유용한 기능 있어?"

"Claude Code 고급 사용법 알려줘"
```

## 주의사항

- 질문은 구체적으로 (막연한 질문 피하기)
- 답변이 이해 안 되면 바로 재질문
- Claude Code도 모르는 것이 있을 수 있음
- 최신 기능은 공식 문서 참고 권장
- 답변 내용이 프로젝트에 맞는지 확인
