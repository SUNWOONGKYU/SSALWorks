# 3가지 이상 대안 요청하기

## 요약
판단 어려울 때 "3가지 대안 제시해줘" 요청. 각 대안의 장단점 비교 후 최선의 선택 가능.

## 상세

### 대안 요청이 유용한 상황

| 상황 | 예시 |
|------|------|
| 기술 선택 | 어떤 라이브러리 쓸지 |
| 아키텍처 결정 | 어떤 패턴 적용할지 |
| 문제 해결 | 버그를 어떤 방식으로 고칠지 |

### 요청 방법

```
"로그인 구현 방법 3가지 제안해줘.
각각 장단점 설명해줘."

"회원 데이터 저장 방법을 제안해줘:
- 최소 3가지 대안
- 장점과 단점
- 우리 프로젝트에 맞는 추천"
```

### 비교표 요청

```
"상태 관리 라이브러리 옵션을 표로 비교해줘:
- Redux, Zustand, Context API
- 학습 곡선, 번들 크기, 기능 비교"
```

### 결정 후 진행

```
"2번 방식으로 진행해줘"
"왜 이 방식을 선택했는지 문서에 기록해줘"
```
