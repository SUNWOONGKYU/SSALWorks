# 판단이 어려운 상황에서 AI에게 3가지 이상 대안과 추천 이유 요청하기

> 이 문서는 판단이 어려운 상황에서 Claude Code에게 여러 대안을 요청하여 최선의 선택을 하는 방법을 설명합니다.

---

## 왜 대안을 요청하는가

하나의 방법만 알면 비교할 수 없습니다. 3가지 이상의 대안을 보면 각각의 장단점을 비교하고 우리 프로젝트에 가장 맞는 방법을 선택할 수 있습니다.

---

## 대안 요청이 유용한 상황

| 상황 | 예시 |
|------|------|
| 기술 선택 | 어떤 라이브러리 쓸지 |
| 아키텍처 결정 | 어떤 패턴 적용할지 |
| 문제 해결 | 버그를 어떤 방식으로 고칠지 |
| UI/UX 디자인 | 레이아웃을 어떻게 구성할지 |
| 성능 최적화 | 어떤 최적화 방법을 적용할지 |

---

## 기본 요청 방법

```
"로그인 구현 방법 3가지 제안해줘.
각각 장단점 설명해줘."

"회원 데이터 저장 방법을 제안해줘:
- 최소 3가지 대안
- 장점과 단점
- 우리 프로젝트에 맞는 추천"
```

---

## 비교표 요청

```
"상태 관리 라이브러리 옵션을 표로 비교해줘:
- Redux, Zustand, Context API
- 학습 곡선, 번들 크기, 기능 비교"
```

---

## 추천까지 요청하기

```
"이미지 업로드 방식 3가지 제안해줘.
그 중에서 우리 프로젝트에 가장 적합한 거 추천해줘.
추천 이유도 설명해줘."
```

---

## 결정 후 진행

```
"2번 방식으로 진행해줘"
"왜 이 방식을 선택했는지 문서에 기록해줘"
```

---

## 대안 요청 템플릿

```
"[기능명]을 구현하려고 해.
다음 조건으로 대안을 제시해줘:

1. 최소 3가지 방법
2. 각각의 장단점
3. 구현 난이도 (쉬움/보통/어려움)
4. 추천 + 추천 이유"
```
