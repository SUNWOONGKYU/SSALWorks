# 헤매면 미련 없이 세션 정리하기

## 핵심 요약

Claude Code가 버벅거리거나 헤매면 미련 없이 세션을 정리하세요. 정리 전에 `PROJECT_STATUS` 현행화, `work_logs` 기록, `outbox`에 결과 저장을 시키면 다음 세션에서 아무 문제 없이 이어서 작업할 수 있습니다.

## 세션 정리가 필요한 신호

### 1. 같은 실수 반복

```
❌ 신호:
- 같은 파일을 계속 잘못 수정
- 이미 설명한 것을 다시 물어봄
- 동일한 오류를 반복
```

### 2. 컨텍스트 혼란

```
❌ 신호:
- 다른 파일과 혼동
- 이전 대화 내용 잊어버림
- 프로젝트 구조 착각
```

### 3. 느려짐 / 지연

```
❌ 신호:
- 응답이 점점 느려짐
- 처리 시간이 길어짐
- 중간에 멈춤
```

### 4. 이상한 방향

```
❌ 신호:
- 요청하지 않은 작업 시작
- 불필요한 파일 생성
- 의도와 다른 결과
```

## 세션 정리 절차

### 1단계: 현재 상태 저장

```
"지금까지 작업 내용을 정리해줘:
1. work_logs/current.md 업데이트
2. 완료된 작업 목록
3. 진행 중인 작업 상태
4. 다음 해야 할 작업"
```

### 2단계: 결과물 저장

```
"작업 결과를 저장해줘:
1. 변경된 파일 목록
2. outbox에 완료 보고서
3. 중요한 결정사항 기록"
```

### 3단계: 커밋 (필요시)

```
"현재까지 변경사항 커밋해줘"
```

### 4단계: 세션 종료

```
터미널에서:
/clear 또는 새 터미널 열기
```

## 빠른 정리 명령

### 한 번에 정리하기

```
"세션 정리해줘. work_logs 업데이트하고,
현재 상태 저장하고, 다음 작업 정리해줘."
```

### 슬래시 명령어 활용

```
/cleanup (정의되어 있다면)
```

## 정리 전 체크리스트

### 필수 저장 항목

- [ ] work_logs/current.md 업데이트
- [ ] 완료된 작업 기록
- [ ] 진행 중 작업 상태 기록
- [ ] 다음 작업 명시
- [ ] 중요 결정사항 메모

### 선택 저장 항목

- [ ] outbox에 결과 보고서
- [ ] Project_Status.md 업데이트 (큰 변화 시)
- [ ] Git 커밋 (안전한 상태라면)

## 새 세션 시작

### 깨끗한 시작

```
새 세션에서:
"work_logs 확인하고 이전 작업 이어서 해줘"
```

### 효과

```
✅ 새 세션의 이점:
- 깨끗한 컨텍스트
- 빠른 응답 속도
- 명확한 작업 방향
- 이전 혼란 없음
```

## 정리하지 않으면 발생하는 문제

### 문제점

```
❌ 정리 없이 계속 진행:
- 컨텍스트 더 꼬임
- 응답 더 느려짐
- 오류 더 많아짐
- 시간 더 낭비

✅ 정리 후 재시작:
- 깨끗한 상태
- 빠른 응답
- 정확한 작업
- 시간 절약
```

### 비유

```
정리 없이 계속 = 엉킨 실타래 계속 당기기
정리 후 재시작 = 새 실타래로 시작

결국 정리 후 재시작이 더 빠릅니다.
```

## 언제 정리할지 판단 기준

### 즉시 정리 (강력 권장)

| 상황 | 대응 |
|------|------|
| 같은 실수 3회 반복 | 즉시 정리 |
| 요청과 다른 작업 시작 | 즉시 정리 |
| 응답 30초 이상 지연 | 정리 고려 |
| 파일 혼동 | 즉시 정리 |

### 조금 더 시도해볼 만한 경우

| 상황 | 대응 |
|------|------|
| 단순 오타/실수 | 수정 지시 |
| 방향 약간 다름 | 명확하게 재지시 |
| 처음 느려짐 | 조금 대기 |

## 효율적인 세션 관리

### 권장 세션 길이

```
✅ 적당한 세션:
- 작업 단위별 (기능 1개)
- 1-2시간 이내
- 헤매기 시작하면 종료

❌ 너무 긴 세션:
- 반나절 이상 연속
- 여러 기능 연속 작업
- 헤매도 계속 진행
```

### 세션 교체 주기

```
권장: 기능 단위로 세션 분리

세션 1: 로그인 기능 → 완료 → 정리
세션 2: 회원가입 기능 → 완료 → 정리
세션 3: 대시보드 → 완료 → 정리
```

## 주의사항

- 미련 없이 정리하는 것이 핵심
- 정리 전 반드시 현재 상태 저장
- 저장 없이 종료하면 작업 손실
- 새 세션은 깨끗한 시작
- work_logs가 있으면 언제든 복구 가능
