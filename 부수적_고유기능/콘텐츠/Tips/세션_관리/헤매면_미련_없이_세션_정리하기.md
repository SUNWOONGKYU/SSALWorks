# 세션이 비효율적이면 미련 없이 /clear하고 새로 시작하기

> 이 문서는 세션이 비효율적일 때 정리하고 새로 시작하는 방법을 설명합니다.

---

## 세션 정리가 필요한 신호

| 신호 | 대응 |
|------|------|
| 같은 실수 3회 반복 | 즉시 정리 |
| 요청과 다른 작업 시작 | 즉시 정리 |
| 응답 30초 이상 지연 | 정리 고려 |
| 파일/프로젝트 혼동 | 즉시 정리 |
| 컨텍스트가 너무 길어짐 | 정리 고려 |

**헤매면서 계속 진행 = 시간 낭비**입니다.

---

## 정리 절차

### 1단계: 현재 상태 저장 요청

```
"세션 정리해줘. work_logs 업데이트하고,
현재 상태 저장하고, 다음 작업 정리해줘."
```

### 2단계: Claude Code가 수행

1. work_logs/current.md 업데이트
2. 변경된 파일 목록 저장
3. 다음 할 작업 명시
4. 커밋 (안전하면)

### 3단계: 세션 종료

- `/clear` 명령어 사용
- 또는 새 터미널 열기

---

## 새 세션 시작

```
"work_logs 확인하고 이전 작업 이어서 해줘"
```

---

## 핵심 원칙

- 정리 후 재시작 = 깨끗한 상태, 빠른 응답
- 미련 없이 정리하는 것이 효율적
- work_logs만 잘 기록되어 있으면 언제든 복구 가능
