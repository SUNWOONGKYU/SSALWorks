# 세션이 끊어져도 work_logs, CLAUDE.md로 이전 작업 상태 빠르게 복구하기

> 이 문서는 Claude Code 세션이 끊어졌을 때 이전 작업을 복구하는 방법을 설명합니다.

---

## 손실되는 것 / 유지되는 것

| 손실됨 | 유지됨 |
|--------|--------|
| 대화 컨텍스트 | 저장된 파일 |
| 진행 중 작업 상태 | Git 커밋 |
| 메모리 내 변수 | work_logs 기록 |
| | Project_Status.md |

대화 내용은 사라지지만, **저장된 파일과 기록은 유지**됩니다.

---

## 간단한 복구

새 세션에서 이렇게 요청합니다:

```
"이전 작업 이어서 해줘"
```

또는

```
"work_logs 확인하고 진행해줘"
```

---

## 상세 복구

컨텍스트를 더 정확히 파악하려면:

```
"다음 파일들 읽고 상황 파악해줘:
1. .claude/work_logs/current.md
2. P0_작업_디렉토리_구조_생성/Project_Status.md
3. git status"
```

---

## 예방 방법

- 작업 단위마다 커밋
- 중간 결과 파일로 저장
- work_logs 수시 업데이트
- 큰 작업은 작은 단위로 분할

---

## Claude Code에게 요청하기

```
"방금 끊겼어. 이전 작업 상태 파악해줘"
"work_logs 확인하고 다음 할 일 알려줘"
"마지막 커밋 이후 변경사항 확인해줘"
```
