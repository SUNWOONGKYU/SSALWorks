# CLI 여러 개 동시 실행하기

## 핵심 요약

Claude Code CLI를 2개 이상 여러 개 동시에 실행해서 작업을 나누어 시키는 것이 효율적입니다.

## 왜 여러 CLI를 동시에 사용해야 할까?

### 1. 컨텍스트 한도 도달 시간 연장

한 CLI에서 모든 작업을 몰아서 하면 컨텍스트 한도에 빨리 도달합니다. 여러 CLI로 작업을 분산하면 각 CLI의 컨텍스트 사용량이 나뉘어 한도 도달 시간을 늘릴 수 있습니다.

### 2. 대기 시간 제거

Claude Code는 작업 사이에 대기 시간이 있습니다. 여러 CLI를 사용하면:
- CLI 1이 작업 중일 때 → CLI 2에 다음 작업 지시
- CLI 2가 작업 중일 때 → CLI 1 결과 확인 및 다음 작업 지시
- 대기 시간 없이 지속적으로 작업 진행 가능

### 3. 업무 종류별 분리로 효율 향상

같은 프로젝트라도 업무 종류별로 CLI를 나누면 작업 효율이 높아집니다.

**예시:**
- **CLI 1**: 프론트엔드 작업 전담
- **CLI 2**: 백엔드/API 작업 전담
- **CLI 3**: 문서화/검증 작업 전담

각 CLI가 특정 분야에 집중하면 해당 맥락을 더 잘 기억하고 일관된 작업이 가능합니다.

## 실전 활용 팁

```
# 터미널 여러 개 열기
터미널 1: claude (프론트엔드)
터미널 2: claude (백엔드)
터미널 3: claude (문서/검증)
```

## 주의사항

- 같은 파일을 동시에 수정하지 않도록 작업 영역을 명확히 분리
- 각 CLI에서 작업한 내용은 Git으로 통합 관리
- 작업 완료 후 결과를 서로 공유하여 충돌 방지
