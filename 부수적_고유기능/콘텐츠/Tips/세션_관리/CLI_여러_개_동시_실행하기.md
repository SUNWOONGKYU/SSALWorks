# 터미널 창 여러 개로 Claude Code 병렬 실행하여 작업 효율 높이기

> 이 문서는 Claude Code CLI를 여러 개 동시에 실행하여 효율적으로 작업하는 방법을 설명합니다.

---

## 왜 여러 CLI를 사용하는가

| 장점 | 설명 |
|------|------|
| 컨텍스트 분산 | 한 CLI에 모든 작업 몰아하면 한도 빨리 도달 |
| 대기 시간 제거 | CLI 1 작업 중에 CLI 2에 다음 작업 지시 |
| 업무 분리 | 프론트/백엔드/문서화 각각 전담 |

---

## 실제 사용 시나리오

```
[터미널 1: Frontend 전담]
$ cd C:\프로젝트
$ claude
> "google-login.html 만들어줘"

[터미널 2: Backend 전담]
$ cd C:\프로젝트
$ claude
> "subscription-cancel API 만들어줘"

[터미널 3: 문서/검증 전담]
$ cd C:\프로젝트
$ claude
> "S2 Stage Gate 검증해줘"
```

---

## 작업 분배 기준

| 터미널 | 담당 Area | 예시 작업 |
|--------|----------|----------|
| 1 | Frontend (F) | HTML, CSS, JS 페이지 |
| 2 | Backend (BA, S) | API, 인증 로직 |
| 3 | 문서/검증 (M, T) | 검증 리포트, 문서화 |

---

## 주의사항

### 같은 파일 동시 수정 금지

```
❌ 잘못된 예:
터미널 1: "login.js 수정해줘"
터미널 2: "login.js 에러 핸들링 추가해줘"

✅ 올바른 예:
터미널 1: "Frontend 폴더 작업"
터미널 2: "Backend 폴더 작업"
```

- 다른 영역/파일을 분리해서 작업
- Git으로 결과 통합 관리
- 작업 완료 후 결과 공유하여 충돌 방지
