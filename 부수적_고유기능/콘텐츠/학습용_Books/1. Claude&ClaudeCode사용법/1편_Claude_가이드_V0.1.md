**2025년 10월 기준**

## 1. 소개

### 1-1. Claude란 무엇인가

Claude는 Anthropic이 개발한 대화형 인공지능 어시스턴트입니다. 자연어 처리, 코드 생성, 데이터 분석, 창작 작업 등 다양한 지적 작업을 수행할 수 있으며, 특히 안전성과 유용성의 균형을 중요하게 여기는 Constitutional AI 원칙을 기반으로 설계되었습니다.

Claude의 핵심 특징:
- **Extended Thinking**: 복잡한 문제에 대해 단계적으로 사고하는 능력
- **Constitutional AI**: 안전하고 유익한 응답을 생성하도록 설계된 AI 정렬 방식
- **높은 컨텍스트 윈도우**: 최대 200K 이상의 토큰을 한 번에 처리 가능
- **멀티모달 기능**: 텍스트뿐만 아니라 이미지 분석, 파일 처리 등 지원
- **개발자 친화적**: CLI, IDE 결합, MCP 프로토콜 등 개발 생태계 제공

### 1-2. Anthropic 회사 소개

Anthropic은 2021년 OpenAI 출신 연구진들이 설립한 AI 안전성 연구 회사입니다. "안전하고 해석 가능한 AI 시스템 구축"을 미션으로 삼고 있으며, Constitutional AI, Scaling Laws 연구 등 AI 안전성 분야에서 선도적인 역할을 하고 있습니다.

**주요 연혁:**
- 2021년: Anthropic 설립
- 2023년: Claude 2 출시
- 2024년: Claude 3 패밀리 출시 (Opus, Sonnet, Haiku)
- 2024년 11월: MCP (Model Context Protocol) 공개
- 2025년 2월: Claude Code (CLI) 연구 프리뷰
- 2025년 5월 22일: Claude 4 패밀리 및 Claude Code 정식 출시 (GA)
- 2025년 8~9월: Opus 4.1, Sonnet 4.5 업그레이드

## 2. 설치 및 시작

### 2-1. 모바일-앱

#### 2-1-1. Claude 모바일-앱 소개

스마트폰과 태블릿에서 사용하는 Claude 전용 모바일 애플리케이션으로, 다운로드를 받은 후에 복잡한 설정 없이 바로 대화를 시작할 수 있으며 언제 어디서나 이동하면서도 접속할 수 있습니다.

**기본 기능**:
- AI 대화: 질문-답변을 통한 정보 제공
- 이미지 분석: 이미지 업로드 및 해석
- 코드 생성 및 분석: 프로그래밍 코드 작성, 분석 및 설명
- 파일 처리: PDF, TXT, 코드 파일 등 처리
- 프로젝트 관리: 주제별로 대화를 그룹화하여 체계적 관리
- 대화 히스토리: 모든 대화 자동 저장 및 검색 기능
- Artifacts: 코드, 문서, 다이어그램을 실시간 미리보기 창에서 표시
- 모델 전환: Haiku, Sonnet, Opus 모델 실시간 전환
- 실시간 동기화: 웹, 데스크톱-앱과 모든 대화 내용 자동 동기화

**모바일 특화 기능**:
- 음성 입력: 마이크 버튼으로 음성 질문 및 명령
- 카메라 연동: 실시간 사진 촬영 후 즉시 분석

**제약사항**:
- 파일 크기 제한 (개당 10MB)
- 모바일 기기 내부 파일 접근 불가
- 화면 크기의 한계로 인한 긴 텍스트 작업 불편

**사용환경**:
- **인터페이스**: 모바일(터치) 입력 방식
- **운영체제**: iOS/Android 스마트폰 및 태블릿

### 2-2. 웹

#### 2-2-1. Claude 웹 소개

웹 브라우저에서 사용하는 Claude 온라인 서비스로, 설치나 다운로드 없이 바로 접속할 수 있으며 PC와 모바일 모두에서 동일한 방식으로 대화를 할 수 있습니다.


**기본 기능**: 모바일-앱과 동일

**웹 추가 기능**:
- 파일 생성 및 분석 (2025년 9월 9일 출시):
  - Excel(.xlsx): 수식, 차트, 다중 시트 포함 스프레드시트 자동 생성
  - PowerPoint(.pptx): 전문 디자인의 프레젠테이션 즉시 제작
  - Word(.docx): 완성된 보고서 및 문서 작성
  - PDF: 최종 배포용 문서 생성
  - Python 기반 고급 데이터 분석 및 시각화
  - 파일 간 자동 변환 (PDF→PPT, Excel→차트 등)
- PWA(Progressive Web App) 지원: 웹사이트를 독립적인 앱처럼 설치하여 사용할 수 있게 해주는 기술. 설치 후에는 일반 프로그램처럼 작업 표시줄에서 바로 실행 가능

**제약사항**:
- 인터넷 연결 필수
- 웹 브라우저의 성능 차이에 따른 영향 발생
- 모바일 기기 및 PC 내부의 파일 접근 불가

**사용환경**:
- **인터페이스**: PC(마우스/키보드) 및 모바일(터치) 입력 방식
- **운영체제**: 모든 운영체제의 최신 웹 브라우저

### 2-3. 데스크톱-앱

#### 2-3-1. Claude 데스크톱-앱 소개

PC에 설치하는 독립 실행형 Claude 애플리케이션으로, 웹보다 더 강력하고 다양한 기능을 제공하며 MCP(Model Context Protocol) 서버 연동을 통해서 로컬 파일에도 접근하여 읽고 쓰기를 할 수 있습니다.

**기본 기능**: 모바일-앱과 동일

**웹 추가 기능**: 웹과 동일

**데스크톱-앱 추가 기능**:
- 내부 MCP 서버 3개 연동:
  - Filesystem MCP: 로컬 파일에 접근하여 읽기/쓰기/관리
  - Memory MCP: 대화 간 정보를 저장하고 학습
  - Anthropic MCP: 6가지 특화 도구(ask, stream, vision, multi_turn, count_tokens, tools)
- 외부 MCP 서버 추가 연동: GitHub, Google Drive, Slack, PostgreSQL, Docker, Puppeteer 등

**제약사항**:
- PC에서만 사용 가능
- MCP 설정하는 과정 복잡

**사용환경**:
- **인터페이스**: PC(마우스/키보드) 입력 방식
- **운영체제**: Windows, Linux 등 데스크톱 환경

(3) **다운로드 시작**: 선택한 운영체제 버튼 클릭

### 2-5. CLI (Claude Code)

💡 **용어 안내**: 본 가이드에서 '터미널'은 Windows의 명령 프롬프트(CMD) 또는 PowerShell, macOS/Linux의 Terminal을 지칭합니다.

#### 2-5-1. Claude Code 소개

터미널에서 가동되는 AI 어시스턴트로, 시스템 명령을 실행하여 복잡한 업무를 자동으로 처리하고 파일도 직접 편집할 수 있는 비개발자와 개발자 모두를 위한 강력한 도구입니다.

**기본 기능**: 모바일-앱과 동일 (단, Artifacts 미지원, 이미지는 파일 경로로 제공)

**Claude Code 특화 기능**:
- 파일 생성 및 편집: Excel, Word, PowerPoint, PDF 등 모든 형식의 파일을 생성/편집하여 로컬에 저장
- 기술 문서 생성: README.md, API 연결 및 용법 문서, 설치 가이드, 사용자 매뉴얼 자동 작성
- 외부 MCP 서버 연동: Google Drive, Slack, Jira 등 외부 서비스와 통합
- 시스템 명령 실행: Git, npm, 파일 작업 등 모든 터미널 명령 실행
- 프로젝트 관리: 파일 구조 분석 및 재구성
- 자동화 작업: 프로그램 설치, 환경 변수 설정, 필요한 패키지 자동 설치, 빌드 및 배포 과정 자동화
- 코드 작성: 전체 프로그램을 처음부터 작성
- 디버깅 지원: 오류 메시지 분석 및 수정

#### 작업 모드 선택하기

Claude Code는 3가지 작업 모드를 제공합니다. 작업 성격에 따라 적절한 모드를 선택해서 사용해야 합니다:

**(1) 일반 모드 (Default Mode)**
- **특징**: 기본 모드로, 파일 수정 전에 매번 사용자 승인을 요청
- **사용 시기**: 중요한 파일 작업, 신중한 검토가 필요한 경우
- **장점**: 모든 변경사항을 사전에 확인 가능
- **단점**: 작업 속도가 느림
- **설정 방법**: 기본값 (별도 설정 불필요)

**(2) 자동 수정 모드 (Accept Edits Mode)**
- **특징**: 파일 수정을 자동으로 승인하여 즉시 실행
- **사용 시기**: 신뢰할 수 있는 작업, 빠른 진행이 필요한 경우
- **장점**: 작업 속도가 매우 빠름
- **단점**: 실수 시 즉시 파일이 변경됨
- **설정 방법**:
  대화 중 `/permission accept` 입력

**(3) 플래닝 모드 (Plan Mode)**
- **특징**: 작업 실행 전에 전체 계획을 먼저 제시하고 승인 받음
- **사용 시기**: 복잡한 프로젝트, 여러 파일 수정, 구조적 변경이 필요한 경우
- **장점**: 전체 작업 흐름을 사전에 파악 가능
- **단점**: 초기 계획 수립 시간이 추가로 소요
- **설정 방법**:
  대화 중 `/permission plan` 입력

💡 **빠른 모드 전환**: 대화 중 **Shift + Tab** 키를 누르면 모드를 순환하며 전환할 수 있습니다 (일반 → 자동 수정 → 플래닝 → 일반).

💡 **팁**: 처음에는 일반 모드로 시작하고, Claude Code의 작업 방식에 익숙해지면 자동 수정 모드를 활용하세요. 대규모 리팩토링이나 새 기능 개발 시에는 플래닝 모드가 유용합니다.

#### 주요 도구 목록

Claude Code가 사용하는 주요 도구들 (`/tools` 명령어로 확인 가능):
- `Bash` - 시스템 명령어 실행 (프로그램 설치, 파일 작업 등)

💡 **팁**: 이러한 도구들은 Claude Code가 자동으로 선택하여 사용합니다.

### 2-6. MCP 서버 연결

#### MCP란 무엇인가?

MCP(Model Context Protocol)는 Claude와 외부 데이터 소스를 연결하여 실시간 정보 접근 및 작업 자동화를 가능하게 하는 확장 프로토콜입니다.

**주요 기능**:
- **파일 시스템 접근**: 로컬 파일 검색, 읽기/쓰기, 자동 편집
- **외부 서비스 연결**: Google Drive, GitHub, Slack, 데이터베이스 등
- **작업 자동화**: 복잡한 멀티스텝 작업을 자동으로 처리

**사용 예시**:
- "내 Google Drive에서 'Q3 보고서' 파일 찾아서 요약해줘" → MCP로 Drive 실시간 접근
- "GitHub의 최근 이슈 10개를 분석해줘" → MCP로 GitHub API 연결
- "PostgreSQL 데이터베이스에서 지난주 매출 조회해줘" → MCP로 DB 쿼리 실행

💡 **쉽게 말하면**: 스마트폰에 앱을 설치하듯이, Claude에 추가 기능을 설치하는 것입니다.

💡 **참고**: MCP에 대한 자세한 내용은 Section 3-6을 참조하세요.

#### 2-7-1. VS Code 소개

Microsoft가 제공하는 무료 오픈소스 텍스트 편집기로, 문서 작성부터 코딩까지 모든 텍스트 작성 작업을 지원하며 수많은 확장 프로그램을 설치하여 활용할 수 있고 Claude Code와 결합을 통해 AI 지원 통합 작업 환경을 제공합니다.

**기능**:
- **파일 관리**: 프로젝트 탐색기, 전역 검색, 다중 창 분할
- **코드 편집**: 문법 강조, 자동 완성, 오류 검출, 리팩토링
- **IntelliSense**: 스마트 코드 완성
- **멀티 커서**: 여러 위치 동시 편집
- **Git 통합**: 버전 관리 기본 지원
- **통합 터미널**: 내장 터미널로 명령어 실행
- **Claude Code 결합**: 터미널에서 바로 AI 도우미 실행
- **확장 프로그램**: 4만 개 이상의 플러그인으로 필요한 기능 추가 가능

**제약사항**:
- Claude Code와 동일한 제약사항 4가지
- 확장 프로그램 과다 설치 시 성능 저하
- 대용량 파일(50MB+) 생성 및 편집 시 성능 저하

**사용환경**:
- **인터페이스**: PC(마우스/키보드) 입력 방식
- **운영체제**: Windows, Linux 등 데스크톱 환경

**활용 팁**:
- 마크다운 문서를 실시간으로 미리보기 하면서 편집하기
- 통합 터미널에서 Claude Code를 바로 실행하여 AI 지원받기

### 3-5. 작업 환경 (Workspace)

Claude는 **Projects**와 **Artifacts**라는 두 가지 핵심 작업 환경을 제공합니다.

**💡 참고**: Projects와 Artifacts는 **웹/데스크톱/모바일 전용** 기능입니다. CLI에는 해당되지 않습니다.

---

#### Projects

**개요:**
- 주제별 대화 그룹화 및 체계적 관리
- 프로젝트 전용 지식 베이스 구축
- 맞춤 지시사항 설정

**주요 기능:**
- **지식 베이스**: 최대 100MB 파일 업로드 (PDF, 문서, 코드 등), 프로젝트 내 모든 대화에서 자동 참조
- **맞춤 지시사항**: 프로젝트별 어조, 형식, 작업 방식 정의로 일관된 출력 유지
- **대화 관리**: 프로젝트별 대화 그룹화, 관련 대화 쉽게 찾기, 컨텍스트 유지

**활용 사례:**
- 마케팅 프로젝트: 브랜드 가이드라인 업로드 → 일관된 브랜드 보이스 유지
- 개발 프로젝트: 코딩 스타일 가이드 명시 → 프로젝트 컨벤션 맞는 코드 생성
- 연구 프로젝트: 관련 논문 업로드 → 레퍼런스 기반 정확한 분석

**접근 방식**: 웹, 데스크톱-앱 (Pro 이상), 모바일-앱 (읽기 전용) | CLI 불가

---

#### Artifacts

**개요:**
- 코드, 문서, 다이어그램을 별도 창에 실시간 표시
- 대화 중 즉시 수정 및 개선
- 완성된 결과물 다운로드/복사

**지원 형식:**
- 코드: React, HTML/CSS, JavaScript, Python, Mermaid
- 문서: Markdown, 일반 텍스트
- 다이어그램: Mermaid (플로우차트, 시퀀스, ER 다이어그램)
- 그래픽: SVG

**주요 특징:**
- 실시간 미리보기: 변경사항 즉시 확인
- 반복 개선: 대화로 지속적 수정
- 다운로드 지원: 완성된 파일 저장
- 버전 관리: 이전 버전으로 되돌리기 가능

**활용:**
- React 컴포넌트 생성 및 실시간 미리보기
- 다이어그램 작성 (시스템 설계, 플로우차트)
- 긴 문서 작성 및 반복 수정

**접근 방식**: 웹, 데스크톱-앱, 모바일-앱 (읽기 전용) | CLI 불가

💡 **팁**: Artifacts는 반복 수정이 많은 작업(프로토타입, 문서 작성, 다이어그램)에 이상적입니다.

### 3-6. 확장 생태계 (Extension Ecosystem)

Claude는 MCP와 Skills를 통한 개방형 확장 생태계를 제공합니다.

#### MCP (Model Context Protocol)

**개요:**
- AI가 외부 데이터/도구에 연결되는 표준 프로토콜
- Anthropic이 2024년 11월 공개
- 플러그인보다 안전하고 표준화됨

**주요 MCP 서버:**

**Claude 기본 제공:**
- Filesystem, Memory, Anthropic MCP

**커뮤니티/기업 제공:**
- 개발: GitHub, GitLab, Jira
- DB: PostgreSQL, MySQL, MongoDB
- 자동화: Puppeteer, Playwright
- 클라우드: AWS, GCP, Azure
- 검색: Brave, Tavily, Google

**설치 방법:**
```bash
claude mcp add @modelcontextprotocol/server-github
```

💡 자세한 내용은 Section 2-6 (MCP 설치 및 설정) 참조.

---

#### Skills

**개요:**
- 2025년 10월 발표된 새로운 확장 기능
- 커뮤니티가 만든 재사용 가능한 기능 모음
- Claude.ai와 Claude Code 모두에서 사용 가능

**주요 특징:**
- 작업 자동화: 반복 워크플로우 간소화
- 도메인 특화: 특정 분야 최적화
- 커뮤니티 공유: GitHub에서 공유/설치
- 쉬운 사용: 간단한 명령어로 복잡한 작업

**사용 예시:**
- 코드 리뷰 자동화
- 데이터 분석 파이프라인
- 문서 자동 생성
- 특정 프레임워크 지원

**MCP vs Skills:**
- MCP: 외부 데이터/서비스 연결
- Skills: 작업 워크플로우 자동화

💡 MCP와 Skills를 함께 사용하면 강력한 워크플로우 구축 가능.

### 4-1. 5시간 리셋 시스템

#### 리셋 시스템이란?

Claude는 **5시간 단위 롤링 리셋 시스템**을 적용하고 있습니다.

**핵심 원칙:**
- 당신이 **첫 메시지를 보낸 시점부터 정확히 5시간 후**에 한도가 리셋됩니다
- 모든 플랫폼(웹, 데스크톱, 모바일, CLI) 공유 - 한 곳에서 사용하면 다른 곳 한도도 감소
- 고정된 시간대가 아닌, 개인별 타이머 방식

**예시: Pro 플랜 사용자 (45개/5시간)**

```
오전 9:00  - 첫 메시지 전송 (타이머 시작)
          - 45개 중 30개 사용 → 15개 남음

오후 2:00  - 정확히 5시간 경과, 한도 리셋!
          - 새로 45개 충전
          - 남은 15개 + 새 45개 = 총 60개 사용 가능

오후 3:00  - 60개 중 20개 사용 → 40개 남음

오후 7:00  - 다시 5시간 경과 (오후 2시부터), 리셋!
          - 남은 40개 + 새 45개 = 총 85개 사용 가능
```

**중요:**
- 메시지 개수는 **짧은 메시지 기준 "약" 45개**입니다
- 긴 메시지, 파일 첨부, 긴 대화는 더 많은 한도를 소비합니다
- 실제로는 메시지 길이에 따라 20-70개까지 변동 가능합니다

#### 리셋 시스템의 장점

(1) **유연한 사용**: 한 번에 몰아쓰고 잠시 쉴 수 있음
(2) **빠른 회복**: 최대 5시간만 기다리면 됨
(3) **누적 가능**: 안 쓴 것은 쌓임 (일일 최대치까지)

시간대별 활용 전략도 중요합니다. 새벽 3시에서 7시는 서버가 한산하여 응답이 빠르고 한도가 최대로 누적되어 있어 대량 처리나 긴 문서 분석에 적합합니다. 오전 9시에서 12시는 업무 시작 시간으로, 9시에 시작하면 오후 2시에 리셋되므로 중요한 작업을 우선 처리하는 것이 좋습니다. 오후 2시에서 6시는 피크타임으로 Free 사용자는 제한을 받을 수 있지만, Pro 이상은 우선순위를 받습니다. 저녁 7시에서 11시는 개인 사용이 증가하는 시간입니다.

### 4-2. API와 웹 구독의 독립성

**중요**: API 사용량과 웹/앱 구독은 **완전히 별개**입니다.

- Claude Pro 구독으로는 API 사용 불가
- API 키는 별도 종량제 요금 (사용량 기반)
- API 사용 시 웹 구독 혜택(무제한 메시지 등) 적용 안 됨

**예시**:
```
Pro 구독자 = 웹에서 5시간당 45개 메시지 사용 가능
           ≠ API 무료 사용 불가 (별도 요금 발생)
```
