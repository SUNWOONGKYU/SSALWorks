# 4편 | 코드 에디터와 IDE 이해하기

---

3편에서 개발 환경이 준비된 상태에서, 여기서는 코드를 작성할 도구를 선택해봅니다. 개발자에게 코드 에디터는 일상의 파트너와 같습니다. 이번 편에서는 메모장부터 VS Code, Cursor까지 다양한 코드 에디터의 특징을 비교하고, 실제로 웹개발에 최적화된 환경을 구축하는 방법을 살펴봅니다. 코드 작성뿐만 아니라 AI 도구 활용, 문서 작성까지 현대적인 개발 워크플로우를 완성해봅니다.

## 1. 개발 도구의 진화와 선택

### 1-1 메모장에서 전문 도구까지

개발 도구는 마치 **메모장 → 워드 → 전문 출판 프로그램**처럼 단계적으로 진화했습니다. 처음 프로그래밍을 시작하던 시절에는 윈도우 메모장 같은 단순한 텍스트 에디터에 코드를 작성했습니다. 하지만 이제는 훨씬 더 똑똑하고 편리한 도구들이 개발되었습니다.

**1단계: 텍스트 에디터**
가장 기본적인 도구로 메모장이나 Notepad++ 같은 프로그램이 여기에 속합니다. 이들은 단순히 텍스트를 편집하는 기능만 제공합니다. 코드를 작성할 수는 있지만 별다른 도움을 주지 않습니다. 마치 연필과 종이로 코드를 쓰는 것과 비슷한 수준이죠.

**2단계: 코드 에디터**
VS Code, Sublime Text, Cursor 같은 현대적인 코드 에디터는 프로그래밍을 위해 특별히 설계되었습니다. 이들은 다양한 편의 기능을 제공합니다. 코드의 문법을 색깔로 구분해서 보여주는 문법 강조(Syntax Highlighting) 기능이 있습니다. 타이핑하다가 Tab키만 누르면 나머지를 자동으로 완성해주는 자동완성(Auto Complete) 기능도 제공합니다. 또한 필요한 기능을 확장 프로그램으로 추가할 수 있습니다. 웹개발자의 90% 이상이 이 단계의 도구를 사용합니다.

**3단계: IDE (통합개발환경)**
IntelliJ IDEA나 Visual Studio 같은 IDE(Integrated Development Environment, 통합개발환경)는 훨씬 더 강력한 도구입니다. 코드 에디터의 모든 기능에 더해 다양한 고급 기능을 제공합니다. 컴파일(코드를 실행 파일로 변환), 디버깅(오류 찾기), 프로젝트 관리 등 개발에 필요한 모든 기능을 하나로 통합했습니다. 강력하지만 무겁고 복잡해서 주로 대규모 프로젝트나 특정 언어 전문 개발에 사용됩니다.

### 1-2 코드 에디터를 선택하는 이유

웹개발을 시작하는 단계에서 IDE 대신 코드 에디터를 선택하는 데는 분명한 이유가 있습니다.

**코드 에디터의 장점:**

첫째, **가벼우면서도 강력한 기능**을 제공합니다. VS Code는 설치 용량이 300MB 정도에 불과합니다. 하지만 대부분의 개발 작업을 충분히 처리할 수 있습니다. 컴퓨터 사양이 높지 않아도 빠르게 실행됩니다. 여러 파일을 동시에 편집해도 느려지지 않습니다.

둘째, **다양한 언어와 프로젝트에 유연하게 적용**됩니다. HTML, CSS, JavaScript는 물론이고 Python, Java, C++ 등 거의 모든 프로그래밍 언어를 지원합니다. 하나의 도구로 프론트엔드와 백엔드 개발을 모두 할 수 있습니다. 이는 매우 큰 장점입니다.

셋째, **학습 곡선이 완만**합니다. 처음 사용하는 사람도 10분이면 기본 사용법을 익힐 수 있습니다. 필요한 기능을 하나씩 배워나가면 됩니다. IDE처럼 복잡한 설정이나 프로젝트 구성을 이해할 필요가 없습니다.

넷째, **웹개발에 최적화**되어 있습니다. Live Server로 실시간 미리보기가 가능합니다. 자동 저장, Git 통합 등 웹개발에 필요한 기능들이 기본으로 제공됩니다. 또는 확장 프로그램으로 쉽게 추가할 수 있습니다.

**결론**: 이러한 이유로 웹개발 입문 단계에서는 VS Code 같은 코드 에디터가 최적의 선택입니다. 나중에 특정 언어나 프레임워크에 집중하게 되면 그때 IDE를 고려해도 늦지 않습니다.

## 2. VS Code 완벽 가이드

### 2-1 VS Code가 세계 1위 에디터가 된 이유

VS Code(Visual Studio Code)는 2015년 Microsoft에서 출시했습니다. 이후 급속도로 성장하여 현재 전 세계 개발자의 70% 이상이 사용하는 가장 인기 있는 코드 에디터가 되었습니다.

**주요 선택 이유:**

**무료 오픈소스**: Microsoft에서 개발했지만 완전 무료입니다. 소스코드가 공개되어 있어 누구나 개선에 참여할 수 있습니다. 유료 라이선스를 구매해야 하는 다른 도구들과 달리, 개인이든 기업이든 무료로 사용할 수 있습니다.

**압도적인 점유율**: Stack Overflow 개발자 설문조사에서 매년 1위를 차지합니다. GitHub 프로젝트의 대부분이 VS Code로 작성됩니다. 이는 곧 문제 해결 방법을 찾기 쉽다는 의미입니다. 또한 튜토리얼이 풍부하다는 뜻입니다.

**풍부한 확장 프로그램**: 3만 개가 넘는 확장 프로그램이 마켓플레이스에 등록되어 있습니다. 언어 지원, 테마, 디버거, 린터(Linter) 등 필요한 기능을 마음대로 추가할 수 있습니다. 이를 통해 자신만의 개발 환경을 구축할 수 있습니다.

**웹개발 최적화**: HTML, CSS, JavaScript를 기본 지원합니다. Emmet 약어(HTML 코드 단축 입력), IntelliSense(지능형 코드 자동완성), 실시간 오류 검사 등 웹개발에 필요한 기능이 내장되어 있습니다.

**AI 도구와의 완벽한 연동**: GitHub Copilot, Claude, ChatGPT 등 최신 AI 코딩 도구들이 모두 VS Code 확장으로 제공됩니다. 이를 통해 코드를 작성하면서 실시간으로 AI의 도움을 받을 수 있습니다.

### 2-2 설치부터 필수 설정까지

VS Code 설치는 매우 간단합니다. 하지만 몇 가지 중요한 옵션을 놓치면 나중에 불편할 수 있습니다.

**다운로드 과정:**

먼저 공식 웹사이트 https://code.visualstudio.com에 접속합니다. 메인 페이지에 큰 "Download for Windows" 버튼이 보일 것입니다. 이를 클릭하면 자동으로 사용 중인 운영체제에 맞는 버전이 다운로드됩니다.

**설치 시 중요한 옵션들 (반드시 체크):**

설치 과정에서 "Select Additional Tasks" 화면이 나타납니다. 이때 다음 옵션들을 반드시 체크해야 합니다:

☑ **Add "Open with Code" action to Windows Explorer file context menu**
   파일을 우클릭했을 때 "Code로 열기" 메뉴가 추가됩니다.

☑ **Add "Open with Code" action to Windows Explorer directory context menu**
   폴더를 우클릭했을 때도 "Code로 열기" 메뉴가 나타나 전체 프로젝트를 쉽게 열 수 있습니다.

☑ **Register Code as an editor for supported file types**
   .html, .css, .js 같은 코드 파일을 더블클릭하면 자동으로 VS Code에서 열립니다.

☑ **Add to PATH**
   명령 프롬프트에서 `code` 명령어로 VS Code를 실행할 수 있게 됩니다.

이 옵션들을 체크하지 않으면 나중에 레지스트리를 수정하거나 재설치해야 합니다. 따라서 처음부터 제대로 설정하는 것이 중요합니다.

**한국어 설정:**

설치 후 처음 실행하면 영어로 표시됩니다. 한국어로 변경하려면:

1. `Ctrl + Shift + X`를 눌러 확장 프로그램 패널을 엽니다
2. 검색창에 "Korean"을 입력합니다
3. "Korean Language Pack for Visual Studio Code"를 찾아 Install 버튼을 클릭합니다
4. 설치 완료 후 VS Code를 재시작하면 한국어로 변경됩니다

### 2-3 기본 사용법과 필수 확장

**화면 구성 이해하기**

VS Code를 처음 열면 크게 세 영역으로 나뉘어져 있습니다. 왼쪽 사이드바에는 파일 탐색기, 검색, Git 등의 도구가 있습니다. 중앙에는 코드를 편집하는 공간이 있습니다. 하단에는 터미널을 열 수 있는 공간이 있습니다.

**파일과 프로젝트 관리**

파일 하나만 편집할 때는 그냥 열면 됩니다. 하지만 웹사이트 프로젝트처럼 여러 파일을 함께 관리해야 할 때는 다릅니다. "폴더 열기"로 프로젝트 전체를 한 번에 관리하는 것이 훨씬 편리합니다. 새 파일을 만들 때는 왼쪽 사이드바에서 폴더명을 우클릭하고 "새 파일"을 선택하면 됩니다. 파일을 저장할 때는 `Ctrl + S`를 누르거나 자동 저장 기능을 켜두면 됩니다.

**필수 확장 프로그램**

VS Code의 진정한 힘은 확장 프로그램에서 나옵니다. 웹개발을 시작할 때 반드시 설치해야 할 세 가지 확장을 살펴봅니다.

**Prettier - Code formatter**

코드를 자동으로 보기 좋게 정렬해주는 도구입니다. 들여쓰기가 뒤죽박죽이거나 줄바꿈이 일관성 없어도, 저장할 때마다 자동으로 깔끔하게 정리됩니다. 예를 들어 `<div><p>안녕</p></div>`처럼 한 줄로 쓴 코드도 자동으로 계층 구조에 맞게 정렬해줍니다. 팀 프로젝트에서 코드 스타일을 통일할 때도 필수입니다.

**Live Server**

HTML 파일을 저장하면 브라우저가 자동으로 새로고침되어 변경사항을 즉시 확인할 수 있습니다. 파일을 우클릭하고 "Open with Live Server"를 선택하면 로컬 서버가 실행되며, 코드를 수정할 때마다 실시간으로 결과를 볼 수 있습니다. 두 개의 모니터를 사용한다면 한쪽에는 코드, 다른 쪽에는 브라우저를 띄워놓고 작업하면 매우 효율적입니다.

**Auto Rename Tag**

HTML 태그를 수정할 때 짝이 되는 태그를 자동으로 함께 수정해줍니다. 예를 들어 `<div>`를 `<section>`으로 바꾸면 닫는 태그 `</div>`도 자동으로 `</section>`으로 바뀝니다. 단순해 보이지만 실수를 크게 줄여주는 유용한 기능입니다.

### 2-4 VS Code로 문서 작성하기

**왜 VS Code로 문서를 작성할까?**

기존 Word의 한계점은 파일 개별 관리로 일관성 유지가 어렵고, 여러 파일 간 이동이 불편하며, 검색/치환 기능이 제한적이라는 것입니다.

반면 VS Code의 장점은 여러 문서를 하나의 프로젝트로 관리할 수 있고, 전체 문서에서 키워드 검색이 가능하며, Git으로 자동 버전 관리가 되고, 마크다운으로 전문적인 문서를 작성할 수 있다는 점입니다.

**마크다운으로 문서 작성하기**

마크다운은 VS Code에서 지원하는 문서 작성 방식으로 `.md` 확장자로 저장하며, 간단한 기호로 문서를 구조화할 수 있습니다.

**실습해보기:**
1. VS Code에서 `연습.md` 파일 생성
2. 다음 내용을 작성:

```markdown
# 프로젝트 계획서

## 1단계: 기획
- 아이디어 정리
- **핵심 기능** 정의

## 2단계: 개발
- 코딩 시작
- 테스트 진행
```

3. 파일 탭 우클릭 → "미리 보기로 열기"
4. 왼쪽에는 코드, 오른쪽에는 결과 확인

### 2-5 VS Code에서 AI 도구 활용하기

VS Code에서 AI를 사용하면 코드 작성 효율이 대폭 향상됩니다. 에디터를 벗어나지 않고 AI 도움을 받을 수 있고, 코드 작성 중 즉시 질문과 해결이 가능하며, 파일 관리와 AI 상담을 한 곳에서 처리할 수 있습니다.

**Claude Code (터미널)**는 VS Code 터미널에서 `claude` 명령어로 실행하며, 파일 편집과 프로젝트 분석 등 실제 작업을 수행할 수 있습니다. **Claude Chat (하단 패널)**은 웹 Claude를 VS Code 안에 임베드한 것으로 빠른 질문과 답변에 적합합니다.

실제로 에러가 발생하면 Chat으로 빠른 질문을 하고, 복잡한 기능 구현이 필요하면 Claude Code로 실제 작업을 수행하는 방식으로 활용할 수 있습니다.

이외에도 GitHub Copilot, ChatGPT 등 다양한 AI 코딩 도구들이 VS Code 확장으로 제공되므로, 자신에게 맞는 도구를 선택해서 사용할 수 있습니다.

### 2-6 실제 작업 흐름과 활용법

**VS Code로 간단한 웹페이지 만들기**

VS Code로 실제 웹페이지를 만드는 과정을 단계별로 살펴보겠습니다.

**1단계: 프로젝트 폴더 준비**
먼저 바탕화면에 "내첫웹사이트"라는 폴더를 만듭니다. VS Code를 열고 "폴더 열기"를 클릭한 후 해당 폴더를 선택합니다. 왼쪽 파일 탐색기에서 우클릭 후 "새 파일"을 선택하고 `index.html`이라는 이름으로 파일을 생성합니다.

**2단계: 기본 코드 작성**
```html
<!DOCTYPE html>
<html>
<head>
    <title>내 첫 웹사이트</title>
</head>
<body>
    <h1>안녕하세요</h1>
    <p>이것은 VS Code로 만든 첫 웹페이지입니다.</p>
</body>
</html>
```

**3단계: 브라우저에서 확인**
Live Server 확장을 설치한 후 파일을 우클릭하고 "Open with Live Server"를 선택하면 브라우저가 자동으로 열리고 결과를 즉시 확인할 수 있습니다. 코드를 수정하고 저장하면 브라우저가 자동으로 새로고침되어 변경사항이 바로 반영됩니다.

**문서 작성 워크플로우**

VS Code는 코드 작성뿐만 아니라 대량 문서 프로젝트에도 효율적입니다. 예를 들어 35개 장으로 구성된 전자책을 작성한다고 가정해보겠습니다.

첫째, VS Code로 전체 프로젝트 구조를 생성합니다. 각 장별로 폴더를 만들고 파일을 체계적으로 구성합니다. 둘째, 마크다운으로 각 장을 작성합니다. 마크다운은 간단한 기호로 문서를 구조화할 수 있어 효율적입니다. 셋째, 전체 검색 기능으로 용어를 통일합니다. 예를 들어 "인공지능"과 "AI"가 혼용되고 있다면 한 번에 모두 찾아서 통일할 수 있습니다. 넷째, Git으로 변경 이력을 추적합니다. 언제 무엇을 수정했는지 모두 기록되어 필요시 이전 버전으로 돌아갈 수 있습니다. 다섯째, 완성된 마크다운 파일을 HTML이나 PDF로 자동 변환할 수 있습니다.

이러한 워크플로우를 통해 파일 관리 시간을 50% 절약하고, 내용 일관성을 대폭 향상시키며, 수정/보완 작업을 70% 효율화할 수 있습니다.

## 3. Cursor - AI 특화 코드 에디터

### 3-1 Cursor를 고려하는 이유

VS Code로 기본기를 다진 후, 더 강력한 AI 기능이 필요하다면 Cursor를 고려해볼 수 있습니다. Cursor는 2023년에 등장한 새로운 코드 에디터로, AI와의 협업을 중심으로 설계되었습니다.

**Cursor의 특징:**

Cursor는 VS Code를 포크(fork)해서 만든 완전 독립적인 에디터입니다. 포크란 기존 소프트웨어의 소스코드를 복사해서 새로운 프로젝트로 개발하는 것을 말합니다. 따라서 VS Code의 모든 장점을 그대로 가지면서도 AI 기능에 특화된 추가 기능들을 제공합니다.

Cursor의 가장 큰 특징은 AI 자동완성에 특화되어 있다는 점입니다. 코드를 작성하다가 Tab 키만 누르면 AI가 다음에 올 코드를 예측해서 자동으로 생성해줍니다. 예를 들어 함수 이름만 작성하면 전체 함수 본문을 AI가 작성해주는 식입니다.

또한 자연어로 코드를 요청할 수 있습니다. "사용자 로그인 폼을 만들어줘"라고 한국어로 입력하면 HTML, CSS, JavaScript 코드를 한 번에 생성해줍니다. 프로그래밍 언어를 잘 모르는 초보자도 원하는 기능을 설명하기만 하면 코드를 얻을 수 있습니다.

### 3-2 Cursor 설치와 기본 사용

**설치 과정:**

Cursor 설치는 VS Code만큼이나 간단합니다. 먼저 공식 웹사이트 https://cursor.sh에 접속합니다. 메인 페이지에서 "Download for Windows" 버튼을 클릭하면 설치 파일이 다운로드됩니다.

설치 과정에서 특별한 점은 VS Code 설정을 자동으로 가져올 수 있다는 것입니다. 이미 VS Code를 사용하고 있다면 설치된 확장 프로그램, 테마, 단축키 설정 등을 그대로 Cursor로 옮겨올 수 있습니다. 새로운 도구를 배우는 부담 없이 바로 사용할 수 있다는 큰 장점입니다.

**핵심 기능 3가지:**

첫째, **Tab 자동완성** 기능입니다. 코드를 작성하다가 주석으로 "// 사용자 정보를 데이터베이스에 저장하는 함수"라고 쓰고 Tab 키를 누르면, AI가 해당 기능을 구현한 전체 함수 코드를 자동으로 생성합니다. 마치 숙련된 개발자가 옆에서 코드를 대신 써주는 것 같은 경험입니다.

둘째, **Ctrl+K 수정** 기능입니다. 이미 작성된 코드를 선택한 후 Ctrl+K를 누르고 "이 코드를 더 효율적으로 만들어줘" 또는 "에러 처리를 추가해줘"같은 자연어 요청을 입력하면, AI가 코드를 자동으로 수정해줍니다. 코드 리뷰를 받는 것처럼 개선점을 즉시 적용할 수 있습니다.

셋째, **AI 채팅** 기능입니다. 오른쪽 패널에 있는 채팅창에서 코드에 대한 질문을 할 수 있습니다. "이 함수가 무엇을 하는지" "이 에러가 발생하는 이유" 같은 질문에 AI가 상세하게 답변해줍니다. 현재 열려 있는 파일의 컨텍스트를 이해하고 있어서 더 정확한 답변을 제공합니다.

### 3-3 VS Code vs Cursor 언제 사용할까?

두 도구 모두 훌륭하지만, 상황에 따라 적절한 선택이 다를 수 있습니다.

**VS Code를 주로 사용하는 경우:**

안정적인 개발 환경이 필요한 경우 VS Code가 더 적합합니다. 수년간 검증된 도구이며, 대부분의 버그가 해결되어 있고, 방대한 커뮤니티 지원을 받을 수 있습니다. 프로덕션 환경의 중요한 프로젝트를 진행할 때는 안정성이 검증된 VS Code가 안전한 선택입니다.

팀 협업에서 표준 도구를 사용해야 하는 경우에도 VS Code가 적합합니다. 대부분의 개발팀이 VS Code를 사용하고 있어서 설정 공유, 문제 해결, 팀원 간 도움을 주고받기가 쉽습니다.

기본적인 AI 도움으로 충분한 경우라면 굳이 Cursor로 바꿀 필요가 없습니다. VS Code에도 GitHub Copilot이나 Claude 같은 AI 도구를 설치할 수 있으며, 대부분의 AI 코딩 작업을 충분히 처리할 수 있습니다.

**Cursor를 추가로 사용하는 경우:**

AI 자동완성을 적극 활용하고 싶을 때는 Cursor가 더 나은 선택입니다. Cursor의 AI 기능은 VS Code의 확장 프로그램보다 더 깊게 통합되어 있어서 반응 속도가 빠르고 정확도가 높습니다.

실험적인 코드 작성이나 프로토타이핑을 할 때도 Cursor가 유용합니다. 빠르게 아이디어를 코드로 구현해보고 싶을 때, AI의 도움으로 신속하게 프로토타입을 만들 수 있습니다.

새로운 기술을 학습할 때도 Cursor가 도움이 됩니다. 처음 접하는 프로그래밍 언어나 프레임워크를 사용할 때, AI가 실시간으로 코드 예제를 생성하고 설명해주므로 학습 속도가 빨라집니다.

많은 개발자들이 두 도구를 병행해서 사용합니다. 메인 프로젝트는 VS Code로, 실험이나 학습은 Cursor로 진행하는 식입니다. 두 도구 모두 무료이므로 필요에 따라 유연하게 선택할 수 있습니다.


## 4. 기타 에디터 및 IDE 소개

### 4-1 기타 에디터

**Sublime Text**
매우 빠른 속도가 장점이지만 확장 기능이 제한적이고 유료입니다. 가벼운 편집 작업에는 좋지만 대규모 프로젝트에는 부족합니다.

**Notepad++**
Windows 전용 무료 에디터로 가볍지만 기본 기능만 제공합니다. 간단한 텍스트 편집이나 로그 파일 확인용으로 적합합니다.

**Vim**
터미널 기반 에디터로 매우 빠르지만 학습 곡선이 높습니다. 서버 환경에서 직접 파일을 편집할 때 유용합니다.

### 4-2 기타 IDE

**IntelliJ IDEA**
Java 개발에 최적화된 강력한 IDE이지만 유료이고 무겁습니다. 스프링 프레임워크 개발이나 대규모 자바 프로젝트에 적합합니다.

**Visual Studio**
Microsoft .NET 전용 IDE로 윈도우 앱과 게임 개발에 특화되어 있습니다. C#이나 C++ 개발에 강력한 도구입니다.

**Eclipse**
오픈소스 Java IDE로 무료이지만 복잡하고 느립니다. 예전에는 많이 사용했지만 현재는 IntelliJ IDEA에 밀려 점유율이 감소하고 있습니다.
