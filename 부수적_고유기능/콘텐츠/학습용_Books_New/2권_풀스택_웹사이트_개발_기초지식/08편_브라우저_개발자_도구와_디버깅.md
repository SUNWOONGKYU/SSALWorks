# 08편 | 브라우저 개발자 도구와 디버깅

---

7편에서 JavaScript를 배웠으니, 이제 **브라우저 개발자 도구**를 활용하는 방법을 배워보겠습니다. 개발자 도구는 **웹 개발자의 현미경**과 같습니다. 웹페이지가 어떻게 구성되어 있는지, 어디서 문제가 발생하는지를 상세하게 들여다볼 수 있게 해주는 필수 도구입니다.

## 1. 개발자 도구란?

### 1-1. 개발자 도구의 역할

**개발자 도구(Developer Tools, DevTools)**는 모든 최신 브라우저에 내장된 도구입니다. **자동차의 계기판**과 비슷하게, 웹페이지의 상태를 실시간으로 보여주고 문제를 진단할 수 있게 해줍니다.

개발자 도구로 할 수 있는 일:
- HTML 구조 확인 및 수정
- CSS 스타일 실시간 조정
- JavaScript 코드 실행 및 디버깅
- 네트워크 요청 모니터링
- 성능 분석

### 1-2. 개발자 도구 열기

**Chrome/Edge**:
- 키보드: `F12` 또는 `Ctrl + Shift + I` (Mac: `Cmd + Option + I`)
- 마우스: 페이지에서 우클릭 → "검사" 클릭

**Firefox**:
- 키보드: `F12` 또는 `Ctrl + Shift + I`
- 마우스: 우클릭 → "요소 검사"

## 2. Elements 패널 - HTML/CSS 탐색

### 2-1. HTML 구조 확인

Elements 패널에서는 웹페이지의 **DOM(Document Object Model) 구조**를 볼 수 있습니다.

**주요 기능**:
- 요소 선택: 돋보기 아이콘 클릭 후 페이지 요소 클릭
- 요소 수정: HTML을 직접 더블클릭하여 수정
- 요소 삭제: 요소 선택 후 `Delete` 키

### 2-2. CSS 스타일 실시간 수정

선택한 요소의 CSS를 오른쪽 **Styles** 패널에서 확인하고 수정할 수 있습니다.

**활용 팁**:
- 속성값 클릭하여 직접 수정
- 체크박스로 속성 on/off 토글
- 새 속성 추가 가능
- **변경사항은 새로고침하면 사라집니다** (임시 테스트용)

### 2-3. Box Model 시각화

선택한 요소의 **박스 모델**을 시각적으로 확인할 수 있습니다:
- Content (내용): 파란색
- Padding (안쪽 여백): 초록색
- Border (테두리): 노란색
- Margin (바깥 여백): 주황색

## 3. Console 패널 - JavaScript 실행

### 3-1. Console 기본 사용법

Console은 **JavaScript 명령어를 직접 실행**할 수 있는 대화형 환경입니다.

**자주 사용하는 명령어**:

```javascript
// 메시지 출력
console.log("Hello, World!");

// 경고 메시지 (노란색)
console.warn("이것은 경고입니다");

// 에러 메시지 (빨간색)
console.error("에러가 발생했습니다");

// 데이터 테이블 형식 출력
console.table([{name: "철수", age: 25}, {name: "영희", age: 30}]);

// 실행 시간 측정
console.time("작업");
// ... 코드 실행 ...
console.timeEnd("작업");
```

### 3-2. 에러 메시지 읽기

Console에 표시되는 에러 메시지는 문제 해결의 **첫 번째 단서**입니다.

**에러 메시지 구조**:
```
Uncaught TypeError: Cannot read property 'name' of undefined
    at script.js:15:10
```

- **에러 유형**: `TypeError` - 잘못된 타입의 값을 사용
- **에러 내용**: `undefined`에서 `name` 속성을 읽으려 함
- **위치**: `script.js` 파일의 15번째 줄, 10번째 문자

### 3-3. 자주 발생하는 에러 유형

| 에러 유형 | 의미 | 해결 방법 |
|----------|------|----------|
| `ReferenceError` | 존재하지 않는 변수 참조 | 변수명 오타 확인 |
| `TypeError` | 잘못된 타입 사용 | null/undefined 체크 |
| `SyntaxError` | 문법 오류 | 괄호, 따옴표 확인 |

## 4. Sources 패널 - 디버깅

### 4-1. Breakpoint (중단점) 설정

**Breakpoint**는 코드 실행을 특정 지점에서 멈추게 하는 기능입니다.

**사용 방법**:
1. Sources 패널에서 파일 선택
2. 멈추고 싶은 줄 번호 클릭 (파란색 마커 표시)
3. 페이지 새로고침 또는 해당 코드 실행
4. 코드가 해당 줄에서 멈춤

### 4-2. 디버깅 제어

코드가 멈추면 다음 버튼들로 실행을 제어할 수 있습니다:

- **Resume (▶)**: 다음 breakpoint까지 실행
- **Step Over (⤵)**: 현재 줄 실행 후 다음 줄로
- **Step Into (↓)**: 함수 내부로 들어감
- **Step Out (↑)**: 현재 함수 밖으로 나감

### 4-3. Watch와 Scope

**Watch**: 특정 변수나 표현식의 값을 계속 모니터링
**Scope**: 현재 위치에서 접근 가능한 모든 변수 확인

## 5. Network 패널 - 네트워크 분석

### 5-1. 네트워크 요청 확인

Network 패널에서는 페이지가 **서버와 주고받는 모든 데이터**를 확인할 수 있습니다.

**표시 정보**:
- **Name**: 요청한 리소스 이름
- **Status**: 응답 상태 코드 (200, 404 등)
- **Type**: 리소스 유형 (document, script, xhr 등)
- **Size**: 파일 크기
- **Time**: 응답 시간

### 5-2. 상태 코드 이해하기

| 코드 | 의미 | 설명 |
|------|------|------|
| 200 | OK | 정상 응답 |
| 301 | Moved | 영구 이동 |
| 304 | Not Modified | 캐시 사용 |
| 400 | Bad Request | 잘못된 요청 |
| 404 | Not Found | 리소스 없음 |
| 500 | Server Error | 서버 오류 |

### 5-3. API 응답 확인

XHR/Fetch 필터를 사용하면 **API 요청만** 필터링하여 볼 수 있습니다.

요청을 클릭하면:
- **Headers**: 요청/응답 헤더 정보
- **Preview**: 응답 데이터 미리보기
- **Response**: 원본 응답 데이터

## 6. Application 패널 - 저장소 관리

### 6-1. 브라우저 저장소

**Local Storage**: 브라우저를 닫아도 유지되는 데이터
**Session Storage**: 탭을 닫으면 사라지는 데이터
**Cookies**: 서버와 주고받는 작은 데이터

### 6-2. 저장소 조작

Application 패널에서 직접:
- 저장된 데이터 확인
- 새 데이터 추가
- 기존 데이터 수정/삭제
- 전체 데이터 초기화

## 7. Performance 패널 - 성능 분석

### 7-1. 성능 측정하기

1. Performance 패널 열기
2. 녹화 버튼 (●) 클릭
3. 측정하려는 동작 수행
4. 정지 버튼 클릭
5. 결과 분석

### 7-2. 성능 지표 이해

- **FPS (Frames Per Second)**: 초당 프레임 수 (60이 이상적)
- **CPU**: CPU 사용량
- **NET**: 네트워크 활동
- **Main**: 메인 스레드 활동

## 8. 실전 디버깅 팁

### 8-1. 효과적인 디버깅 순서

```
1. 에러 메시지 확인 (Console)
2. 에러 발생 위치 확인 (Sources)
3. Breakpoint로 값 확인
4. 원인 파악 및 수정
5. 테스트
```

### 8-2. console.log 대신 debugger

코드에 `debugger;`를 작성하면 해당 위치에서 자동으로 멈춥니다:

```javascript
function calculateTotal(items) {
    debugger;  // 여기서 자동으로 멈춤
    return items.reduce((sum, item) => sum + item.price, 0);
}
```

### 8-3. 조건부 Breakpoint

특정 조건에서만 멈추도록 설정할 수 있습니다:
1. 줄 번호 우클릭
2. "Add conditional breakpoint" 선택
3. 조건 입력 (예: `i === 5`)

## 핵심 정리

| 패널 | 용도 | 단축키 |
|------|------|--------|
| Elements | HTML/CSS 검사 | `Ctrl + Shift + C` |

| Sources | 디버깅/Breakpoint | `Ctrl + Shift + S` |
| Network | 네트워크 모니터링 | `Ctrl + Shift + E` |
| Application | 저장소 관리 | - |
| Performance | 성능 분석 | - |
