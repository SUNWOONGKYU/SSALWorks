## 3. 필요한 내용 찾기

### 상황별 참조 가이드

**"Task를 어떻게 나눠야 하지?"**
→ 섹션 6: Task 선정 원칙

**"Instruction은 어떻게 작성하지?"**
→ 섹션 7: Task Instruction 작성
→ 섹션 8: Verification Instruction 작성

**"검증은 어떻게 하지?"**
→ 섹션 9: 3단계 검증 시스템
→ 섹션 10: Stage Gate 시스템

**"Supabase 설정이 안 돼"**
→ 섹션 15: 설치 확인 및 문제 해결
→ 섹션 22: 문제 해결 A to Z

**"22개 속성이 뭐였지?"**
→ 섹션 3: 22개 속성 정의

**"Stage/Area 정의가 뭐였지?"**
→ 섹션 2: SSALWorks 표준 프레임워크

### 검색 팁

**MD 파일에서 검색:**
```
Ctrl+F (또는 Cmd+F) → 키워드 입력
```

**키워드 예시:**
- `Task ID` → Task ID 체계 관련
- `dependencies` → 의존성 관련
- `verification` → 검증 관련
- `Stage Gate` → Gate 통과 조건

## 4. 매뉴얼 업데이트 방법

### 업데이트가 필요한 경우

- 새로운 규칙 추가
- 기존 규칙 변경
- 오류 수정
- 예시 추가

### 업데이트 절차

```
1. 변경 사항 확인
2. 해당 섹션 수정
3. 버전 번호 증가 (v3.4 → v3.5)
4. 섹션 27 버전 이력에 기록
5. 최종 수정일 업데이트
```

### 버전 이력 기록 형식

```markdown
## 27. 버전 이력

### v3.5 (2025-12-21)
- 변경 내용 1
- 변경 내용 2

### v3.4 (2025-12-19)
- .claude/rules/ 참조 추가
- 파일 명명 규칙 업데이트
```

## 5. HTML vs MD 버전

### 두 가지 버전

```
manual/
├── PROJECT_SAL_GRID_MANUAL.md  (원본, 170KB)
└── manual.html                  (HTML 변환본)
```

### 각각의 용도

| 버전 | 용도 | 장점 |
|------|------|------|
| MD | 원본, 수정용 | 편집 쉬움, Git 추적 |
| HTML | 조회용 | 브라우저에서 바로 열기, 목차 내비게이션 |

### 언제 어떤 것을 사용하나

**MD 버전 사용:**
- 매뉴얼 수정할 때
- 특정 섹션 복사할 때
- AI가 참조할 때

**HTML 버전 사용:**
- 브라우저에서 빠르게 조회할 때
- 목차 클릭으로 이동할 때
- 비개발자가 읽을 때

### HTML 변환

MD 파일을 수정한 후 HTML도 업데이트해야 한다.

```bash
# 빌드 스크립트 실행
node Production/build-web-assets.js --manual
```

## 6. AI가 매뉴얼을 활용하는 방법

### 새 세션 시작 시

```
1. CLAUDE.md 읽기 (핵심 규칙)
2. 필요 시 매뉴얼 특정 섹션 참조
3. 작업 수행
```

### 규칙 확인 시

```
매뉴얼보다 .claude/rules/ 폴더 우선 참조

규칙 파일:
- 01_file-naming.md
- 02_save-location.md
- 03_area-stage.md
- 04_grid-writing.md
- 05_execution-process.md
- 06_verification.md
```

### 상세 내용 필요 시

규칙 파일에 없는 상세 내용은 매뉴얼에서 찾는다.

```
예: "Stage Gate 리포트 형식이 뭐지?"
→ 매뉴얼 섹션 10: Stage Gate 시스템
```

## 7. 매뉴얼 활용 팁

### Tip 1: 목차 활용

27개 섹션이 있으므로 목차를 먼저 확인하고 필요한 섹션으로 이동한다.

### Tip 2: PART 단위로 파악

전체를 다 읽기보다 현재 작업에 해당하는 PART만 집중해서 읽는다.

```
Grid 생성 중 → PART 2
검증 중 → PART 3
문제 발생 → PART 6
```

### Tip 3: 예시 중심으로 이해

매뉴얼에는 많은 예시가 있다. 정의보다 예시를 보면 더 빨리 이해된다.

### Tip 4: 최신 버전 확인

매뉴얼 상단의 버전과 최종 수정일을 확인한다. 오래된 버전이면 최신 내용이 누락되었을 수 있다.
