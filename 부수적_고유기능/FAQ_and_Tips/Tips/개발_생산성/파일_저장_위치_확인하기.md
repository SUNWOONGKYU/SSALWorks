# 파일 저장 위치 확인하기

## 💡 Tip

파일을 만들기 전에 **PROJECT_DIRECTORY_STRUCTURE.md**를 확인하세요.

## 왜 중요한가?

- 파일이 흩어지면 찾기 어려움
- 프로젝트 구조가 무너짐
- 팀원 간 혼란 발생
- Git 관리 어려움

## SSAL Works 폴더 구조

```
C:\!SSAL_Works_Private\
├── P0_예비-0차/           # 아이디어
├── P1_예비-1차/           # 분석
├── P2_예비-2차/           # 설계
├── P3_프로토타입_제작/    # 프로토타입
├── S0_Project-SSAL-Grid/  # SAL Grid
├── S1_개발_준비/          # 설정
├── S2_개발-1차/           # 개발
├── Production/            # 배포용
└── 부수적_고유기능/        # 부가 기능
```

## 저장 위치 결정 규칙

**1. Stage + Area 기반**
```
Task ID: S2BA1
→ S2_개발-1차/Backend_API/
```

**2. 코드는 이중 저장**
```
Frontend 코드:
  - P3_프로토타입_제작/Frontend/Prototype/
  - Production/Frontend/
```

## 모르겠을 때

**PROJECT_DIRECTORY_STRUCTURE.md 확인 후에도 모르겠으면:**

```
"이 파일을 저장할 위치를 모르겠습니다.

파일명: config.json
용도: API 설정

제안:
A) S1_개발_준비/Backend_Infra/
B) Production/config/

어디가 맞을까요?"
```

## 절대 금지

❌ 루트 디렉토리에 임의 파일 생성
❌ 예상치 못한 위치에 파일 생성
❌ 승인 없이 새 폴더 생성
