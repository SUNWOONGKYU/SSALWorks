# Task를 작게 나누는 기준

## 💡 Tip

하나의 Task는 **2시간 이내**에 완료할 수 있는 크기로 나누세요.

## 왜 작게 나눠야 하나?

- 진행 상황 파악이 쉬움
- 문제 발생 시 빠른 대응 가능
- 검증이 용이함
- 커밋 단위가 명확해짐

## Task 크기 판단 기준

| 크기 | 시간 | 예시 | 권장 |
|------|------|------|------|
| Too Small | 30분 미만 | 변수명 변경 | ❌ 합치기 |
| Good | 30분~2시간 | API 1개 구현 | ✅ 적절 |
| Too Big | 2시간 초과 | 전체 인증 시스템 | ❌ 분리 |

## 나누는 방법 예시

**❌ 너무 큰 Task:**
```
Task: 회원 인증 시스템 구현
```

**✅ 적절하게 분리:**
```
Task 1: 회원가입 API 구현 (1시간)
Task 2: 로그인 API 구현 (1시간)
Task 3: 비밀번호 재설정 API 구현 (1시간)
Task 4: 인증 미들웨어 구현 (30분)
Task 5: 프론트엔드 로그인 폼 구현 (1시간)
```
