# 서브에이전트 효과적으로 활용하기

## 💡 Tip

복잡한 작업은 서브에이전트에게 맡기면 더 효율적입니다.

## SSAL Works 서브에이전트 목록

| 서브에이전트 | 용도 |
|------------|------|
| `backend-developer` | API, 서버 로직 개발 |
| `frontend-developer` | UI, 프론트엔드 개발 |
| `database-developer` | DB 스키마, 쿼리 작성 |
| `devops-troubleshooter` | 배포, 인프라 문제 해결 |
| `code-reviewer` | 코드 리뷰 |
| `test-engineer` | 테스트 작성 |

## 활용 예시

**API 개발 요청:**
```
"backend-developer 서브에이전트로 회원가입 API 만들어줘"
```

**코드 리뷰 요청:**
```
"code-reviewer로 이 코드 리뷰해줘"
```

## 서브에이전트 투입 원칙

SSAL Works의 절대 규칙:
- **작업**: Task Agent 서브에이전트
- **검증**: Verification Agent 서브에이전트
- **작성자와 검증자는 분리!**

```
잘못된 예: Main Agent가 직접 Task 수행 ❌
올바른 예: 서브에이전트에게 Task 위임 ✅
```

## 병렬 실행

여러 작업을 동시에 진행:
```
"frontend-developer로 UI 만들고,
동시에 backend-developer로 API 만들어줘"
```
