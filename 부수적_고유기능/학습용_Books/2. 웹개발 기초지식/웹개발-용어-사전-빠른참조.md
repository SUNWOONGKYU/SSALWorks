# 웹개발 용어 사전 & 빠른 참조 가이드

---

## 📚 웹개발 용어 사전 (가나다순)

### ㄱ

**가비지 컬렉션 (Garbage Collection)**
- 메모리에서 더 이상 사용하지 않는 데이터를 자동으로 제거
- JavaScript가 자동으로 처리하는 메모리 관리 기능

**가상 DOM (Virtual DOM)**
- React가 사용하는 메모리상의 가상 문서 구조
- 실제 DOM 업데이트를 최적화하는 기술

**가상 호스팅 (Virtual Hosting)**
- 하나의 서버에서 여러 웹사이트를 운영하는 기술
- 공유 호스팅의 기반 기술

**게이트웨이 (Gateway)**
- 서로 다른 네트워크를 연결하는 통로
- API Gateway는 마이크로서비스의 진입점 역할

**고차 함수 (Higher-Order Function)**
- 다른 함수를 인자로 받거나 함수를 반환하는 함수
- map, filter, reduce가 대표적

**공개키 암호화 (Public Key Cryptography)**
- 공개키와 비밀키를 사용하는 암호화 방식
- HTTPS의 기반 기술

**구조 분해 할당 (Destructuring)**
- 배열이나 객체의 값을 개별 변수로 추출하는 문법
- ES6에서 추가된 JavaScript 기능

**그래프QL (GraphQL)**
- Facebook이 만든 API 쿼리 언어
- REST API의 대안으로 사용

**그리드 (Grid)**
- CSS Grid Layout으로 2차원 레이아웃 구성
- 행과 열을 동시에 제어 가능

**글로벌 스코프 (Global Scope)**
- 프로그램 전체에서 접근 가능한 변수 범위
- window 객체가 글로벌 스코프

**기능 테스트 (Functional Testing)**
- 소프트웨어의 기능이 제대로 작동하는지 확인
- 사용자 시나리오 기반 테스트

**깃 (Git)**
- 분산 버전 관리 시스템
- 코드 변경 이력을 추적하고 협업을 도와주는 도구

**깃허브 (GitHub)**
- Git 저장소 호스팅 서비스
- 오픈소스 프로젝트의 중심지

**깃랩 (GitLab)**
- Git 저장소 호스팅과 CI/CD 통합 플랫폼
- DevOps 라이프사이클 전체 지원

### ㄴ

**네임스페이스 (Namespace)**
- 변수나 함수의 충돌을 방지하는 공간 분리
- 모듈 시스템의 기본 개념

**네임서버 (Name Server)**
- 도메인 이름을 IP 주소로 변환하는 서버
- DNS의 핵심 구성 요소

**네이티브 앱 (Native App)**
- 특정 운영체제용으로 개발된 애플리케이션
- 웹앱과 대비되는 개념

**노드 (Node)**
- DOM 트리의 각 요소
- Node.js의 약칭

**노드 모듈 (Node Modules)**
- Node.js에서 사용하는 패키지 저장 폴더
- npm이 관리하는 의존성 파일들

**노드제이에스 (Node.js)**
- 서버에서 JavaScript를 실행하는 런타임 환경
- Chrome V8 엔진 기반

**논블로킹 (Non-blocking)**
- 작업이 완료될 때까지 기다리지 않는 처리 방식
- Node.js의 핵심 특징

### ㄷ

**다크 모드 (Dark Mode)**
- 어두운 배경에 밝은 글자를 사용하는 UI 테마
- CSS media query로 구현

**단방향 데이터 흐름 (Unidirectional Data Flow)**
- React의 데이터 전달 방식
- 부모에서 자식으로만 데이터 전달

**단위 테스트 (Unit Test)**
- 가장 작은 단위의 코드를 테스트
- 함수나 메서드 단위로 검증

**대역폭 (Bandwidth)**
- 네트워크로 전송 가능한 데이터 양
- 웹사이트 속도에 직접적 영향

**데이터 바인딩 (Data Binding)**
- 데이터와 UI를 연결하는 기술
- 단방향/양방향 바인딩으로 구분

**데이터베이스 (Database)**
- 체계적으로 정리된 데이터 저장소
- 관계형(SQL)과 비관계형(NoSQL)로 구분

**데이터베이스 스키마 (Database Schema)**
- 데이터베이스의 구조와 제약조건 정의
- 테이블, 컬럼, 관계 등을 설계

**도커 (Docker)**
- 컨테이너 기반 가상화 플랫폼
- 애플리케이션을 격리된 환경에서 실행

**도메인 (Domain)**
- 웹사이트의 인터넷 주소 (예: google.com)
- IP 주소를 사람이 기억하기 쉬운 문자로 변환

**도메인 네임 시스템 (DNS)**
- 도메인을 IP 주소로 변환하는 시스템
- 인터넷의 전화번호부 역할

**동기/비동기 (Synchronous/Asynchronous)**
- 동기: 순차적으로 처리
- 비동기: 동시에 여러 작업 처리

**동적 타이핑 (Dynamic Typing)**
- 변수의 타입을 실행 시간에 결정
- JavaScript의 특징

**디버깅 (Debugging)**
- 코드의 오류를 찾아 수정하는 과정
- console.log, 브레이크포인트 활용

**디바운싱 (Debouncing)**
- 연속적인 이벤트를 그룹화하여 처리
- 검색 입력 최적화에 사용

**디펜던시 (Dependency)**
- 프로젝트가 의존하는 외부 패키지
- package.json에서 관리

### ㄹ

**라우터 (Router)**
- URL에 따라 다른 페이지를 표시하는 기능
- SPA의 핵심 구성 요소

**라우팅 (Routing)**
- 요청된 경로에 따라 처리를 분기
- 프론트엔드/백엔드 모두에서 사용

**라이브러리 (Library)**
- 재사용 가능한 코드 모음
- 특정 기능을 제공하는 도구

**람다 함수 (Lambda Function)**
- 익명 함수의 다른 이름
- 서버리스 컴퓨팅의 실행 단위

**랜더링 (Rendering)**
- HTML, CSS를 해석해 화면에 표시하는 과정
- 서버사이드/클라이언트사이드 렌더링으로 구분

**레디스 (Redis)**
- 메모리 기반 키-값 데이터베이스
- 캐싱과 세션 저장에 활용

**레이아웃 (Layout)**
- 웹페이지 요소의 배치 구조
- Flexbox, Grid로 구성

**레이지 로딩 (Lazy Loading)**
- 필요한 시점에 리소스를 로드하는 기법
- 초기 로딩 속도 개선

**레지스트리 (Registry)**
- 패키지를 저장하고 배포하는 저장소
- npm registry가 대표적

**레퍼런스 (Reference)**
- 메모리 주소를 가리키는 참조
- 객체와 배열은 참조로 전달

**레플리카 (Replica)**
- 데이터베이스의 복제본
- 가용성과 성능 향상 목적

**로그 (Log)**
- 시스템이나 애플리케이션의 동작 기록
- 디버깅과 모니터링에 필수

**로드 밸런서 (Load Balancer)**
- 트래픽을 여러 서버에 분산
- 서버 부하를 균등하게 배분

**로컬 스토리지 (Local Storage)**
- 브라우저에 데이터를 영구 저장하는 공간
- 5-10MB 용량 제한

**리덕스 (Redux)**
- JavaScript 앱의 상태 관리 라이브러리
- React와 자주 함께 사용

**리듀서 (Reducer)**
- 상태 변화를 처리하는 순수 함수
- Redux의 핵심 개념

**리액트 (React)**
- Facebook이 만든 UI 라이브러리
- 컴포넌트 기반 개발

**리스트 (List)**
- HTML의 ul, ol 태그로 만드는 목록
- 순서 있는/없는 목록으로 구분

**리팩토링 (Refactoring)**
- 코드의 기능은 유지하며 구조를 개선
- 가독성과 유지보수성 향상

**린트 (Lint)**
- 코드의 오류나 스타일 문제를 검사
- ESLint가 JavaScript에서 사용

### ㅁ

**마이그레이션 (Migration)**
- 데이터나 시스템을 이전하는 작업
- 데이터베이스 스키마 변경 관리

**마이크로서비스 (Microservices)**
- 작은 독립적 서비스로 구성된 아키텍처
- 각 서비스가 독립적으로 배포 가능

**마진 (Margin)**
- 요소의 외부 여백
- CSS box model의 구성 요소

**마크다운 (Markdown)**
- 간단한 문법의 문서 작성 포맷
- README 파일 작성에 사용

**마크업 (Markup)**
- HTML로 문서 구조를 표시하는 것
- 태그를 사용해 내용의 의미를 정의

**맵 (Map)**
- 키-값 쌍을 저장하는 자료구조
- JavaScript의 Map 객체

**메서드 (Method)**
- 객체에 속한 함수
- HTTP 요청 방식 (GET, POST 등)

**메모이제이션 (Memoization)**
- 함수 실행 결과를 캐싱하는 최적화 기법
- React.memo, useMemo 활용

**메타 태그 (Meta Tag)**
- HTML 문서의 메타데이터 정의
- SEO와 소셜 미디어 공유에 중요

**명령어 (Command)**
- CLI에서 실행하는 텍스트 명령
- git, npm 등의 명령어

**모듈 (Module)**
- 재사용 가능한 코드 단위
- ES6 모듈, CommonJS 모듈

**모킹 (Mocking)**
- 테스트를 위해 가짜 객체나 함수를 만드는 것
- 단위 테스트에서 활용

**목업 (Mockup)**
- 디자인의 시각적 프로토타입
- 실제 기능 없이 모양만 구현

**몽고DB (MongoDB)**
- 대표적인 NoSQL 데이터베이스
- JSON 형태로 데이터 저장

**뮤테이션 (Mutation)**
- 데이터를 직접 변경하는 것
- GraphQL에서 데이터 변경 작업

**미니파이 (Minify)**
- 코드에서 불필요한 문자를 제거하여 용량 축소
- 공백, 줄바꿈, 주석 제거

**미들웨어 (Middleware)**
- 요청과 응답 사이에서 동작하는 소프트웨어
- Express.js에서 중요한 개념

### ㅂ

**바닐라 자바스크립트 (Vanilla JavaScript)**
- 프레임워크나 라이브러리 없는 순수 JavaScript
- 기본 JavaScript만 사용

**바벨 (Babel)**
- JavaScript 트랜스파일러
- 최신 문법을 구버전 브라우저에서 동작하도록 변환

**반응형 디자인 (Responsive Design)**
- 화면 크기에 따라 레이아웃이 변하는 디자인
- 미디어 쿼리로 구현

**배열 (Array)**
- 순서가 있는 데이터 집합
- JavaScript의 기본 자료구조

**배포 (Deploy)**
- 개발한 애플리케이션을 서버에 올려 공개
- CI/CD로 자동화 가능

**백그라운드 (Background)**
- 사용자에게 보이지 않는 곳에서 실행
- 백그라운드 작업, 백그라운드 이미지

**백엔드 (Backend)**
- 서버 측 개발 영역
- 데이터 처리, 비즈니스 로직 담당

**번들러 (Bundler)**
- 여러 파일을 하나로 묶는 도구
- Webpack, Rollup, Parcel 등

**번들링 (Bundling)**
- 여러 모듈을 하나의 파일로 결합
- HTTP 요청 횟수 감소 목적

**벡터 그래픽 (Vector Graphics)**
- 수학적 계산으로 그리는 그래픽
- SVG가 대표적

**변수 (Variable)**
- 값을 저장하는 공간
- let, const, var로 선언

**병합 (Merge)**
- Git에서 브랜치를 합치는 작업
- 충돌 해결이 필요할 수 있음

**보안 (Security)**
- 시스템과 데이터를 보호하는 활동
- XSS, CSRF 등 방어

**복제 (Clone)**
- Git 저장소를 로컬에 복사
- git clone 명령어 사용

**부모/자식 요소 (Parent/Child Element)**
- HTML 요소의 포함 관계
- DOM 트리의 계층 구조

**부트스트랩 (Bootstrap)**
- 인기 있는 CSS 프레임워크
- 반응형 디자인 쉽게 구현

**뷰 (Vue)**
- 프로그레시브 JavaScript 프레임워크
- 쉬운 학습 곡선이 장점

**뷰포트 (Viewport)**
- 브라우저에서 보이는 영역
- 반응형 디자인의 기준

**브라우저 (Browser)**
- 웹페이지를 보여주는 프로그램
- Chrome, Firefox, Safari 등

**브랜치 (Branch)**
- Git의 독립적인 작업 공간
- 기능 개발이나 버그 수정에 사용

**블로킹 (Blocking)**
- 작업이 완료될 때까지 대기하는 방식
- 동기 처리의 특징

**빌드 (Build)**
- 소스코드를 실행 가능한 형태로 변환
- 압축, 번들링, 트랜스파일링 포함

**빌드 도구 (Build Tool)**
- 빌드 과정을 자동화하는 도구
- Webpack, Gulp, Grunt 등

### ㅅ

**사스/SCSS (Sass/SCSS)**
- CSS 전처리기
- 변수, 중첩, 믹스인 등 고급 기능 제공

**사용자 경험 (UX)**
- User Experience
- 사용자가 제품을 사용하며 느끼는 전반적 경험

**사용자 인터페이스 (UI)**
- User Interface
- 사용자와 시스템 간 상호작용 인터페이스

**상속 (Inheritance)**
- CSS에서 부모 요소의 스타일을 자식이 물려받음
- JavaScript에서 프로토타입 체인

**상태 (State)**
- 컴포넌트가 가진 데이터
- React의 핵심 개념

**상태 관리 (State Management)**
- 애플리케이션의 데이터를 중앙에서 관리
- Redux, MobX 등 사용

**샌드박스 (Sandbox)**
- 격리된 실행 환경
- 안전한 테스트 환경

**서버 (Server)**
- 서비스를 제공하는 컴퓨터
- 클라이언트의 요청을 처리하고 응답

**서버리스 (Serverless)**
- 서버 관리 없이 코드만 실행하는 방식
- AWS Lambda, Vercel Functions

**서버사이드 렌더링 (SSR)**
- 서버에서 HTML을 생성하여 전송
- SEO와 초기 로딩 속도 개선

**서브도메인 (Subdomain)**
- 메인 도메인의 하위 도메인
- mail.google.com의 mail 부분

**선택자 (Selector)**
- CSS에서 스타일을 적용할 요소를 지정
- 클래스, ID, 태그 선택자 등

**세션 (Session)**
- 서버가 클라이언트를 식별하는 방법
- 로그인 상태 유지에 사용

**세션 스토리지 (Session Storage)**
- 브라우저 탭이 열려있는 동안만 데이터 저장
- 탭을 닫으면 데이터 삭제

**소켓 (Socket)**
- 네트워크 통신의 끝점
- WebSocket은 실시간 양방향 통신

**소스 맵 (Source Map)**
- 압축된 코드를 원본과 연결
- 디버깅 시 원본 코드 확인 가능

- Git에서 원격 저장소에 업로드
- 배열에 요소 추가

**풀 (Pull)**
- Git에서 원격 저장소에서 가져오기
- Pull Request는 코드 검토 요청

**프라미스 (Promise)**
- 비동기 작업의 결과를 나타내는 객체
- then, catch로 처리

**프레임워크 (Framework)**
- 애플리케이션 개발의 전체 구조 제공
- React, Vue, Angular 등

**프로그레시브 웹 앱 (PWA)**
- 웹과 네이티브 앱의 장점 결합
- 오프라인 작동, 푸시 알림

**프로덕션 (Production)**
- 실제 사용자가 사용하는 환경
- 배포된 최종 버전

**프로세스 (Process)**
- 실행 중인 프로그램
- Node.js의 process 객체

**프로토콜 (Protocol)**
- 통신 규약
- HTTP, HTTPS, FTP 등

**프로토타입 (Prototype)**
- JavaScript의 상속 메커니즘
- 모든 객체가 프로토타입 가짐

**프론트엔드 (Frontend)**
- 사용자가 보고 상호작용하는 부분
- HTML, CSS, JavaScript로 구성

**프록시 (Proxy)**
- 중간에서 요청을 대신 처리
- 리버스 프록시, 프록시 서버

**플렉스박스 (Flexbox)**
- CSS의 1차원 레이아웃 시스템
- 유연한 박스 배치

**플러그인 (Plugin)**
- 기능을 확장하는 추가 프로그램
- Webpack 플러그인, VS Code 확장

### ㅎ

**하드 코딩 (Hard Coding)**
- 값을 코드에 직접 입력
- 유연성이 떨어짐

**하이드레이션 (Hydration)**
- 서버 렌더링된 HTML에 JavaScript 연결
- React SSR의 과정

**함수 (Function)**
- 재사용 가능한 코드 블록
- 입력받아 출력 생성

**함수형 프로그래밍 (Functional Programming)**
- 함수 중심의 프로그래밍 패러다임
- 순수 함수, 불변성 강조

**해시 (Hash)**
- 데이터를 고정 길이 문자열로 변환
- 비밀번호 저장, URL 라우팅

**핸들러 (Handler)**
- 이벤트를 처리하는 함수
- onClick, onSubmit 등

**헤더 (Header)**
- HTTP 요청/응답의 메타데이터
- HTML의 <head> 영역

**호스팅 (Hosting)**
- 웹사이트 파일을 서버에 올려 공개
- 공유 호스팅, VPS, 클라우드 등

**호이스팅 (Hoisting)**
- 선언을 스코프 최상단으로 끌어올림
- JavaScript의 특징

**환경 변수 (Environment Variable)**
- 시스템이나 앱 설정을 저장하는 변수
- .env 파일로 관리

**훅 (Hook)**
- React의 상태 관리 기능
- useState, useEffect 등

### Git 명령어
```bash

# 변경사항 관리
git status                 # 상태 확인
git add .                  # 모든 변경사항 스테이징

git commit -m "msg"        # 커밋
git commit --amend        # 마지막 커밋 수정

# 브랜치 관리
git branch                 # 브랜치 목록
git branch [name]         # 새 브랜치 생성
git checkout [branch]     # 브랜치 전환
git checkout -b [branch]  # 브랜치 생성 후 전환
git merge [branch]        # 브랜치 병합
git branch -d [branch]    # 브랜치 삭제
git branch -D [branch]    # 브랜치 강제 삭제

# 원격 저장소
git push                   # 원격 저장소에 푸시
git push -u origin [branch] # 브랜치를 원격에 처음 푸시
git pull                   # 원격 저장소에서 가져오기
git fetch                  # 원격 저장소 정보만 가져오기

# 되돌리기
git reset HEAD~1          # 마지막 커밋 취소
git reset --hard HEAD~1   # 마지막 커밋 완전 제거
git revert [commit]       # 특정 커밋 되돌리기
git stash                 # 작업 임시 저장
git stash pop            # 임시 저장 복원
git stash list           # 스태시 목록

# 로그 확인
git log                   # 커밋 이력
git log --oneline        # 간단한 로그
git log --graph          # 그래프로 표시
git diff                 # 변경사항 확인
git diff [branch1] [branch2] # 브랜치 비교
```
