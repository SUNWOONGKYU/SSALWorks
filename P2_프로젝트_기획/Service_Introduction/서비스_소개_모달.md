# [요약본] SSAL Works 한눈에 보기

## SSAL Works란?

**Claude Code를 활용한 3차원 SAL Grid 기반 풀스택 웹사이트 개발 플랫폼**

비개발자도 AI와 함께 풀스택 웹사이트를 만들 수 있는 곳입니다.

```
┌─────────────────────────────────────────────────────────┐
│                    SSAL Works                           │
│                  "황금쌀 방앗간"                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   [창의적 아이디어] ──→ [AI가 코딩] ──→ [풀스택 웹사이트]  │
│        (당신)           (Claude Code)    (수익 창출)     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

## 이런 분들을 위한 플랫폼입니다

SSAL Works는 아래 **3가지 조건**을 갖춘 분들을 위한 플랫폼입니다.

| # | 조건 | 설명 |
|---|------|------|
| 1 | **사업 아이디어** | 풀스택 웹사이트로 구현할 사업 아이디어가 있을 것 |
| 2 | **성공 염원** | 이 사업이 잘 되기를 바라는 염원이 있을 것 |
| 3 | **실행 의지** | 이걸 꼭 해보려는 의지가 있을 것 |

# 섹션 1: SSAL Works란 무엇인가

## 1-1. SSAL Works의 정체성

SSAL Works는 **Claude Code를 활용한 3차원 SAL Grid 기반 풀스택 웹사이트 개발 실행 및 관리 플랫폼**입니다.

이 정의를 하나씩 풀어서 설명하겠습니다.

### Claude Code

Claude Code는 Anthropic이 개발한 AI 도구입니다. 터미널에서 실행되며, 사용자의 지시를 받아 코드를 작성하고, 파일을 만들고, 프로젝트를 관리합니다.

**Claude Code의 역할:**
- 코드 작성 (HTML, CSS, JavaScript, TypeScript 등)
- 파일 생성 및 수정
- Git 버전 관리
- 터미널 명령 실행
- 프로젝트 구조 관리

SSAL Works는 이 Claude Code를 핵심 도구로 사용하여, 비개발자도 풀스택 웹사이트를 개발할 수 있게 합니다.

### 3차원 SAL Grid

SAL Grid는 SSAL Works가 개발한 프로젝트 관리 방법론입니다.

**SAL = Stage + Area + Level**

```
        Area (Y축: 11개 개발 영역)
          ↑
          │    ┌───┬───┬───┬───┬───┐
          │    │   │   │   │   │   │
          │    ├───┼───┼───┼───┼───┤
          │    │   │ Task │   │   │  ← Level (Z축)
          │    ├───┼───┼───┼───┼───┤
          │    │   │   │   │   │   │
          │    └───┴───┴───┴───┴───┘
          └──────────────────────────→ Stage (X축: 5개 개발 단계)
```

**Stage (개발 단계):**
| Stage | 이름 | 내용 |
|-------|------|------|
| S1 | 개발 준비 | 환경설정, DB 스키마, Auth 설정 |
| S2 | 개발 1차 | OAuth, 이메일, 회원가입 |
| S3 | 개발 2차 | AI 연동, AI Q&A |
| S4 | 개발 3차 | 결제, 관리자, 크레딧 |
| S5 | 개발 마무리 | 배포, QA, 안정화 |

**Area (개발 영역):**
| Area | 이름 | 내용 |
|------|------|------|
| M | Documentation | 문서화 |
| U | Design | UI/UX 디자인 |
| F | Frontend | 프론트엔드 |
| BI | Backend Infra | 백엔드 기반 |
| BA | Backend APIs | 백엔드 API |
| D | Database | 데이터베이스 |
| S | Security | 보안/인증 |
| T | Testing | 테스트 |
| O | DevOps | 운영/배포 |
| E | External | 외부 연동 |
| C | Content | 콘텐츠 시스템 |

**Level (Task 순서):**
같은 Stage, 같은 Area 안에서 여러 Task가 있을 때 그 순서를 나타냅니다.

**왜 SAL Grid가 필요한가?**

AI는 체계 없이 방치하면 여기저기에 폴더와 파일을 만듭니다. 관리 시스템 없이는 프로젝트가 엉망이 됩니다.

SAL Grid는 모든 개발 작업(Task)을 3차원 좌표계 위에 배치하여 체계적으로 관리합니다. 각 Task는 22개의 속성으로 관리되며, 작업자와 검증자가 분리되어 품질을 보장합니다.

### 풀스택 웹사이트

프론트엔드(사용자가 보는 화면)와 백엔드(서버, 데이터베이스, API)가 모두 갖춰진 완전한 웹사이트입니다.

단순한 랜딩 페이지가 아니라, 회원가입, 로그인, 결제, 데이터 저장, 관리자 기능 등 실제 비즈니스를 운영할 수 있는 웹사이트입니다.

---

## 1-2. SSAL이라는 이름의 의미

SSAL이라는 이름은 두 가지 의미가 결합되어 있습니다.

**첫째, S + SAL**

S는 개발자 Sunny의 이름 첫 글자이고, SAL은 3차원 좌표계(Stage, Area, Level)의 약자입니다.

**둘째, 쌀**

SSAL의 한글 발음은 "쌀"입니다. 쌀은 한국인에게 핵심 먹을거리입니다. SSAL Works에서 만드는 웹사이트는 사용자에게 돈을 벌어다주는 핵심 먹을거리가 됩니다.

---

## 1-3. 황금쌀 방앗간

SSAL Works를 비유적으로 표현하면 **"황금쌀 방앗간"**입니다.

| 방앗간 | SSAL Works |
|--------|------------|
| 벼 (재료) | 창의적인 아이디어 |
| 방아 (반복 노동) | AI (Claude Code) |
| 쌀 (결과물) | 풀스택 웹사이트 |
| 황금쌀 | 돈을 버는 웹사이트 |

방앗간에서 농부가 직접 방아를 찧지 않아도 쌀이 나오듯이, SSAL Works에서 사용자가 직접 코딩을 하지 않아도 웹사이트가 만들어집니다.

---

## 1-4. 인간과 AI의 역할 분담

**AI(Claude Code)가 담당하는 것:**
- 코딩 (HTML, CSS, JavaScript 등)
- 문법과 문서 형식
- 도구 사용 (Git, npm, Vercel 등)
- 반복적인 작업
- 기술적인 구현

**인간(빌더)이 담당하는 것:**
- 무엇을 만들 것인지 결정 (기획)
- 어떤 방향으로 갈 것인지 결정 (의사결정)
- 비즈니스 모델 설계
- 최종 승인

# 섹션 2: 왜 SSAL Works를 만들었는가

## 2-1. Sunny의 10년간 3번의 시도 (2013-2022)

SSAL Works의 개발자 Sunny는 2013년부터 2022년까지 10년간 3개의 IT 플랫폼 개발을 시도했습니다.

| # | 프로젝트 | 내용 |
|---|----------|------|
| 1 | 기업 지원 서비스 거래 플랫폼 | 기업들이 필요한 서비스를 거래하는 플랫폼 |
| 2 | 온라인 엑셀러레이터 플랫폼 | 스타트업 온라인 멘토링 플랫폼 |
| 3 | 스타트업 지식 제공 플랫폼 | 스타트업 지식/정보 제공 플랫폼 |

세 가지 모두 미완으로 끝났습니다. 아이디어는 있었지만, 웹사이트로 구현하지 못했습니다.

원인은 매번 같았습니다:
- 개발자를 구하기 어려움
- 개발 비용이 너무 높음
- 의사소통에 한계
- 원하는 것을 정확히 구현하기 어려움

매번 같은 생각이 들었습니다. **"내가 직접 개발할 수 있다면..."**

---

## 2-2. Sunny의 배경

Sunny는 공인회계사입니다.

경영학과를 졸업했고, 공인회계사 시험 경영학 과목에서 수석을 했습니다. 만 19세 이전에 18개월 만에 공인회계사에 합격했습니다.

합격 후에는 회계감사 업무를 담당했습니다. 회계감사는 여러 가지 자료를 체계적으로 분류하고, 정리하고, 평가하는 업무입니다.

이 경험이 SAL Grid 방법론을 만드는 데 결정적인 역할을 했습니다.

| 회계감사 | SAL Grid |
|----------|----------|
| 감사 계획서 | Order Sheet |
| 감사 절차서 | Task Instruction |
| 내부통제 테스트 | Verification Instruction |
| Working Paper | Task Results |
| Review Process | Stage Gate 검증 |
| 파트너 승인 | PO 승인 |

또한 Sunny는 10년 이상 스타트업 엑셀러레이터로 활동했습니다. 수많은 스타트업들이 같은 진입장벽에 막히는 것을 목격했습니다.

---

## 2-3. AI와의 만남 (2025년 5월)

2025년 5월, Sunny는 AI를 발견했습니다.

직접 해보니까 놀라운 사실을 깨달았습니다.

개발팀이 필요 없었습니다. 돈도 필요 없었습니다. 혼자서 웹사이트를 만들 수 있었습니다.

10년간 갈망했던 것의 해결 실마리를 발견한 순간이었습니다.

---

## 2-4. 7개월간의 몰입 (2025년 5월 ~ 12월)

2025년 5월부터 12월까지 7개월간, Sunny는 하루 평균 12시간 이상을 AI와 함께 개발에 투입했습니다.

**총 2,500시간 이상, 수천만원의 토큰 비용 투입**

공인회계사 합격 이후 처음으로 그 수준의 몰입이었습니다.

2,500시간 안에는 많은 것들이 담겨 있습니다:
- 수많은 공부 (웹개발 기초부터 고급 기술까지)
- 수많은 AI 사용 경험 (Claude, ChatGPT, Gemini, Perplexity 등)
- 수많은 좌절과 극복
- 수많은 시행착오와 노하우 축적

---

## 2-5. 3개월 만에 만든 웹사이트

7개월간 이 모든 일을 하면서, 웹사이트 개발은 10월부터 시작했습니다. 3개월도 안 되는 기간에 풀스택 웹사이트를 완성했습니다.

10년간 원하는 결과를 만들지 못했던 것을, 3개월 만에 해냈습니다. 혼자서. AI와 함께.

이제 Sunny는 풀스택 웹사이트를 일주일에 하나씩 만들 수 있는 능력이 생겼습니다.

이 능력을 다른 사람들에게 공유해서, 많은 사람들이 같은 능력을 갖게 되기를 바라는 마음이 간절합니다.

---

## 2-6. AI에 대한 현실적 인식

**체계 없는 코딩은 시행착오가 많습니다.**

개발 기술을 조금 공부해서 AI에게 시키면 된다고요? 비개발자가 체계 없이 시도하면 토큰 비용만 낭비됩니다.

**AI도 관리 시스템이 필요합니다.**

AI가 아무리 뛰어나도 관리/통제/검증 시스템이 준비되어 있지 않으면, 파일과 폴더가 체계 없이 널부러집니다. 수정 작업도 매우 어렵습니다.

필요한 것은 기술이 아니라 **방법론**입니다.

개발 기술을 몰라도 웹사이트를 만들 수 있는 방법론. 그 방법론을 만들어서 제공하려는 것이 SSAL Works입니다.

**별도 학습 없이 바로 시작할 수 있습니다.**

웹사이트를 만들고 싶은 의지가 있다면, 공부하지 말고 바로 시작하세요. 개발하는 과정에서 필요한 지식은 SSAL Works에 다 정리해놓았습니다.

# 섹션 3: SSAL Works가 제공하는 것

## 3-1. PROJECT SAL Grid 방법론

3차원 좌표계(Stage, Area, Level)로 프로젝트 전체를 관리합니다.

**핵심 특징:**
- 각 Task는 22개의 속성으로 관리
- 작업자(Task Agent)와 검증자(Verification Agent) 분리
- 모든 작업의 추적 가능 (Audit Trail)
- 단계별 Stage Gate 승인
- 철저한 문서화

회계감사 경험이 녹아있는 방법론으로, 비개발자가 만들었지만 개발자들이 놓치기 쉬운 관리/통제/검증 측면이 탄탄합니다.

---

## 3-2. Order Sheet 체계

AI에게 체계적으로 작업을 지시하는 표준화된 양식입니다.

**Order Sheet가 무엇인가:**
- AI에게 "이거 해줘"라고만 하면 엉터리가 됩니다
- Order Sheet는 AI가 지켜야 할 규칙, 작업 내용, 기대 결과물을 명확하게 정의합니다

**Order Sheet 구성:**
- Part A: 모든 프로젝트에 공통으로 적용되는 표준 규칙
- Part B: 프로젝트별로 커스터마이징하는 부분

**활용법:**
1. 대시보드에서 Order Sheet 템플릿 선택
2. 프로젝트에 맞게 Part B 작성
3. "복사하기" 버튼으로 클립보드에 복사
4. Claude Code 터미널에 붙여넣기
5. AI가 Order Sheet를 읽고 작업 수행

---

## 3-3. Briefing (안내문) 체계

각 개발 단계마다 무엇을 해야 하는지 상세히 안내합니다.

**Briefing이 무엇인가:**
- 각 단계에서 해야 할 일을 명확하게 안내하는 문서
- "뭘 해야 할지 몰라서 멍하니 있는 시간"을 없앱니다

**Briefing 구성:**
- 회원가입 후 환영 메시지 및 다음 단계 안내
- 프로젝트 등록 후 플랫폼 사용법 안내
- 각 Stage별 안내문 (P0~S5)
- 상황별 안내문 (완료된 프로젝트 수정 등)

**활용법:**
1. 현재 단계의 Briefing 확인
2. 안내에 따라 작업 진행
3. 해당 단계의 Order Sheet 템플릿 활용
4. 다음 단계로 이동

## 3-5. 외부 연동 설정 가이드

| 3 | **Resend** | 이메일 발송 |
| 4 | **Vercel** | 배포 + 도메인 연결 |
| 5 | **TossPayments** | 결제 시스템 |

이 가이드를 따라하면 작업 시간이 획기적으로 줄어듭니다.
