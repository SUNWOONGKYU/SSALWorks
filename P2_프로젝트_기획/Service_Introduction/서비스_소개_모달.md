# [개요] SSAL Works 소개

## SSAL Works가 뭔가요?

**"코딩 못해도 AI랑 같이 웹사이트 만들 수 있는 곳"**

SSAL Works는 개발자가 아닌 분들이 AI의 도움을 받아 직접 웹사이트를 만들 수 있도록 돕는 플랫폼입니다.

보통 웹사이트를 만들려면 개발자를 고용하거나 외주를 맡겨야 합니다. 비용도 많이 들고, 원하는 대로 수정하기도 어렵습니다.

SSAL Works에서는 **당신이 아이디어를 말하면, AI가 코드를 작성합니다.** 당신은 개발을 배울 필요 없이, AI에게 "이렇게 해줘"라고 지시만 하면 됩니다.

---

## 누구를 위한 플랫폼인가요?

SSAL Works는 아래 세 가지에 해당하는 분들을 위한 플랫폼입니다.

**첫째, 웹사이트로 만들고 싶은 사업 아이디어가 있는 분**

온라인 강의 플랫폼, 예약 시스템, 커뮤니티 사이트, 쇼핑몰 등... 머릿속에 "이런 웹사이트가 있으면 좋겠다"는 아이디어가 있으신 분이라면 SSAL Works가 도움이 됩니다.

**둘째, 그 사업이 잘 되길 바라는 분**

단순히 "한번 만들어볼까?"가 아니라, 진지하게 사업으로 키워나가고 싶은 분께 적합합니다.

**셋째, 직접 해보려는 의지가 있는 분**

AI가 코딩을 대신해주지만, 프로젝트 방향 결정과 AI 지시는 당신이 해야 합니다. "남이 다 해주겠지"가 아니라 "내가 직접 하겠다"는 마음가짐이 필요합니다.

---

## 무엇을 얻을 수 있나요?

**1. 혼자서도 웹사이트를 만들 수 있습니다**

개발팀을 꾸리거나 외주를 맡기지 않아도 됩니다. AI와 함께라면 혼자서도 로그인, 결제, 데이터베이스가 있는 진짜 웹사이트를 만들 수 있습니다.

**2. 2,500시간의 시행착오를 건너뜁니다**

SSAL Works 창업자 Sunny가 직접 겪은 시행착오와 노하우가 모두 정리되어 있습니다. 같은 실수를 반복하지 않고 바로 시작할 수 있습니다.

**3. 3개월간 1:1 코칭을 받습니다**

빌더 계정을 개설하면 3개월간 Sunny의 맞춤형 코칭을 받습니다. 개발뿐 아니라 창업 전반에 대한 멘토링도 포함됩니다.

**4. 풍부한 학습 콘텐츠가 제공됩니다**

AI와 협업하는 방법을 담은 Books 80편, 실전 팁 64개가 준비되어 있습니다. 차근차근 따라하면 됩니다.

**5. 실제 작동하는 예시 프로젝트를 받습니다**

SSAL Works 플랫폼 자체의 전체 소스코드를 제공합니다. 이걸 참고해서 내 웹사이트를 만들면 됩니다.

**6. AI에게 지시하는 양식(Order Sheet)이 있습니다**

AI에게 뭘 어떻게 시켜야 할지 모르겠다면, 미리 만들어진 32개의 표준 양식을 사용하면 됩니다.

**7. 막히면 언제든 물어볼 수 있습니다**

24시간 Sunny에게 직접 질문할 수 있습니다. ChatGPT, Gemini 등 AI Q&A 기능도 이용할 수 있습니다.

## 비용은 얼마인가요?

**빌더 계정 개설비: 300만원 (얼리버드)**

이 금액에는 플랫폼 평생 이용권, AI 크레딧 5만원, 3개월간 1:1 코칭이 포함됩니다.

**50% 환불 조건이 있습니다.** 3개월 내에 아래 조건을 모두 달성하면 150만원을 돌려받습니다:

- **완성 기준**: 로그인/결제 기능 작동 + 실제 도메인 배포
- **고객 기준**: 실제 가입자 30명 이상 (Supabase auth.users 테이블 증빙)
- **신청 기한**: 빌더 계정 개설일로부터 90일 이내

실질적으로 150만원에 시작하는 셈입니다.

**월 이용료는 5만원**이며, 처음 3개월은 무료입니다. 4개월차부터 월 5만원이 발생합니다. 서버 유지, 기능 업데이트, 지속적인 지원이 포함됩니다.

**AI 크레딧**은 AI를 사용할 때마다 차감됩니다. 초기 5만원이 제공되고, 부족하면 1만원 단위로 충전합니다.

---

## 왜 지금 시작하면 좋은가요?

**지금은 얼리버드 가격입니다.** 현재 300만원인 빌더 계정 개설비는 앞으로 단계적으로 인상될 예정입니다.

**예시 프로젝트가 늘어날수록 가치가 올라갑니다.** 지금은 예시 프로젝트가 1개(SSAL Works)이지만, 50개, 100개로 늘어나면 웹사이트 제작 기간이 3개월에서 1주일로 단축됩니다.

**빌더 계정은 양도할 수 있습니다.** 나중에 사용하지 않게 되면 타인에게 시가로 양도할 수 있습니다.

---

## 어떻게 시작하나요?

**1단계: 회원가입 (무료)** - 먼저 회원가입을 합니다. 무료이고, 부담 없이 둘러볼 수 있습니다.

**2단계: 콘텐츠 확인** - Books, Tips, Order Sheet 등을 둘러보면서 "이게 나한테 맞겠다" 싶은지 판단합니다.

**3단계: 빌더 계정 개설** - 유용하다고 판단되면 빌더 계정을 개설합니다. 결제 후 바로 모든 기능을 이용할 수 있습니다.

**4단계: 프로젝트 시작** - 내 프로젝트를 등록하고, AI와 함께 개발을 시작합니다.

**고객센터 문의하기**로 일반적인 문의사항을, **Sunny에게 질문하기**로 개발, 코칭, 사업 관련 상담을 할 수 있습니다. 24시간 언제든 문의 가능하며, 보통 당일 내 답변드립니다.

# 1. SSAL Works란 무엇인가

---

## 1-1. SSAL Works의 정체성

SSAL Works는 **Claude Code를 활용한 3차원 SAL Grid 기반 풀스택 웹사이트 개발 실행 및 관리 플랫폼**입니다.

이 정의를 하나씩 풀어서 설명하겠습니다.

---

### Claude Code란?

Claude Code는 Anthropic이 개발한 AI 도구입니다. 터미널에서 실행되며, 사용자의 지시를 받아 코드를 작성하고, 파일을 만들고, 프로젝트를 관리합니다.

**Claude Code의 역할:**

| 역할 | 설명 |
|------|------|
| 코드 작성 | HTML, CSS, JavaScript, TypeScript, Python 등 |
| 파일 생성 및 수정 | 프로젝트 파일 자동 생성 |
| Git 버전 관리 | 커밋, 푸시, 브랜치 관리 |
| 터미널 명령 실행 | npm, git, 배포 명령 등 |
| 프로젝트 구조 관리 | 폴더 생성, 파일 정리 |
| 코드 리뷰 | 작성한 코드 검토 |
| 버그 수정 | 에러 분석 및 해결 |
| 테스트 실행 | 자동화 테스트 수행 |

**Claude Code vs 일반 Claude:**

| 비교 항목 | Claude (웹/앱) | Claude Code (터미널) |
|----------|---------------|---------------------|
| 실행 환경 | 브라우저/앱 | 터미널 (CLI) |
| 파일 접근 | 불가 | 로컬 파일 직접 접근 |
| 코드 실행 | 불가 | 직접 실행 가능 |
| Git 작업 | 불가 | 직접 수행 |
| 배포 | 불가 | 직접 수행 |
| 용도 | 대화, 질문, 문서 작성 | 실제 개발 작업 |

SSAL Works는 이 Claude Code를 핵심 도구로 사용하여, 비개발자도 풀스택 웹사이트를 개발할 수 있게 합니다.

---

### 3차원 SAL Grid란?

SAL Grid는 SSAL Works가 개발한 프로젝트 관리 방법론입니다.

**SAL = Stage + Area + Level**

```
              Area (Y축: 11개 개발 영역)
                ↑
                │    ┌───┬───┬───┬───┬───┐
                │    │   │   │   │   │   │
                │    ├───┼───┼───┼───┼───┤
                │    │   │ Task │   │   │  ← Level (Z축)
                │    ├───┼───┼───┼───┼───┤
                │    │   │   │   │   │   │
                │    └───┴───┴───┴───┴───┘
                └──────────────────────────→ Stage (X축: 5개 개발 단계)
```

**Stage (X축 - 개발 단계):**

| Stage | 이름 | 한글명 | 주요 내용 |
|-------|------|--------|----------|
| S1 | Development Setup | 개발 준비 | 환경설정, DB 스키마, Auth 설정 |
| S2 | Auth & Registration | 개발 1차 | OAuth, 이메일, 회원가입 |
| S3 | AI Integration | 개발 2차 | AI 연동, AI Q&A |
| S4 | Payment & Admin | 개발 3차 | 결제, 관리자, 크레딧 |
| S5 | Development Stabilization | 개발 마무리 | 배포, QA, 안정화 |

**Area (Y축 - 개발 영역):**

| Area | 코드 | 한글명 | 설명 |
|------|------|--------|------|
| Documentation | M | 문서화 | 기획서, 명세서 등 |
| Design | U | UI/UX 디자인 | 화면 설계, 디자인 |
| Frontend | F | 프론트엔드 | 사용자가 보는 화면 |
| Backend Infrastructure | BI | 백엔드 기반 | 서버 설정, 라이브러리 |
| Backend APIs | BA | 백엔드 API | API 엔드포인트 |
| Database | D | 데이터베이스 | DB 스키마, 쿼리 |
| Security | S | 보안/인증 | 로그인, 권한 관리 |
| Testing | T | 테스트 | 단위/통합 테스트 |
| DevOps | O | 운영/배포 | 배포, CI/CD |
| External | E | 외부 연동 | 외부 서비스 연동 |
| Content System | C | 콘텐츠 시스템 | 콘텐츠 관리 |

**Level (Z축 - Task 순서):**

같은 Stage, 같은 Area 안에서 여러 Task가 있을 때 그 순서를 나타냅니다.

예: S2F1, S2F2, S2F3 → Stage 2, Frontend Area의 1번, 2번, 3번 Task

---

### Task의 22개 속성

SAL Grid에서 각 Task는 22개의 속성으로 관리됩니다.

| # | 속성명 | 설명 |
|---|--------|------|

| 2 | task_name | Task 이름 |

| 6 | task_status | 작업 상태 (Pending → In Progress → Executed → Completed) |
| 7 | task_progress | 진행률 (0~100) |
| 8 | dependencies | 선행 Task |
| 9 | task_instruction | Task 수행 지침 |
| 10 | task_agent | Task 수행 Agent |
| 11 | generated_files | 생성된 파일 |
| 12 | duration | 소요 시간 |
| 13 | build_result | 빌드 결과 |
| 14 | verification_instruction | 검증 지침 |
| 15 | verification_agent | 검증 Agent |
| 16 | test_result | 테스트 결과 |
| 17 | build_verification | 빌드 검증 |
| 18 | integration_verification | 통합 검증 |
| 19 | blockers | 차단 요소 |
| 20 | comprehensive_verification | 종합 검증 |
| 21 | ai_verification_note | AI 검증 의견 |
| 22 | stage_gate_status | Stage Gate 상태 |

---

### 왜 SAL Grid가 필요한가?

**체계 없는 코딩은 시행착오가 많습니다.**

AI가 아무리 뛰어나도 관리 시스템 없이 방치하면:
- 여기저기에 폴더와 파일을 만듭니다
- 일관성 없이 코드를 작성합니다
- 수정 작업이 매우 어려워집니다
- 어떤 파일이 어디에 있는지 찾기 힘듭니다

**AI도 관리 시스템이 필요합니다.**

SAL Grid는 모든 개발 작업(Task)을 3차원 좌표계 위에 배치하여:
- 체계적으로 관리합니다
- 모든 작업을 추적합니다 (Audit Trail)
- 작업자와 검증자를 분리합니다
- 단계별 품질을 보장합니다

---

### 풀스택 웹사이트란?

프론트엔드(사용자가 보는 화면)와 백엔드(서버, 데이터베이스, API)가 모두 갖춰진 완전한 웹사이트입니다.

**풀스택 웹사이트의 구성:**

```
┌─────────────────────────────────────────────────────────┐
│                    풀스택 웹사이트                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌──────────────────────────────────────┐              │
│   │          Frontend (프론트엔드)         │              │
│   │   - HTML (구조)                       │              │
│   │   - CSS (스타일)                      │              │
│   │   - JavaScript (동작)                 │              │
│   │   - 사용자가 보고 상호작용하는 화면     │              │
│   └──────────────────────────────────────┘              │
│                         ↓↑                              │
│   ┌──────────────────────────────────────┐              │
│   │          Backend (백엔드)              │              │
│   │   - API 서버                          │              │
│   │   - 비즈니스 로직                      │              │
│   │   - 인증/권한 관리                     │              │
│   └──────────────────────────────────────┘              │
│                         ↓↑                              │
│   ┌──────────────────────────────────────┐              │
│   │          Database (데이터베이스)        │              │
│   │   - 데이터 저장                        │              │
│   │   - 데이터 조회                        │              │
│   └──────────────────────────────────────┘              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

단순한 랜딩 페이지가 아니라, 회원가입, 로그인, 결제, 데이터 저장, 관리자 기능 등 실제 비즈니스를 운영할 수 있는 웹사이트입니다.

---

## 1-2. SSAL이라는 이름의 의미

SSAL이라는 이름은 두 가지 의미가 결합되어 있습니다.

### 첫째, S + SAL

| 구성 요소 | 의미 |
|----------|------|
| S | 개발자 Sunny의 이름 첫 글자 |
| SAL | 3차원 좌표계 (Stage, Area, Level)의 약자 |

따라서 SSAL = Sunny + Stage + Area + Level

---

### 둘째, 쌀

SSAL의 한글 발음은 "쌀"입니다.

| 쌀의 의미 | SSAL Works의 의미 |
|----------|-------------------|
| 쌀은 한국인의 핵심 먹을거리 | 웹사이트는 사업의 핵심 먹을거리 |
| 쌀이 있어야 밥을 먹는다 | 웹사이트가 있어야 수익을 낸다 |
| 쌀은 기본 중의 기본 | 풀스택 웹사이트는 IT 사업의 기본 |

SSAL Works에서 만드는 웹사이트는 사용자에게 돈을 벌어다주는 핵심 먹을거리가 됩니다.

## 1-3. 황금쌀 방앗간

- 한 곳을 고치면 다른 곳이 망가져서 수정이 불가능해짐
- 무엇이 왜 그렇게 되었는지 추적이 불가능해짐

# 3. SSAL Works가 제공하는 것

## 3-1. PROJECT SAL Grid 방법론

3차원 좌표계(Stage, Area, Level)로 프로젝트 전체를 관리합니다.

### SAL Grid의 핵심 특징

- 각 Task를 22개 속성으로 체계적으로 관리
- Task Agent(작업자)와 Verification Agent(검증자) 분리
- 모든 작업의 Audit Trail 보장으로 추적 가능
- 단계별 Stage Gate 승인 절차
- 철저한 문서화 및 기록

### SAL Grid View 옵션

2D Card View(카드 형태로 Task 목록 표시)와 3D Block View(3차원 블록 형태로 시각화) 중 선택할 수 있습니다.

### Task 상태 전이

```
┌─────────┐    ┌─────────────┐    ┌──────────┐    ┌───────────┐
│ Pending │ →  │ In Progress │ →  │ Executed │ →  │ Completed │
└─────────┘    └─────────────┘    └──────────┘    └───────────┘
                                       ↑              ↑
                                   작업 완료      검증 후만 가능
```

Pending(대기) → In Progress(진행) → Executed(완료, 검증 전) → Completed(검증까지 완료) 순서로 진행됩니다.

---

### 검증 상태 전이

```
┌──────────────┐    ┌───────────┐    ┌──────────┐
│ Not Verified │ →  │ In Review │ →  │ Verified │
└──────────────┘    └───────────┘    └──────────┘
                                           │
                                           ↓
                                      Needs Fix (문제 발견 시)
```

Not Verified(검증 전) → In Review(검증 중) → Verified(통과) 또는 Needs Fix(수정 필요)로 진행됩니다.

---

### Stage Gate 검증

각 Stage가 완료될 때마다 Stage Gate 검증을 수행합니다. S1은 개발 환경, S2는 인증/회원가입, S3는 AI 연동, S4는 결제/관리자, S5는 전체 QA 및 배포 준비를 확인합니다.

---

## 3-2. Order Sheet 체계

AI에게 체계적으로 작업을 지시하는 표준화된 양식입니다.

### Order Sheet가 무엇인가

AI에게 "이거 해줘"라고만 하면 엉터리가 됩니다.

Order Sheet는:
- AI가 지켜야 할 규칙을 명확하게 정의합니다
- 작업 내용을 구체적으로 기술합니다
- 기대 결과물을 명시합니다

---

### Order Sheet 구성

**Part A**는 모든 프로젝트에 공통으로 적용되는 표준 규칙입니다. 7대 작업 규칙(파일명, 저장위치, Area/Stage 등), 5대 절대 규칙, 상태 전이 규칙, Supabase CRUD 방법, 6단계 실행 프로세스가 포함됩니다.

**Part B**는 프로젝트별로 커스터마이징하는 부분입니다. 프로젝트 정보, 작업 지시, 참고 파일, 기대 결과물을 작성합니다.

---

### Order Sheet 활용법

```
┌─────────────────────────────────────────────────────────┐
│                Order Sheet 사용 프로세스                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   1. 대시보드에서 Order Sheet 템플릿 선택                 │
│        ↓                                                │
│   2. 프로젝트에 맞게 Part B 작성                         │
│        ↓                                                │
│   3. "복사하기" 버튼으로 클립보드에 복사                  │
│        ↓                                                │
│   4. Claude Code 터미널에 붙여넣기                       │
│        ↓                                                │
│   5. AI가 Order Sheet를 읽고 작업 수행                   │
│        ↓                                                │
│   6. 결과 확인 및 검증                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

### Order Sheet 템플릿 (32개)

총 32개의 템플릿이 준비되어 있습니다. P0~P3 기획 단계 19개, S0~S5 개발 단계 12개, 완료된 프로젝트 수정용 1개입니다.

## 3-3. Briefing (안내문) 체계

## 3-4. SSAL Works Dev Package

AI 협업 웹 개발을 위한 올인원 통합 패키지입니다.

### Dev Package란?

Claude Code(AI)와 함께 웹 프로젝트를 처음부터 끝까지 수행할 수 있도록 설계된 **통합 개발 환경 패키지**입니다.

AI 협업 웹 개발을 위한 표준화된 프로젝트 템플릿으로, 비개발자도 AI와 함께 웹 서비스를 만들 수 있게 합니다. 프로세스, 자동화, AI 설정이 하나로 통합되어 있습니다.

### 한 문장 정의

> **"다운로드 → 압축 해제 → Claude Code 실행" 3단계로 프로젝트 개발 환경이 완성되는 패키지**

### Dev Package 구성 요소

**AI 도구 확장:** commands/(커스텀 슬래시 명령어), subagents/(전문 Agent 호출), skills/(PDF, Playwright 등 특수 기능)

**AI 작업 규칙:** rules/(7대 작업 규칙), methods/(Supabase CRUD 등 실행 방법), compliance/(12대 AI 행동 규칙)

**작업 기록:** work_logs/(세션별 작업 내역 기록)

#### 3) 자동화 스크립트 (scripts/)

**자동화 워크플로우:**

```
       ↓
배포용 Root 폴더로 자동 복사
```

## 3-6. 외부 연동 설정 가이드

외부 서비스 연동은 AI가 할 수 없습니다. 사람이 직접 해야 하는데, 굉장히 어렵고 복잡합니다.

SSAL Works는 이미 성공한 외부 연동 설정 가이드 5가지를 제공합니다.

### 5가지 외부 연동 가이드

| # | 서비스 | 용도 | 난이도 |
|---|--------|------|:------:|
| 1 | **Supabase** | 데이터베이스 + 인증 기반 | ★★★ |
| 2 | **Google OAuth** | 소셜 로그인 | ★★★★ |
| 3 | **Resend** | 이메일 발송 | ★★ |
| 4 | **Vercel** | 배포 + 도메인 연결 | ★★★ |
| 5 | **TossPayments** | 결제 시스템 | ★★★★★ |

이 가이드를 따라하면 작업 시간이 획기적으로 줄어듭니다.

---

### 각 서비스 설명

**Supabase**: 데이터베이스, 인증, 스토리지, API를 제공하는 Firebase의 오픈소스 대안입니다. 무료 티어가 충분하고 PostgreSQL 기반입니다.

**Google OAuth**: 가장 많이 사용되는 소셜 로그인으로, 사용자 편의성을 향상시킵니다.

**Resend**: 개발자 친화적 이메일 서비스로, 간단한 API와 합리적 가격이 장점입니다.

**Vercel**: 배포와 도메인 연결을 담당하며, Git 연동 자동 배포와 Serverless Functions를 지원합니다.

**TossPayments**: 한국 결제 환경에 최적화된 국내 결제 서비스입니다.

## 4-4. PROJECT SAL GRID

### 기능 설명

2D Card View와 3D Block View 중 선택할 수 있으며, Task의 상태, 진행률, 결과물을 관리합니다. Task 검증과 Stage Gate 검증 시스템이 포함되어 있습니다.

### 22개 속성 관리

모든 Task는 22개 속성으로 관리됩니다:
- 기본 정보 (task_id, task_name, stage, area, level)
- 작업 정보 (status, progress, dependencies, instruction, agent)
- 결과 정보 (generated_files, duration, build_result)
- 검증 정보 (verification_*, test_result, blockers)

# 5. 가격 정책 상세

---

## 5-1. 빌더 계정 개설비

### 현재 가격

**300만원 (얼리버드)**, 최초 1회 납부입니다.

### 포함 내역

플랫폼 모든 기능 이용권, AI 크레딧 5만원, Sunny 1:1 코칭 3개월, Books/Tips/가이드 전체 접근, Dev Package 다운로드 권한, 예시 프로젝트 소스코드 접근 권한이 포함됩니다.

---

### 50% 환불 인센티브

3개월 내에 다음 조건을 모두 달성하면 150만원을 환불합니다:

- **완성 기준**: 로그인/결제 기능 작동 + 실제 도메인 배포
- **고객 기준**: 실제 가입자 30명 이상 (Supabase auth.users 테이블 증빙)
- **신청 기한**: 빌더 계정 개설일로부터 90일 이내

### 가치 상승 예고

빌더 계정의 가치는 지속적으로 상승할 예정입니다. 현재 얼리버드 300만원에서 단계적으로 인상될 예정입니다.

**왜 가치가 상승하는가?**

예시 프로젝트 100개가 제공되면, 1주일 안에 풀스택 웹사이트를 만들 수 있게 됩니다.

| 비용 | 수천만원~수억원 | 300만원 (현재) |
| 기간 | 수개월~1년 | 1주일 |
| 결과 | 의도와 다를 수 있음 | 직접 통제 가능 |

# 6. 기술 스택

SSAL Works가 7개월간 검증한 기술 조합입니다.

---

## 6-1. 기술 스택 요약

| 분야 | 기술 | 역할 |
|------|------|------|
| AI 도구 | **Claude Code** | 코딩, 파일 관리, 프로젝트 관리 |
| 데이터베이스 | **Supabase** | DB, 인증, 스토리지, API |
| 배포 | **Vercel** | 자동 배포, 도메인 연결 |
| 이메일 | **Resend** | 이메일 발송 |
| 결제 | **TossPayments** | 결제 시스템 |

---

## 6-2. Claude Code

Anthropic이 개발한 터미널 기반 AI 코딩 도구입니다. 로컬 파일에 직접 접근하고 명령을 실행할 수 있습니다. 모든 프로그래밍 언어로 코드를 작성하고, 파일을 생성/수정/삭제하며, Git 작업(커밋, 푸시, 브랜치)과 npm, 배포 명령 등을 실행합니다.

---

## 6-3. Supabase

PostgreSQL 기반의 Backend as a Service(BaaS)로, Firebase의 오픈소스 대안입니다. Database, Auth(이메일/소셜 로그인), Storage(파일 저장), Edge Functions(서버리스 함수), Realtime(실시간 동기화)을 제공합니다.

---

## 6-4. Vercel

Git 연동 자동 배포 플랫폼입니다. Git push 시 자동 배포, Serverless Functions(API 함수 호스팅), 커스텀 도메인 연결, 자동 HTTPS(SSL)를 제공합니다.

---

## 6-5. Resend

개발자 친화적 API를 제공하는 이메일 발송 서비스입니다. 회원가입 인증 코드, 비밀번호 재설정 링크, 서비스 알림 발송에 사용됩니다.

---

## 6-6. TossPayments

국내 결제 환경에 최적화된 결제 서비스입니다. 신용카드, 계좌이체, 가상계좌, 간편결제를 지원합니다.

---

## 6-7. 보안 주의사항

풀스택 웹사이트 개발 시 반드시 지켜야 할 보안 사항입니다.

### 필수 보안 설정

| # | 항목 | 설명 |
|---|------|------|
| 1 | **Supabase RLS 활성화** | Row Level Security를 반드시 활성화하세요. RLS 없이 배포하면 데이터가 무방비 상태입니다. |
| 2 | **.env 파일 Git 커밋 금지** | API 키가 담긴 .env 파일은 절대 Git에 올리면 안 됩니다. .gitignore에 추가하세요. |
| 3 | **API Key 노출 금지** | Service Role Key를 클라이언트(프론트엔드) 코드에 넣지 마세요. 서버에서만 사용하세요. |
| 4 | **CORS 설정 확인** | 프로덕션 배포 전 CORS 설정을 확인하세요. 허용된 도메인만 API 접근을 허용하세요. |

### 보안 체크리스트

개발 완료 후 배포 전 반드시 확인하세요:

- [ ] Supabase RLS 정책이 모든 테이블에 설정되어 있는가?
- [ ] .env 파일이 .gitignore에 포함되어 있는가?
- [ ] 클라이언트 코드에 Service Role Key가 없는가?
- [ ] Vercel 환경변수에 민감 정보가 올바르게 설정되어 있는가?
