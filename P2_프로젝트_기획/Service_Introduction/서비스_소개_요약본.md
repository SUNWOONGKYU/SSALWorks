# 1. SSAL Works 개요

## 1-1. 한 문장 정의

**Claude Code를 활용한 3차원 SAL Grid 기반 풀스택 웹사이트 개발 플랫폼**

비개발자도 AI와 함께 풀스택 웹사이트를 만들 수 있는 곳입니다.

---

## 1-2. 핵심 컨셉 - 황금쌀 방앗간

```
┌─────────────────────────────────────────────────────────┐
│   [창의적 아이디어] ──→ [AI가 코딩] ──→ [풀스택 웹사이트]  │
│        (당신)           (Claude Code)    (수익 창출)     │
└─────────────────────────────────────────────────────────┘
```

| 방앗간 비유 | SSAL Works |
|-----------|------------|
| 벼 (재료) | 창의적 아이디어 |
| 방아 (노동) | AI (Claude Code) |
| 쌀 (결과) | 풀스택 웹사이트 |
| 황금쌀 | 돈 버는 웹사이트 |

---

## 1-3. 대상 고객 (3가지 조건)

| # | 조건 | 설명 |
|---|------|------|
| 1 | 사업 아이디어 | 풀스택 웹사이트로 구현할 아이디어 |

# 3. SAL Grid 방법론

## 3-1. SAL = Stage + Area + Level

**3차원 좌표계로 프로젝트 전체를 관리**

| 축 | 구성 | 개수 |
|----|------|:----:|
| X축 | Stage (개발 단계) | 5개 |
| Y축 | Area (개발 영역) | 11개 |
| Z축 | Level (Task 순서) | 다수 |

---

## 3-2. Stage (개발 단계)

| Stage | 이름 | 주요 내용 |
|-------|------|----------|
| S1 | 개발 준비 | 환경설정, DB 스키마 |
| S2 | 개발 1차 | OAuth, 회원가입 |
| S3 | 개발 2차 | AI 연동 |
| S4 | 개발 3차 | 결제, 관리자 |
| S5 | 개발 마무리 | 배포, QA |

---

## 3-3. Area (개발 영역)

| 코드 | 영역 | 코드 | 영역 |
|------|------|------|------|
| M | 문서화 | S | 보안/인증 |
| U | UI/UX 디자인 | T | 테스트 |
| F | 프론트엔드 | O | 운영/배포 |
| BI | 백엔드 기반 | E | 외부 연동 |
| BA | 백엔드 API | C | 콘텐츠 |
| D | 데이터베이스 | | |

---

## 3-4. Task 관리 (22개 속성)

각 Task는 22개 속성으로 관리:
- 기본 정보: task_id, task_name, stage, area, level
- 작업 정보: status, progress, dependencies, instruction, agent
- 결과 정보: generated_files, duration, build_result
- 검증 정보: verification 관련 필드들

# 4. 제공 도구 및 콘텐츠

## 4-1. Order Sheet 체계

AI에게 작업을 지시하는 표준화된 양식

| 구성 | 내용 |
|------|------|
| Part A | 공통 규칙 (7대 작업 규칙, 5대 절대 규칙) |
| Part B | 프로젝트별 커스터마이징 |
| 템플릿 | 32개 제공 |

---

## 4-2. Briefing (안내문)

각 단계에서 해야 할 일을 안내하는 문서

| 종류 | 내용 |
|------|------|
| 환영 메시지 | 회원가입 후 안내 |
| 플랫폼 사용법 | 전체 흐름 안내 |
| P0~S5 안내문 | 단계별 상세 안내 |

---

## 4-3. Dev Package

AI 협업 웹 개발 올인원 통합 패키지

| 구성 요소 | 내용 |
|----------|------|
| 프로젝트 구조 | P0~S5 10단계 폴더 |
| AI 설정 | .claude/ 폴더 (rules, methods, compliance) |
| Order Sheet | 32개 템플릿 |
| 자동화 스크립트 | 3개 (hooks, sync, build) |
| SAL Grid | 83개 Task |
| 배포 구조 | Vercel 즉시 배포 가능 |

**절약 효과: 4~6주**

# 8. 이용 프로세스

## 8-1. 4단계 프로세스

```
1. 회원가입 (무료)
       ↓
2. 둘러보기 (콘텐츠 확인)
       ↓
3. 빌더 계정 개설 (유용하다 판단 시)
       ↓
4. 프로젝트 등록 → 개발 시작
```

---

## 8-2. 개발 프로세스

```
Dev Package 다운로드
       ↓
Claude Code 실행
       ↓
Briefing 확인 → Order Sheet 작성
       ↓
Claude Code에 전달 → 작업 수행
       ↓
SAL Grid 업데이트 → 다음 Task
       ↓
Stage Gate 검증 → 다음 Stage
```
