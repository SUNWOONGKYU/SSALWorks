# Order Sheet - S2 개발 1차 (Core Development)

> **작성일**: 2025-12-19
> **버전**: 3.2
> **목적**: S2 Stage 전체 작업 실행 지시

---

## 실행 지시

```
S2 개발 1차 Stage 작업을 실행해라.

[Step 1] SAL Grid 확인
- Supabase ssalworks_tasks 테이블 조회
- Stage = 2 (S2)인 Task 목록 전부 가져와라

[Step 2] Task 실행 순서 결정
- Grid의 dependencies 필드 확인
- 의존성 없는 Task부터 실행
- 의존성 충족된 Task만 순차 실행

[Step 3] 각 Task 실행
- Grid의 task_instruction 파일 읽고 수행
- Grid의 task_agent에 지정된 서브에이전트 투입
- 서브에이전트가 작업 수행 후 결과 반환
- execution_type 확인:
  * AI-Only: 바로 실행
  * Human-AI: AI 작업 완료 후 PO에게 도움 요청

[Step 4] 검증 (필수! 생략 금지!)
- Grid의 verification_agent 서브에이전트 투입
- verification_instruction 파일에 따라 검증 수행
- 서브에이전트가 검증 완료 후 결과 반환
- ⚠️ 검증 없이 다음 Task로 넘어가면 안 됨!

[Step 5] 결과 JSON 파일 저장 (Main Agent)
- 저장 위치: S0_Project-SAL-Grid_생성/sal-grid/task-results/{TaskID}_result.json
- Task 실행 결과 + Verification 결과 전부 포함:

{
  "task_id": "S2XX1",
  "task_progress": 100,
  "task_status": "Completed",
  "generated_files": ["파일경로1", "파일경로2"],
  "modification_history": "[2025-12-19] 초기 구현",
  "test": {
    "unit_test": "✅ 통과",
    "integration_test": "✅ 통과"
  },
  "build": {
    "compile": "✅ 오류 없음",
    "lint": "✅ 통과"
  },
  "integration_verification": {
    "dependency_propagation": "✅",
    "cross_task_connection": "✅"
  },
  "blockers": {
    "status": "No Blockers ✅"
  },
  "comprehensive_verification": {
    "task_instruction": "✅ 충족",
    "test": "✅ 통과",
    "build": "✅ 통과",
    "final": "✅ Passed"
  },
  "verification_status": "Passed",
  "remarks": "비고"
}

[Step 6] Supabase 동기화
- Sync 스크립트 실행:
  cd S0_Project-SAL-Grid_생성/supabase
  node sync_task_results_to_db.js
- JSON 파일 → ssalworks_tasks 테이블 자동 UPDATE

[Step 7] 결과물 파일 저장
- Task ID의 Stage + Area로 폴더 결정
  예: S2BA1 → S2_개발-1차/Backend_APIs/
- 코드 파일은 Production/ 폴더에도 이중 저장

[Step 8] 다음 Task 진행
- Step 3~7 반복
- 모든 S2 Task 완료까지 반복

[Step 9] Stage Gate (모든 Task 완료 후)
- 모든 S2 Task 완료 확인
- Stage Gate 검증 수행
- 리포트 저장: S0_Project-SAL-Grid_생성/sal-grid/stage-gates/S2GATE_verification_report.md

[Step 10] PO 기능 테스트 요청
- PO 테스트 체크리스트를 Web_ClaudeCode_Bridge/Outbox/에 저장
- PO 테스트 완료 및 승인 후 Stage Gate 최종 통과

작업 시작해라.
```

## 핵심 원칙

1. **Grid가 진실의 원천**: 모든 Task 정보는 ssalworks_tasks 테이블에 있음
2. **서브에이전트 투입**: task_agent, verification_agent 사용
3. **JSON 파일로 기록**: task-results/{TaskID}_result.json 저장
4. **Sync 스크립트로 DB 업데이트**: node sync_task_results_to_db.js
5. **검증 필수**: Verification 생략 금지!
