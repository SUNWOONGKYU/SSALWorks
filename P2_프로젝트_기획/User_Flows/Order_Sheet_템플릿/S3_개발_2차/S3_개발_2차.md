> **작성일**: 2025-12-18
> **버전**: 3.0
> **목적**: S3 Stage 전체 작업 실행 지시

---

## 실행 지시

**아래 내용을 Claude Code에게 전달하면 됩니다:**

```
S3 개발 2차 Stage 작업을 실행해라.

[Step 0] Order Sheet 저장
- 이 Order Sheet를 Web_ClaudeCode_Bridge/Orders/ 폴더에 JSON 형식으로 저장
- 파일명: ORDER-S3-[날짜]-[순번].json
- 저장 완료 후 작업 개시
- 예외: Orders 폴더에서 직접 발견한 경우 저장 생략 (이미 저장됨)

[Step 1] SSAL Grid 확인
- S0_Project-SSAL-Grid_생성/ssal-grid/ 폴더 확인
- 또는 Supabase project_ssal_grid 테이블 조회
- Stage = 'S3'인 Task 목록 전부 가져와라

[Step 2] Task 실행 순서 결정
- Grid의 Dependencies(#9) 필드 확인
- 의존성 없는 Task부터 실행
- 의존성 충족된 Task만 순차 실행

[Step 3] 각 Task 실행
- Grid의 Task Instruction(#5) 읽고 수행
- Grid의 Task Agent(#7)에 지정된 서브에이전트 투입
- Execution Type(#8) 확인:
  * AI-Only: 바로 실행
  * Human-AI: AI 작업 완료 후 PO에게 도움 요청
  * PO 실행: 가이드 작성 후 PO에게 실행 요청

[Step 3-1] Human-AI Task 처리 (해당 시)
- AI가 할 수 있는 부분 먼저 완료 (코드 작성, 가이드 문서)
- PO 필수 작업 가이드를 Web_ClaudeCode_Bridge/Outbox/에 저장
- PO에게 알림: "해당 Task의 PO 작업이 필요합니다. 가이드를 확인해주세요."
- PO 작업 완료 확인 후 다음 Task 진행

[Step 4] 결과물 저장
- Task ID의 Stage + Area로 폴더 결정
  예: S3BA1 → S3_개발-2차/Backend_APIs/
- 코드 파일은 Production/ 폴더에도 이중 저장

[Step 5] 검증
- Grid의 Verification Agent(#15) 서브에이전트로 검증
- 검증 결과 Grid에 기록 (#16~#21)

[Step 6] Grid 업데이트
- Task Status(#11) = 'Completed'
- Generated Files(#12) = 생성된 파일 목록
- Task Progress(#10) = 100

[Step 7] Stage Gate
- 모든 S3 Task 완료 후 Stage Gate 검증
- 리포트 저장: S0_Project-SSAL-Grid_생성/ssal-grid/stage-gates/S3GATE_verification_report.md

[Step 8] PO 기능 테스트 요청
- 실제 기능 테스트는 PO가 수행해야 함 (외부 서비스 연동, API 키 필요)
- PO 테스트 체크리스트를 Web_ClaudeCode_Bridge/Outbox/에 저장
- 테스트 항목별로 "테스트 방법"과 "예상 결과" 명시
- PO 테스트 완료 및 승인 후 Stage Gate 최종 통과

작업 시작해라.
```

## 핵심 원칙

1. **Grid가 진실의 원천**: 모든 Task 정보는 SSAL Grid에 있음
2. **Grid를 읽고 실행**: Order Sheet는 "Grid 보고 해라"라고 지시할 뿐
3. **Grid에 기록**: 작업 결과는 Grid에 업데이트
4. **의존성 순서**: Dependencies(#9) 필드가 실행 순서 결정
5. **서브에이전트 투입**: Task Agent(#7), Verification Agent(#15) 사용
