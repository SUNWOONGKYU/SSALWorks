2. Stage 개요

S2 (개발 1차)는 핵심 기능을 구현하는 단계입니다.

Stage 목표

  • 인증/인가 시스템 구축
  • 이메일 시스템 연동
  • 핵심 API 구현
  • 프론트엔드 핵심 페이지 구현
  • 에러 핸들링 시스템 구축
  • 테스트 인프라 구축
  • API 문서화

3. Stage 작업 영역

이 Stage에서는 다음 Area의 작업이 진행됩니다:

| Area | 작업 내용 | 유형 |

|------|----------|------|

| Documentation | API 문서화 | AI-Only |

| Frontend | 인증 페이지, 핵심 UI | AI-Only |

| Backend_Infra | 이메일 서비스, 에러 핸들링 | Human-AI |

| Backend_APIs | 인증 API, 핵심 API | AI-Only |

| Database | 인덱스 최적화 | PO 실행 |

| Security | 인증 미들웨어 | AI-Only |

| Testing | API 테스트 | AI-Only |

| Content | 초기 콘텐츠 설정 | PO 실행 |

상세 Task 목록은 Order Sheet를 참조하세요.

4. 작업 규칙

파일 저장 규칙 (이중 저장)

  • Frontend, Backend_APIs, Database → Stage 폴더 + Production 폴더
  • 문서 (Documentation, Testing) → Stage 폴더에만

새 폴더 생성 금지

기존 폴더 구조 내에서만 작업

5. 실행 프로세스 (10단계)

```

[Step 0] Order Sheet 저장

[Step 1] Project SAL Grid 확인 - Stage Task 목록 조회

[Step 2] Task 실행 순서 결정 - Dependencies 기반

[Step 3] 각 Task 실행 - Task Agent 서브에이전트 투입

[Step 3-1] Human-AI Task 처리 (해당 시)

[Step 4] 결과물 저장 - Stage/Area 폴더 + Production 이중 저장

[Step 5] 검증 - Verification Agent 서브에이전트로 검증

[Step 6] Grid 업데이트 - Status, Files, Progress 기록

[Step 7] Stage Gate - 모든 Task 완료 후 Stage Gate 검증

[Step 8] PO 기능 테스트 요청 - 최종 승인

```

핵심 원칙:

  • Task Agent ≠ Verification Agent (작성자와 검증자 분리)
  • Grid가 진실의 원천: 모든 Task 정보는 Project SAL Grid에 있음
  • 서브에이전트 투입: Task Agent, Verification Agent 사용

6. PO(Project Owner) 필수 작업

Backend_Infra 영역

  • 이메일 서비스 계정 생성 및 API Key 발급
  • 환경변수 설정
  • 이메일 도메인 인증 (DNS 레코드 설정)

Database 영역

  • SQL Editor에서 인덱스 SQL 실행
  • 인덱스 생성 확인

Content 영역

  • 초기 시드 데이터 SQL 실행
  • 데이터 확인
