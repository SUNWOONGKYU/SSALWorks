2. Stage 개요

S1 (개발 준비)는 모든 후속 개발의 기반을 구축하는 단계입니다.

Stage 목표

  • 프로젝트 기본 구조 설정
  • 디자인 시스템 구축 (SSAL Works 브랜딩)
  • DB 스키마 확정 및 적용
  • 인증 Provider 설정 (Google OAuth)
  • Vercel 배포 환경 구축
  • 기술 문서화

4. 작업 규칙

파일 저장 규칙 (이중 저장)

  • Frontend, Database 코드 → Stage 폴더 + Production 폴더
  • 문서 → Stage 폴더에만

Task ID 코멘트

모든 파일 첫 줄에 Task ID 명시:

```javascript

// Task ID: S1F1 - 기본 프론트엔드 구조

```

새 폴더 생성 금지

기존 폴더 구조 내에서만 작업

5. 검증 프로세스 (3단계)

```

1단계: Task Agent → 작업 수행

2단계: Verification Agent → 검증 (작성자 ≠ 검증자)

3단계: Stage Gate → Main Agent 직접 수행

4단계: PO 승인 → 테스트 후 승인/거부

```

6. PO(Project Owner) 필수 작업

S1S1: Google OAuth 설정

1. Google Cloud Console에서 OAuth Client 생성

2. Supabase Dashboard에서 Google Provider 활성화

3. Client ID, Secret 입력

S1D1: DB 스키마 실행

1. Supabase SQL Editor 접속

2. SQL 파일들 순서대로 실행

3. 테이블 생성 확인

S1BI1: Vercel 배포 설정

1. Vercel 프로젝트 생성

2. GitHub 연결

3. 환경변수 설정

4. 배포
