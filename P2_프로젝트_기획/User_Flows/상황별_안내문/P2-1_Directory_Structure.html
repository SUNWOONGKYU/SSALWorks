3.1 SSALWorks 표준 구조 적용

  • P0~P3 예비단계 폴더
  • S0~S5 실행단계 폴더
  • Production 배포용 폴더
  • Web_ClaudeCode_Bridge 브릿지 폴더

3.2 폴더별 역할 정의

  • 각 폴더의 용도 설명
  • 파일 저장 규칙
  • Area별 하위 폴더 구조

3.3 네이밍 규칙 수립

  • 대분류 폴더: 한글+숫자 (P1_사업계획)
  • 하위 폴더: 영문 (Frontend, Database)
  • 파일: Task ID 포함 권장
