## 2. Stage 개요

**S1 (개발 준비)**는 모든 후속 개발의 기반을 구축하는 단계입니다.

### Stage 목표
- 배포 환경 설정 및 구성
- 환경변수 설정
- DB 스키마 확정 및 적용
- 인증 Provider 설정
- 테스트 환경 설정
- DNS 설정 및 도메인 연결
- 개발 가이드 문서화

---

## 3. Stage 작업 영역

이 Stage에서는 다음 Area의 작업이 진행됩니다:

| Area | 작업 내용 | 유형 |
|------|----------|------|
| Documentation | 개발 가이드 문서화 | AI-Only |
| Frontend | 프로젝트 설정, 빌드 구성 | Human-AI |
| Backend_Infra | 환경변수 설정 | Human-AI |
| Database | DB 스키마 적용 | PO 실행 |
| Security | 인증 Provider 설정 | Human-AI |
| Testing | 테스트 환경 설정 | AI-Only |
| DevOps | DNS 및 도메인 설정 | Human-AI |

**상세 Task 목록은 Order Sheet를 참조하세요.**

---

## 4. 작업 규칙

### 파일 저장 규칙 (이중 저장)
- Frontend, Database 코드 → Stage 폴더 + Production 폴더
- 문서 → Stage 폴더에만

### 새 폴더 생성 금지
기존 폴더 구조 내에서만 작업

---

## 5. 실행 프로세스 (10단계)

```
[Step 0] Order Sheet 저장
[Step 1] Project SAL Grid 확인 - Stage Task 목록 조회
[Step 2] Task 실행 순서 결정 - Dependencies 기반
[Step 3] 각 Task 실행 - Task Agent 서브에이전트 투입
[Step 3-1] Human-AI Task 처리 (해당 시)
[Step 4] 결과물 저장 - Stage/Area 폴더 + Production 이중 저장
[Step 5] 검증 - Verification Agent 서브에이전트로 검증
[Step 6] Grid 업데이트 - Status, Files, Progress 기록
[Step 7] Stage Gate - 모든 Task 완료 후 Stage Gate 검증
[Step 8] PO 기능 테스트 요청 - 최종 승인
```

**핵심 원칙**:
- Task Agent ≠ Verification Agent (작성자와 검증자 분리)
- Grid가 진실의 원천: 모든 Task 정보는 Project SAL Grid에 있음
- 서브에이전트 투입: Task Agent, Verification Agent 사용

## 6. PO(Project Owner) 필수 작업

### Frontend 영역
- 배포 플랫폼 프로젝트 생성
- 저장소 연결
- 빌드 설정 확인

### Backend_Infra 영역
- 환경변수 설정 (API 키, DB 연결 정보 등)

### Database 영역
- DB 콘솔에서 SQL 스키마 실행
- 테이블 생성 확인

### Security 영역
- OAuth Provider 설정 (Google 등)
- Client ID, Secret 입력

### DevOps 영역
- 도메인 추가 및 DNS 설정
- SSL 인증서 확인
