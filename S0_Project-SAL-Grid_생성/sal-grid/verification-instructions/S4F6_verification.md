# S4F6: 인앱 알림 UI - 검증 지침

## 검증 정보
- **Task ID**: S4F6
- **Verification Agent**: code-reviewer

## 검증 항목

### 1. UI 검증
- [ ] 헤더에 🔔 알림 벨 아이콘이 표시되는가
- [ ] 읽지 않은 알림 배지가 표시되는가 (숫자 또는 빨간 점)
- [ ] 배지는 알림이 없을 때 숨겨지는가
- [ ] 알림 벨 클릭 시 드롭다운이 토글되는가
- [ ] 드롭다운 스타일이 적절한가 (위치, 그림자, 테두리)

### 2. 기능 테스트
- [ ] `loadUserNotifications()`: 알림 목록 로드 성공
- [ ] `renderNotificationList()`: 알림 항목 정상 렌더링
- [ ] `markNotificationRead()`: 개별 알림 읽음 처리 성공
- [ ] `markAllNotificationsRead()`: 전체 읽음 처리 성공
- [ ] `formatTimeAgo()`: 상대 시간 정확히 표시

### 3. 알림 유형별 표시
- [ ] 각 알림 유형에 맞는 아이콘 표시
- [ ] 제목과 메시지 정상 표시
- [ ] 시간(방금 전, N분 전 등) 정확히 표시
- [ ] 읽은 알림과 읽지 않은 알림 스타일 구분

### 4. 관리자 연동 확인
- [ ] 입금 확인 시 사용자에게 알림 생성되는가
- [ ] 잔액 부족 알림 발송 시 알림 생성되는가
- [ ] 개발자 계정 개설 시 알림 생성되는가
- [ ] 무료 기간 종료 안내 시 알림 생성되는가

### 5. 에러 처리
- [ ] API 오류 시 적절한 에러 처리
- [ ] 알림이 없을 때 "알림이 없습니다" 메시지 표시

## 통과 기준

1. 알림 벨 클릭 시 드롭다운 정상 표시
2. 테스트 알림이 목록에 정상 표시
3. 읽음 처리 후 배지 카운트 감소
4. 관리자 작업 시 사용자 알림 자동 생성
