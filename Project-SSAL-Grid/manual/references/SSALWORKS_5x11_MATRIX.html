SSALWorks 5×11 표준 매트릭스

작성일: 2025-11-23 수정일: 2025-12-12 버전: v3.0 용도: 모든 프로젝트에 적용 가능한 표준 Stage×Area 프레임워크

📊 5 Stages × 11 Areas = 55개 표준 영역

중요 변경사항 (v3.0):

  • 기존 6×11 → 5×11 매트릭스로 변경
  • P1-P3 (Preliminary)는 GRID 범위 밖에서 별도 관리
  • PROJECT SAL GRID는 S1-S5 실행단계만 관리

기호 설명

  • P (Preliminary): 예비단계 - GRID 범위 밖 (P1, P2, P3)
  • S (Stage): 실행단계 - GRID 관리 범위 (S1, S2, S3, S4, S5)

5 Stages (GRID 관리 범위)

Stage 1: Development Setup (개발 준비)
Stage 2: Core Development (개발 1차)
Stage 3: Advanced Features (개발 2차)
Stage 4: QA & Optimization (개발 3차)
Stage 5: Operations (운영)

11 Areas (영역)

M  - Documentation (문서화)
U  - Design (UI/UX 디자인)
F  - Frontend (프론트엔드)
BI - Backend Infrastructure (백엔드 기반)
BA - Backend APIs (백엔드 API)
D  - Database (데이터베이스)
S  - Security (보안/인증/인가)
T  - Testing (테스트)
O  - DevOps (운영/배포)
E  - External (외부 연동)
C  - Content System (콘텐츠 시스템)

📋 Preliminary 단계 (GRID 범위 밖)

중요: P1, P2, P3는 PROJECT SAL GRID 생성 전에 완료되어야 합니다. 이 단계들은 GRID가 아닌 별도의 관리 체계로 진행합니다.

P1: 사업계획 (Business Planning)

항목 설명
비전/미션 사업 방향성 정의
시장 분석 타겟 시장 및 경쟁사 분석
비즈니스 모델 수익 모델 설계

산출물 위치: P1_사업계획/

P2: 프로젝트 기획 (Project Planning)

항목 설명
디렉토리 구조 프로젝트 폴더 구조 설계
요구사항 정의 기능 요구사항, 사용자 플로우
UI/UX 설계 디자인 시스템, 목업
DB 설계 데이터베이스 스키마 설계

산출물 위치: P2_프로젝트_기획/

P3: 프로토타입 제작 (Prototype Development)

항목 설명
프론트엔드 핵심 화면 프로토타입 (HTML/CSS/JS)
데이터베이스 Supabase 스키마 및 샘플 데이터
검증 핵심 기능 작동 확인

산출물 위치: P3_프로토타입_제작/

⚠️ P3 완료 후 → PROJECT SAL GRID 생성 → S1 시작

🎯 매트릭스 활용 방법

1. 프로젝트 초기 설정

1. P1, P2, P3 완료 (GRID 범위 밖)
2. P3 → S1 Gate 통과
3. PROJECT SAL GRID 생성
4. 5개 Stage 확인
5. 각 Stage에서 필요한 Area 선택
6. 55개 조합 중 실제 사용할 영역만 활성화

2. Task 생성

예시: "회원가입 API 구현"
→ Stage 2 (개발 1차) + Area BA (Backend APIs)
→ Task ID: S2BA1

🎯 원칙

모든 작업은 5×11 매트릭스 안에 배치

  • 새로운 작업이 생기면 → 기존 55개 조합 중 하나에 매핑
  • 어디에도 맞지 않으면 → 가장 가까운 영역에 포함
  • 절대 새로운 Stage나 Area 추가하지 않음

프로젝트 특성에 따라 선택적 사용

  • 간단한 프로젝트: 15-30개 영역 사용
  • 복잡한 프로젝트: 40-55개 영역 사용

P1-P3는 GRID 밖에서 관리

  • Preliminary 단계는 별도 폴더 및 관리 체계
  • P3 완료 후 Gate 검증 → GRID 생성 → S1 시작
