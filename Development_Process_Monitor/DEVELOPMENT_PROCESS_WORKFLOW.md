## 개요

SSALWorks 프로젝트는 **8단계 프로세스**로 구성됩니다:
- **Preliminary (예비단계)**: P1 → P2 → P3 (사업 기획 → 검증된 프로토타입)
- **Stage (실행단계)**: S1 → S2 → S3 → S4 → S5 (개발 → 운영)

## PRELIMINARY (예비단계)

예비단계는 본격적인 개발 전에 **사업 타당성 검토**, **기획**, **프로토타입 검증**을 수행합니다.

### P1: 사업계획 (Business Planning)

**목적**: 사업의 방향성과 수익 모델을 정의

**폴더**: `P1_사업계획/`

| 하위 항목 | 설명 |
|----------|------|
| P1-1_Vision_Mission | 비전과 미션 정의 |
| P1-2_Market_Analysis | 시장 분석 및 타겟 고객 정의 |
| P1-3_Business_Model | 비즈니스 모델 및 수익 모델 설계 |

**주요 산출물**:
- `vision_mission.md` - 비전/미션 선언문
- `market_analysis.md` - 시장 분석 보고서
- `revenue_model.md` - 수익 모델 문서
- `target_users.md` - 타겟 사용자 정의

**Gate 조건**: 비즈니스 모델 확정, 타겟 시장 정의 완료

---

### P2: 프로젝트 기획 (Project Planning)

**목적**: 기술적 설계와 UI/UX 기획 완료

**폴더**: `P2_프로젝트_기획/`

| 하위 항목 | 설명 |
|----------|------|
| P2-1_Directory_Structure | 프로젝트 디렉토리 구조 설계 |
| P2-2_Project_Plan | 요구사항, 사용자 플로우, 워크플로우 |
| P2-3_UI_UX_Design | 디자인 가이드라인, 목업, 프로토타입 |
| P2-4_Database_Design | 데이터베이스 스키마 설계 |

**주요 산출물**:
- `PROJECT_DIRECTORY_STRUCTURE.md` - 디렉토리 구조
- `PROJECT_PLAN.md` - 프로젝트 계획서
- `DESIGN_SYSTEM.md` - 디자인 시스템
- `database_schema.md` - DB 스키마 설계

**Gate 조건**: 요구사항 정의, UI/UX 목업, DB 스키마 완료

---

### P3: 프로토타입 제작 (Prototype Development)

**목적**: 핵심 기능을 검증하는 작동하는 프로토타입 구축

**폴더**: `P3_프로토타입_제작/`

| 하위 항목 | 설명 |
|----------|------|
| P3-1_Frontend | 프론트엔드 프로토타입 (HTML/CSS/JS) |
| P3-2_Database | Supabase 스키마 SQL 파일 |
| P3-3_Scripts | 자동화 스크립트 |

**주요 산출물**:
- 프로토타입 웹페이지 (HTML/CSS/JS)
- Supabase DB 스키마 (SQL 파일들)
- 샘플 데이터

**Gate 조건**: 핵심 기능 작동 확인, 사용자 피드백 반영

---

## STAGE (실행단계)

실행단계는 **본격적인 개발**부터 **운영**까지의 단계입니다.

### S1: 개발 준비 (Development Setup)

**목적**: 개발 환경 구축 및 인프라 설정

**폴더**: `S1_개발_준비/`

| 하위 항목 | 설명 |
|----------|------|
| S1-1_Tech_Stack | 기술 스택 문서 |
| S1-2_Architecture | 시스템 아키텍처 설계 |
| S1-3_Development_Setup | 개발 환경 설정 (Git, Node, Supabase, Vercel) |
| S1-4_Backend_Infra | 백엔드 인프라 설정 |
| S1-5_Project_Grid | 프로젝트 그리드 (Task 관리 시스템) |

**주요 작업**:
- Git/GitHub 저장소 설정
- Node.js 개발 환경 구축
- Supabase 프로젝트 생성
- Vercel 프로젝트 연결
- CI/CD 파이프라인 설정

**Gate 조건**: 개발 환경 완전 가동, 팀원 온보딩 완료

---

### S2: 개발 1차 (Auth & Registration)

**목적**: 핵심 기능 구현

**폴더**: `S2_개발-1차/`

| 하위 항목 | 설명 |
|----------|------|
| S2-1_Frontend | 핵심 페이지 개발 |
| S2-2_Backend_APIs | 핵심 API 구현 |
| S2-3_Authentication | 인증 시스템 구현 |
| S2-4_Database | DB 마이그레이션 및 시드 데이터 |

**주요 작업**:
- 사용자 인증 (회원가입, 로그인, 소셜 로그인)
- 핵심 페이지 구현 (대시보드, 메인 기능)
- 기본 API 엔드포인트 구현
- DB 테이블 생성 및 RLS 정책

**Gate 조건**: 핵심 기능 완료, 기본 테스트 통과

---

### S3: 개발 2차 (AI Integration)

**목적**: 고급 기능 및 외부 서비스 연동

**폴더**: `S3_개발-2차/`

| 하위 항목 | 설명 |
|----------|------|
| Frontend | 추가 페이지 및 고급 UI |
| Backend_APIs | 추가 API 구현 |
| External_Services | 외부 서비스 연동 (결제, 이메일 등) |

**주요 작업**:
- 결제 시스템 연동 (Toss Payments)
- 이메일 서비스 연동 (Resend)
- AI 크레딧 시스템 구현
- 고급 UI 컴포넌트 구현

**Gate 조건**: 모든 기능 구현 완료, 통합 테스트 시작 가능

---

### S4: 개발 3차 (Payment & Admin)

**목적**: 품질 보증 및 성능 최적화

**폴더**: `S4_개발-3차/`

| 하위 항목 | 설명 |
|----------|------|
| Test | 테스트 코드 (Unit, E2E, Integration) |
| Stabilization | 버그 수정, 성능 튜닝, 보안 점검 |

**주요 작업**:
- Unit 테스트 작성
- E2E 테스트 작성
- 성능 최적화 (번들 사이즈, 로딩 속도)
- 보안 점검 및 취약점 수정
- 크로스 브라우저 테스트
- 반응형 테스트

**Gate 조건**: 모든 테스트 통과, 성능 기준 충족, 보안 검증 완료

---

### S5: 운영 (Operation & Maintenance)

**목적**: 프로덕션 배포 및 지속적 운영

**폴더**: `S5_개발_마무리/`

| 하위 항목 | 설명 |
|----------|------|
| Deployment | 배포 설정 (Vercel, 환경 변수) |
| Monitoring | 모니터링 설정 |
| Maintenance | 유지보수 절차 |
| Backup | 백업 전략 |
| Security | 보안 정책 |

**주요 작업**:
- Vercel 프로덕션 배포
- 커스텀 도메인 연결
- 환경 변수 설정
- 모니터링 대시보드 설정
- 백업 자동화
- 장애 대응 절차 수립

**지속 작업**: 버그 수정, 기능 개선, 보안 업데이트

## Gate 체크리스트

### P1 → P2 Gate (사업계획 → 기획)
- [ ] 비전/미션 정의 완료
- [ ] 타겟 시장 및 고객 정의
- [ ] 수익 모델 확정
- [ ] 경쟁사 분석 완료

### P2 → P3 Gate (기획 → 프로토타입)
- [ ] 요구사항 문서 완료
- [ ] UI/UX 목업 완료
- [ ] 데이터베이스 스키마 설계 완료
- [ ] 디자인 가이드라인 확정

### P3 → S1 Gate (프로토타입 → 개발준비)
- [ ] 프로토타입 핵심 기능 작동
- [ ] 사용자 피드백 반영
- [ ] 프로토타입 배포 완료
- [ ] 개발 범위 확정

### S1 → S2 Gate (개발준비 → 개발1차)
- [ ] 개발 환경 구축 완료
- [ ] Git 저장소 설정
- [ ] Supabase 프로젝트 생성
- [ ] CI/CD 파이프라인 구축

### S2 → S3 Gate (개발1차 → 개발2차)
- [ ] 인증 시스템 완료
- [ ] 핵심 페이지 구현 완료
- [ ] 기본 API 작동
- [ ] 기본 테스트 통과

### S3 → S4 Gate (개발2차 → 개발3차)
- [ ] 결제 시스템 연동 완료
- [ ] 외부 서비스 연동 완료
- [ ] 모든 기능 구현 완료
- [ ] 통합 테스트 가능 상태

### S4 → S5 Gate (개발3차 → 운영)
- [ ] 모든 테스트 통과 (Unit, E2E)
- [ ] 성능 기준 충족
- [ ] 보안 검증 완료
- [ ] 프로덕션 배포 준비 완료
