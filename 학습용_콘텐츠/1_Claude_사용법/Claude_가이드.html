1편. Claude 가이드

2025년 10월 기준

1. 소개

Claude란 무엇인가

Claude는 Anthropic이 개발한 대화형 인공지능 어시스턴트입니다. 자연어 처리, 코드 생성, 데이터 분석, 창작 작업 등 다양한 지적 작업을 수행할 수 있으며, 특히 안전성과 유용성의 균형을 중요하게 여기는 Constitutional AI 원칙을 기반으로 설계되었습니다.

Claude의 핵심 특징:

  • Extended Thinking: 복잡한 문제에 대해 단계적으로 사고하는 능력
  • Constitutional AI: 안전하고 유익한 응답을 생성하도록 설계된 AI 정렬 방식
  • 높은 컨텍스트 윈도우: 최대 200K 이상의 토큰을 한 번에 처리 가능
  • 멀티모달 기능: 텍스트뿐만 아니라 이미지 분석, 파일 처리 등 지원
  • 개발자 친화적: CLI, IDE 결합, MCP 프로토콜 등 개발 생태계 제공

Anthropic 회사 소개

Anthropic은 2021년 OpenAI 출신 연구진들이 설립한 AI 안전성 연구 회사입니다. "안전하고 해석 가능한 AI 시스템 구축"을 미션으로 삼고 있으며, Constitutional AI, Scaling Laws 연구 등 AI 안전성 분야에서 선도적인 역할을 하고 있습니다.

주요 연혁:

  • 2021년: Anthropic 설립
  • 2023년: Claude 2 출시
  • 2024년: Claude 3 패밀리 출시 (Opus, Sonnet, Haiku)
  • 2024년 11월: MCP (Model Context Protocol) 공개
  • 2025년 2월: Claude Code (CLI) 연구 프리뷰
  • 2025년 5월 22일: Claude 4 패밀리 및 Claude Code 정식 출시 (GA)
  • 2025년 8~9월: Opus 4.1, Sonnet 4.5 업그레이드

2. 설치 및 시작

2-1. 모바일-앱

Claude 모바일-앱 소개

스마트폰과 태블릿에서 사용하는 Claude 전용 모바일 애플리케이션으로, 다운로드를 받은 후에 복잡한 설정 없이 바로 대화를 시작할 수 있으며 언제 어디서나 이동하면서도 접속할 수 있습니다.

기본 기능:

  • AI 대화: 질문-답변을 통한 정보 제공
  • 이미지 분석: 이미지 업로드 및 해석
  • 코드 생성 및 분석: 프로그래밍 코드 작성, 분석 및 설명
  • 파일 처리: PDF, TXT, 코드 파일 등 처리
  • 프로젝트 관리: 주제별로 대화를 그룹화하여 체계적 관리
  • 대화 히스토리: 모든 대화 자동 저장 및 검색 기능
  • Artifacts: 코드, 문서, 다이어그램을 실시간 미리보기 창에서 표시
  • 모델 전환: Haiku, Sonnet, Opus 모델 실시간 전환
  • 실시간 동기화: 웹, 데스크톱-앱과 모든 대화 내용 자동 동기화

모바일 특화 기능:

  • 음성 입력: 마이크 버튼으로 음성 질문 및 명령
  • 카메라 연동: 실시간 사진 촬영 후 즉시 분석

제약사항:

  • 파일 크기 제한 (개당 10MB)
  • 모바일 기기 내부 파일 접근 불가
  • 화면 크기의 한계로 인한 긴 텍스트 작업 불편

사용환경:

  • 인터페이스: 모바일(터치) 입력 방식
  • 운영체제: iOS/Android 스마트폰 및 태블릿

활용 팁:

  • 회의록이나 화이트보드를 사진으로 찍어 즉시 요약
  • 운전 중 음성으로 아이디어 정리 및 메모
  • 외부에서 급한 데이터 분석이나 계산 처리
  • 이동 중 문서 검토 및 피드백 작성
  • 여러 언어 실시간 번역 (해외 출장 시 유용)

2-2. 웹

Claude 웹 소개

웹 브라우저에서 사용하는 Claude 온라인 서비스로, 설치나 다운로드 없이 바로 접속할 수 있으며 PC와 모바일 모두에서 동일한 방식으로 대화를 할 수 있습니다.

기본 기능: 모바일-앱과 동일

웹 추가 기능:

  • 파일 생성 및 분석 (2025년 9월 9일 출시):
    • Excel(.xlsx): 수식, 차트, 다중 시트 포함 스프레드시트 자동 생성
    • PowerPoint(.pptx): 전문 디자인의 프레젠테이션 즉시 제작
    • Word(.docx): 완성된 보고서 및 문서 작성
    • PDF: 최종 배포용 문서 생성
    • Python 기반 고급 데이터 분석 및 시각화
    • 파일 간 자동 변환 (PDF→PPT, Excel→차트 등)
  • PWA(Progressive Web App) 지원: 웹사이트를 독립적인 앱처럼 설치하여 사용할 수 있게 해주는 기술. 설치 후에는 일반 프로그램처럼 작업 표시줄에서 바로 실행 가능

제약사항:

  • 인터넷 연결 필수
  • 웹 브라우저의 성능 차이에 따른 영향 발생
  • 모바일 기기 및 PC 내부의 파일 접근 불가

사용환경:

  • 인터페이스: PC(마우스/키보드) 및 모바일(터치) 입력 방식
  • 운영체제: 모든 운영체제의 최신 웹 브라우저

활용 팁:

  • 설치 없이 어떤 PC에서도 즉시 사용 시작
  • 보고서나 프레젠테이션 자료를 즉시 만들어서 다운로드
  • CSV 데이터를 업로드하여 즉시 차트와 분석자료 생성
  • 마우스로 대화 내용을 복사하여 바로 활용
  • 여러 탭을 동시에 열어서 각각 다른 주제 작업 진행

2-3. 데스크톱-앱

Claude 데스크톱-앱 소개

PC에 설치하는 독립 실행형 Claude 애플리케이션으로, 웹보다 더 강력하고 다양한 기능을 제공하며 MCP(Model Context Protocol) 서버 연동을 통해서 로컬 파일에도 접근하여 읽고 쓰기를 할 수 있습니다.

기본 기능: 모바일-앱과 동일

웹 추가 기능: 웹과 동일

데스크톱-앱 추가 기능:

  • 내부 MCP 서버 3개 연동:
    • Filesystem MCP: 로컬 파일에 접근하여 읽기/쓰기/관리
    • Memory MCP: 대화 간 정보를 저장하고 학습
    • Anthropic MCP: 6가지 특화 도구(ask, stream, vision, multi_turn, count_tokens, tools)
  • 외부 MCP 서버 추가 연동: GitHub, Google Drive, Slack, PostgreSQL, Docker, Puppeteer 등

제약사항:

  • PC에서만 사용 가능
  • MCP 설정하는 과정 복잡

사용환경:

  • 인터페이스: PC(마우스/키보드) 입력 방식
  • 운영체제: Windows, Linux 등 데스크톱 환경

활용 팁:

  • filesystem MCP로 내 PC 내의 특정 파일을 직접 찾아서 편집
  • Anthropic MCP의 vision 도구로 스크린샷 즉시 분석
  • 여러개의 외부 MCP 서버를 연동시켜서 나만의 AI 워크스테이션 구축
  • PostgreSQL MCP로 데이터베이스를 직접 쿼리하고 분석
  • GitHub MCP로 문서와 코드의 모든 수정 과정을 버전별로 저장하고 복원

2. 설치 파일 다운로드

2-5. CLI (Claude Code)

Claude Code 소개

터미널에서 가동되는 AI 어시스턴트로, 시스템 명령을 실행하여 복잡한 업무를 자동으로 처리하고 파일도 직접 편집할 수 있는 비개발자와 개발자 모두를 위한 강력한 도구입니다.

기본 기능: 모바일-앱과 동일 (단, Artifacts 미지원, 이미지는 파일 경로로 제공)

Claude Code 특화 기능:

  • 파일 생성 및 편집: Excel, Word, PowerPoint, PDF 등 모든 형식의 파일을 생성/편집하여 로컬에 저장
  • 기술 문서 생성: README.md, API 연결 및 용법 문서, 설치 가이드, 사용자 매뉴얼 자동 작성
  • 외부 MCP 서버 연동: Google Drive, Slack, Jira 등 외부 서비스와 통합
  • 시스템 명령 실행: Git, npm, 파일 작업 등 모든 터미널 명령 실행
  • 프로젝트 관리: 파일 구조 분석 및 재구성
  • 자동화 작업: 프로그램 설치, 환경 변수 설정, 필요한 패키지 자동 설치, 빌드 및 배포 과정 자동화
  • 코드 작성: 전체 프로그램을 처음부터 작성
  • 디버깅 지원: 오류 메시지 분석 및 수정
  • 테스트 실행: 자동으로 테스트 작성 및 실행
  • Git 워크플로우 처리: 커밋 생성, 병합 충돌 해결, 릴리스 노트 작성
  • CI/CD 통합: 지속적 통합 파이프라인에서 자동 실행
  • 로그 모니터링: 실시간 로그 감시 및 이상 탐지
  • 기타: 프로그램으로 할 수 있는 거의 모든 작업 가능

제약사항:

  • 마우스 없이 키보드 입력을 통한 텍스트 명령어로만 조작

작업 모드 선택하기

Claude Code는 3가지 작업 모드를 제공합니다. 작업 성격에 따라 적절한 모드를 선택해서 사용해야 합니다:

(1) 일반 모드 (Default Mode)

  • 특징: 기본 모드로, 파일 수정 전에 매번 사용자 승인을 요청
  • 사용 시기: 중요한 파일 작업, 신중한 검토가 필요한 경우
  • 장점: 모든 변경사항을 사전에 확인 가능
  • 단점: 작업 속도가 느림
  • 설정 방법: 기본값 (별도 설정 불필요)

(2) 자동 수정 모드 (Accept Edits Mode)

  • 특징: 파일 수정을 자동으로 승인하여 즉시 실행
  • 사용 시기: 신뢰할 수 있는 작업, 빠른 진행이 필요한 경우
  • 장점: 작업 속도가 매우 빠름
  • 단점: 실수 시 즉시 파일이 변경됨
  • 설정 방법: 대화 중 /permission accept 입력

(3) 플래닝 모드 (Plan Mode)

  • 특징: 작업 실행 전에 전체 계획을 먼저 제시하고 승인 받음
  • 사용 시기: 복잡한 프로젝트, 여러 파일 수정, 구조적 변경이 필요한 경우
  • 장점: 전체 작업 흐름을 사전에 파악 가능
  • 단점: 초기 계획 수립 시간이 추가로 소요
  • 설정 방법: 대화 중 /permission plan 입력

💡 빠른 모드 전환: 대화 중 Shift + Tab 키를 누르면 모드를 순환하며 전환할 수 있습니다 (일반 → 자동 수정 → 플래닝 → 일반).

💡 팁: 처음에는 일반 모드로 시작하고, Claude Code의 작업 방식에 익숙해지면 자동 수정 모드를 활용하세요. 대규모 리팩토링이나 새 기능 개발 시에는 플래닝 모드가 유용합니다.

Claude Code와 데스크톱-앱의 차이점 이해

위의 비교표를 보면 Claude 4종 중에서도 특히 데스크톱-앱과 Claude Code가 대조적인 특성을 보입니다. 이러한 차이는 단순히 기술적 한계를 넘어서, 작업의 본질과 접근 방식의 차이에서 비롯됩니다.

작업 환경의 근본적 차이

Claude Code - 창조적 구현 환경

  • 특징: 새로운 코드 작성, 복잡한 시스템 설계, 예측하지 못한 오류 해결
  • 강점: 시스템 명령 실행, Git 통합, 프로그램 자동 설치 등 개발 작업에 필요한 모든 권한 보유
  • 적합한 작업:
    • 신규 프로젝트 개발
    • 복잡한 디버깅 작업
    • 시스템 레벨 자동화
    • 멀티 파일 코드 리팩토링

데스크톱-앱 - 구조적 정리 환경

  • 특징: 안정적이고 예측 가능한 작업 환경, GUI 기반 상호작용
  • 강점: 대화 동기화, Artifacts 표시, MCP를 통한 선택적 파일 접근
  • 적합한 작업:
    • 문서 작성 및 편집
    • 데이터 분석 및 시각화
    • 아이디어 정리 및 기획
    • 학습 및 리서치

컨텍스트 관리의 차이

Claude Code는 프로젝트 진행 과정에서 지속적으로 변화하는 코드베이스, 디버깅 결과, 시스템 상태 등을 실시간으로 추적해야 합니다. 각 명령의 결과가 다음 작업에 직접적인 영향을 미치기 때문에 매우 동적인 컨텍스트 관리가 필요합니다.

데스크톱-앱은 상대적으로 안정적인 컨텍스트를 유지합니다. 대화 히스토리가 자동 동기화되고, Projects 기능으로 장기적인 작업을 체계적으로 관리할 수 있어 일관성 있는 작업이 가능합니다.

오류 처리와 위험도

Claude Code에서의 오류:

  • 잘못된 명령 하나로 시스템 파일 손상 가능
  • 무한 루프나 과도한 리소스 사용으로 시스템 다운 위험
  • Git 작업 실수로 코드 손실 가능성

데스크톱-앱에서의 오류:

  • 대부분 수정 가능한 텍스트 오류
  • 시스템에 영향을 주지 않는 안전한 환경
  • 실수해도 대화 히스토리에서 복구 가능

Claude Code와 데스크톱-앱의 차이점을 분석해보면, 이는 단순히 기술적 한계의 문제가 아니라 작업의 본질, 복잡성의 수준, 요구되는 사고 방식, 그리고 환경적 요인들의 근본적인 차이에서 비롯됩니다. Claude Code는 창조적이고 복잡한 구현 작업에, 데스크톱-앱은 구조화되고 안정적인 정리 작업에 최적화되어 있습니다.

이러한 차이점을 이해하면 각자의 작업 상황과 목적에 따라 자연스럽게 적절한 도구를 선택할 수 있을 것입니다. 때로는 두 도구를 상호 보완적으로 활용하는 것이 가장 효과적일 수도 있습니다.

Anthropic API는 개발자가 자신의 애플리케이션, 서비스, 웹사이트에 Claude의 기능을 직접 통합할 수 있도록 제공하는 프로그래밍 인터페이스입니다. 웹/모바일/데스크톱-앱/CLI와 달리, 코드를 통해 Claude를 제어하고 대규모로 자동화할 수 있습니다.

기본 기능: Claude의 모든 대화 및 추론 기능

API 특화 기능:

  • Messages API: 기본 대화형 AI 응답 생성
  • Computer Use (Beta): 화면 제어 및 자동화 - 웹 브라우저 조작, 버튼 클릭, 텍스트 입력 등
  • Batch API: 대량 요청을 50% 할인된 가격에 24시간 내 처리
  • Prompt Caching: 반복되는 컨텍스트를 캐싱하여 90% 비용 절감
  • Message Streaming: 실시간 응답 스트리밍
  • Tool Use (Function Calling): 외부 API 및 함수 호출
  • Vision: 이미지 분석 및 이해
  • JSON Mode: 구조화된 JSON 출력 보장

제약사항:

  • 프로그래밍 지식 필수
  • API 키 관리 및 보안 책임
  • 사용량 기반 종량제 요금
  • Rate Limit 관리 필요

사용환경:

  • 인터페이스: 프로그래밍 언어 (Python, JavaScript, Java 등)
  • 운영체제: 모든 환경 (서버, 클라우드, 로컬)

활용 팁:

  • 고객 지원 챗봇 자동화
  • 대규모 문서 분석 및 요약 파이프라인
  • 코드 리뷰 및 생성 시스템

3-7-2. 5시간 리셋 시스템

리셋 시스템이란?

Claude는 5시간 단위 롤링 리셋 시스템을 적용하고 있습니다.

핵심 원칙:

  • 당신이 첫 메시지를 보낸 시점부터 정확히 5시간 후에 한도가 리셋됩니다
  • 모든 플랫폼(웹, 데스크톱, 모바일, CLI) 공유 - 한 곳에서 사용하면 다른 곳 한도도 감소
  • 고정된 시간대가 아닌, 개인별 타이머 방식

예시: Pro 플랜 사용자 (45개/5시간)

오전 9:00  - 첫 메시지 전송 (타이머 시작)
          - 45개 중 30개 사용 → 15개 남음

오후 2:00  - 정확히 5시간 경과, 한도 리셋!
          - 새로 45개 충전
          - 남은 15개 + 새 45개 = 총 60개 사용 가능

오후 3:00  - 60개 중 20개 사용 → 40개 남음

오후 7:00  - 다시 5시간 경과 (오후 2시부터), 리셋!
          - 남은 40개 + 새 45개 = 총 85개 사용 가능

중요:

  • 메시지 개수는 짧은 메시지 기준 "약" 45개입니다
  • 긴 메시지, 파일 첨부, 긴 대화는 더 많은 한도를 소비합니다
  • 실제로는 메시지 길이에 따라 20-70개까지 변동 가능합니다
리셋 시스템의 장점

(1) 유연한 사용: 한 번에 몰아쓰고 잠시 쉴 수 있음 (2) 빠른 회복: 최대 5시간만 기다리면 됨 (3) 누적 가능: 안 쓴 것은 쌓임 (일일 최대치까지)

시간대별 활용 전략도 중요합니다. 새벽 3시에서 7시는 서버가 한산하여 응답이 빠르고 한도가 최대로 누적되어 있어 대량 처리나 긴 문서 분석에 적합합니다. 오전 9시에서 12시는 업무 시작 시간으로, 9시에 시작하면 오후 2시에 리셋되므로 중요한 작업을 우선 처리하는 것이 좋습니다. 오후 2시에서 6시는 피크타임으로 Free 사용자는 제한을 받을 수 있지만, Pro 이상은 우선순위를 받습니다. 저녁 7시에서 11시는 개인 사용이 증가하는 시간입니다.

3-7-3. 토큰과 토큰 사용료

토큰이 뭔가요?

토큰은 AI가 텍스트를 처리하는 기본 단위로서, 1토큰은 대략 영어 3-5개 철자 또는 한글 2-3개 글자 정도에 해당합니다. (정확한 토큰 수는 AI 모델의 토크나이저에 따라 약간 차이가 있음)

토큰 = 사용량 = 사용료/비용

실생활 예시로 이해하기
📝 텍스트별 토큰 사용량

"안녕하세요"          → 약 3토큰
"Hello World"         → 약 2토큰
A4 용지 1장 (2,000자) → 약 1,000토큰
이미지 1장            → 약 1,000-5,000토큰
PDF 10페이지          → 약 10,000토큰
소설 1권              → 약 100,000토큰
Claude 토큰 계산법 - 입력과 출력 토큰의 차이

Claude의 토큰 계산은 **입력 토큰(Input Tokens)**과 **출력 토큰(Output Tokens)**으로 구분됩니다. 이 둘의 비용 구조가 다르기 때문에 정확한 이해가 필요합니다.

입력 토큰: 사용자가 Claude에게 보내는 모든 텍스트 (프롬프트, 대화 내용, 업로드된 문서 등) 출력 토큰: Claude가 생성하여 제공하는 모든 텍스트 (답변, 코드, 문서 등)

일반적으로 출력 토큰이 입력 토큰보다 5배 비쌉니다. 이는 AI 모델이 텍스트를 생성하는 것이 이해하는 것보다 계산적으로 더 복잡하기 때문입니다. 따라서 긴 답변을 요청할 때는 비용이 급격히 증가할 수 있습니다.

모델별 토큰 사용료 부과 기준

Haiku (경량형)

  • 입력: $0.80 per million tokens
  • 출력: $4.00 per million tokens

Sonnet (균형형)

  • 입력: $3 per million tokens
  • 출력: $15 per million tokens

Opus (고성능형)

  • 입력: $15 per million tokens
  • 출력: $75 per million tokens

중요: 단순 작업에 Opus를 사용하면 Haiku 대비 약 19배 비싼 비용을 지불하게 됩니다. 모델 선택에 따른 비용 차이가 극심하므로 작업 유형에 맞는 전략적 선택이 필수입니다.

토큰 사용량에 대한 과금 체계

Claude는 월정액 요금제 + 초과 사용량 별도 과금 체계로 운영되며, 초과 사용량 과금 시에 위에 제시된 모델별 토큰 사용료 부과 기준이 적용됩니다.

작업별 토큰 사용 예시 (Sonnet 4.5 기준)
작업 유형 입력 토큰 출력 토큰 총 토큰 예상 사용료 (Sonnet)
간단한 질문 50 100 150 $0.0002 (무시 가능)
이메일 작성 100 300 400 $0.0048 (약 $0.005)
코드 디버깅 500 1,000 1,500 $0.0165 (약 $0.017)
문서 요약 (5페이지) 5,000 500 5,500 $0.0225 (약 $0.023)
논문 분석 (30페이지) 30,000 2,000 32,000 $0.1200 (약 $0.12)

계산식: (입력 토큰 × $3/1M) + (출력 토큰 × $15/1M)

시나리오 3: 데이터 분석 (API)

단계:

import anthropic

client = anthropic.Anthropic()

# 데이터 파일 업로드 및 분석 요청
with open("sales_data.csv", "rb") as f:
    message = client.messages.create(
        model="claude-sonnet-4.5-20250514",
        max_tokens=4096,
        messages=[{
            "role": "user",
            "content": [
                {
                    "type": "document",
                    "source": {
                        "type": "base64",
                        "media_type": "text/csv",
                        "data": base64.b64encode(f.read()).decode()
                    }
                },
                {
                    "type": "text",
                    "text": "이 판매 데이터를 분석하고 주요 인사이트를 추출해줘"
                }
            ]
        }]
    )

print(message.content)

4-4. 문제 해결

문제: 응답이 너무 느림

  • 해결: Haiku 모델 사용
  • 또는 프롬프트 단순화

문제: 코드가 작동하지 않음

  • 해결: 에러 메시지 전체를 Claude에게 보여주고 디버깅 요청
  • "다음 에러가 발생했어: [에러 메시지]. 원인과 해결 방법을 알려줘"

문제: 원하는 답변이 아님

  • 해결: 더 구체적으로 요청
  • 예시 제공
  • 단계별로 나눠 요청

문제: 파일 업로드 실패

  • 해결: 파일 크기 확인 (제한 있음)
  • 지원 형식 확인
  • 여러 파일로 나눠 업로드

문제: Memory가 잘못된 정보 기억

  • 해결: Settings > Memory에서 해당 정보 삭제
  • 또는 "이전에 [X]라고 했지만, 실제로는 [Y]야. 기억을 업데이트해줘"

5-1. 자주 묻는 질문 (FAQ)

Q1: 한도를 다 쓰면 어떻게 되나요? A: 5시간만 기다리면 다시 충전됩니다. 급하면 다른 AI 서비스를 임시로 사용하거나, 상위 플랜으로 업그레이드를 고려해보세요.

Q2: Pro 플랜이 정말 필요한가요? A: 매일 1시간 이상 사용한다면 필수입니다. 한도 제한 스트레스 없이 작업할 수 있고, Opus 모델로 더 정확한 결과를 얻을 수 있습니다.

Q3: 토큰이 가장 많이 소모되는 경우는? A: 1) 긴 대화 지속, 2) 큰 파일 업로드, 3) 이미지 다량 사용, 4) Opus 모델 과다 사용

Q4: 무료로 계속 쓸 수 있나요? A: 가능합니다! 하루 20-30개 메시지로 충분하다면 Free 플랜으로도 충분합니다.

Q5: 팀 플랜 vs 개인 Pro 여러 개? A: 5명 이상이면 Team 플랜이 유리합니다. 중앙 관리, 공유 Projects, 통합 청구의 장점이 있습니다.
