# 5편 | HTML - 웹사이트 언어

---

4편에서 코드 에디터를 완성한 뒤, 이제 첫 번째 웹 언어인 HTML을 시작해봅니다. 모든 웹사이트의 기초가 되는 HTML을 체계적으로 살펴보겠습니다. 웹페이지의 구조를 만드는 마크업 언어의 기본 개념부터 실무에서 자주 사용하는 핵심 태그들, 그리고 접근성을 고려한 올바른 HTML 작성법까지 웹개발의 첫걸음을 확실하게 완성해봅니다.

## 1. HTML이란 무엇인가

### 1-1. 웹사이트를 만드는 언어

HTML을 **웹사이트를 만드는 기본 언어**라고 생각하면 됩니다. HTML은 HyperText Markup Language의 줄임말인데요, 쉽게 말해 웹페이지의 '설계도'를 그리는 언어입니다. 실제로 여러분이 지금 보고 있는 모든 웹사이트가 HTML로 기본 뼈대가 만들어져 있답니다.

HTML의 역할을 **집 짓기에 비유**하면 정말 이해하기 쉬워요. 기둥과 벽으로 튼튼한 뼈대를 만드는 것과 똑같거든요. 반면에 나중에 배울 CSS가 예쁜 인테리어를 담당한다면, HTML은 집의 **핵심적인 기본 구조**를 만드는 역할을 한다고 생각하면 됩니다.

### 1-2. 마크업 언어의 의미

**마크업(Markup)**이란 쉽게 말해 텍스트에 '이것이 무엇인지'라고 표시를 해주는 것입니다. 여러분이 교과서에서 형광펜으로 중요한 부분을 칠하는 것처럼 말입니다. 마찬가지로 HTML도 컴퓨터에게 "이건 제목이야, 저건 본문이야, 요건 링크야"라고 친절하게 알려주는 **매우 중요한** 역할을 합니다.

구체적인 예시로 '회사소개'라는 글자를 생각해보면, 그냥 쓰면 평범한 텍스트에 불과합니다. 하지만 `<h1>회사소개</h1>`로 감싸는 순간, **마법처럼** 중요한 제목으로 변신합니다. 웹브라우저는 이런 표시를 보고 "아, 이건 중요한 제목이구나"하면서 큰 굵은 글씨로 표시해줍니다.

### 1-3. 다른 웹 기술들과의 관계

이 세 기술의 **훌륭한 협력관계**를 생각해보면 HTML은 웹사이트의 튼튼한 뼈대를 만들고, CSS는 그 위에 아름다운 디자인 옷을 입혀줍니다. 그리고 JavaScript는 살아 숨쉬는 동작과 기능을 불어넣습니다. 이 세 친구가 함께 협력해야만 **완벽한 웹사이트**가 탄생하는 것입니다.

## 2. HTML 문서 기본 구조

### 2-1. HTML 문서의 틀

여기서 **중요한 포인트**가 있습니다. 모든 HTML 문서는 마치 정해진 템플릿처럼 **일정한 기본 구조**를 가지고 있답니다. 맨 위에는 DOCTYPE 선언이 자리잡고 있고, 그 아래에는 html 태그가 전체 문서를 따뜻하게 감싸는 구조죠.

html 태그 안에는 두 개의 주요 부분이 있습니다. head는 웹페이지 설정 정보가 들어가는 곳입니다. body는 실제로 사용자가 보는 내용이 들어가는 곳입니다.

### 2-2. head 영역의 역할

head 영역에는 다음과 같은 정보들이 들어갑니다. 브라우저 탭에 표시될 제목, 문자 인코딩 설정, 검색엔진을 위한 설명 등입니다. 사용자 눈에는 보이지 않지만, 웹페이지가 제대로 작동하는 데 필요한 중요한 정보들입니다.

나중에 CSS 파일이나 JavaScript 파일을 연결하는 정보도 여기에 들어갑니다.

### 2-3. body 영역의 내용

body 영역에는 사용자가 실제로 보고 상호작용하는 모든 내용이 들어갑니다. 제목, 본문, 이미지, 링크, 버튼 등이 모두 body 안에 위치합니다.

## 3. HTML 태그 시스템

### 3-1. 태그의 기본 원리

HTML은 태그(Tag)라는 표시를 사용합니다. **핵심 포인트**를 기억하세요. 태그는 항상 꺾쇠괄호 < > 안에 들어가야 합니다. 게다가 대부분의 태그는 마치 괄호처럼 시작 태그와 끝 태그가 **완벽한 쌍**을 이루고 있어요. 이걸 잊으면 안 됩니다.

기본 형태는 `<태그명>내용</태그명>` 이렇게 생겼는데요, 정말 **직관적**입니다. 예를 들어 `<h1>제목</h1>`이라고 쓰면, 컴퓨터에게 "여기 있는 '제목'이라는 텍스트를 **가장 중요한 제목**으로 표시해줄래요"라고 명령하는 셈입니다.

### 3-2. 브라우저가 태그를 해석하는 방법

브라우저는 HTML 태그를 읽어서 화면에 표시합니다. 예를 들어 `<h1>` 태그를 발견하면 큰 굵은 글씨로 표시합니다. `<p>` 태그를 만나면 일반 문단으로 표시하는 식입니다.

### 3-3. 태그의 중첩

태그는 다른 태그 안에 들어갈 수 있습니다. 예를 들어 `<p>이것은 <strong>중요한</strong> 내용입니다</p>`라고 쓰면, 문단 안의 "중요한"이라는 부분만 굵게 표시됩니다.

## 4. 꼭 알아야 할 HTML 태그들

### 4-1. 제목 태그

여기서 **매우 흥미로운 사실**이 있습니다. HTML에는 h1부터 h6까지 총 6개의 제목 태그가 있는데, 마치 회사의 직급처럼 **계급이 있다**는 것입니다. 숫자가 작을수록 더 높은 직급(더 중요한 제목)이고, 화면에서도 당당하게 더 크게 표시됩니다.

**실무에서 꼭 기억해야 할 점**은 h1이 그 페이지의 **절대 왕**이라는 것입니다. 보통 회사명이나 페이지의 메인 제목에 사용합니다. 그 다음 h2는 주요 섹션의 **중간 보스** 역할을, h3는 그보다 작은 소제목 역할을 합니다. 이런 식으로 **체계적인 계층 구조**를 만들어가는 것이 매우 중요합니다.

### 4-2. 텍스트 관련 태그

일반적인 본문 내용은 p 태그로 감쌉니다. p는 문단(paragraph)을 의미하는 태그입니다. 

중요한 부분을 강조하고 싶을 때는 strong 태그를 사용해서 굵게 표시하고, em 태그를 사용해서 기울임체로 강조할 수도 있습니다.

줄바꿈이 필요할 때는 br 태그를, 구분선이 필요할 때는 hr 태그를 사용합니다.

### 4-3. 목록 태그

여러 개의 항목을 나열할 때는 목록 태그를 사용합니다. 

순서가 중요하지 않은 목록은 ul 태그로 전체를 감쌉니다. 각 항목은 li 태그로 만듭니다. 이렇게 하면 글머리 기호(•)가 붙은 목록이 만들어집니다.

순서가 중요한 목록은 ol 태그를 사용하면 1, 2, 3... 이런 번호가 자동으로 붙습니다.

### 4-4. 링크와 이미지

다른 페이지로 이동하는 링크는 a 태그로 만들고, href 속성에 이동할 주소를 적습니다. 예를 들어 `<a href="https://naver.com">네이버</a>`라고 쓰면 네이버로 이동하는 링크가 만들어집니다.

이미지를 표시할 때는 img 태그를 사용합니다. src 속성에 이미지 파일의 위치를 적고, alt 속성에는 이미지 설명을 반드시 넣어야 합니다.

## 5. HTML 구조 태그

### 5-1. 의미를 가진 구조 태그

HTML5에서는 웹페이지의 각 영역이 어떤 역할을 하는지 명확하게 표시할 수 있는 새로운 태그들이 추가되었습니다.

예전에는 모든 구역을 div라는 태그로 나누었습니다. 하지만 이제는 의미를 가진 태그들을 사용할 수 있습니다. header, nav, main, section, article, aside, footer 같은 태그들입니다.

### 5-2. 주요 구조 태그들

header 태그는 웹사이트나 섹션의 상단부에 사용합니다. 네이버 홈페이지 맨 위의 로고와 메뉴가 있는 부분이 바로 header입니다.

nav 태그는 내비게이션 메뉴를 감쌀 때 사용합니다. main 태그는 그 페이지의 주요 내용을 감쌉니다. 예를 들어 회사 홈페이지에서 "회사소개", "서비스", "연락처" 같은 핵심 내용이 main에 들어갑니다.

section 태그는 주제별로 내용을 나눌 때 사용합니다. 회사소개 섹션, 서비스소개 섹션 이런 식으로 구분합니다. article 태그는 블로그 글이나 뉴스 기사처럼 그 자체로 완결성을 가진 콘텐츠에 사용합니다.

aside 태그는 사이드바나 부가 정보에, footer 태그는 웹사이트 하단의 연락처나 저작권 정보에 사용합니다.

### 5-3. 구조 태그 사용의 장점

이런 구조 태그들을 사용하면 구글 같은 검색엔진이 웹페이지 내용을 더 정확하게 이해할 수 있습니다. 또한 시각장애인이 사용하는 스크린리더 같은 보조 기술도 페이지 구조를 더 잘 파악해서 내용을 읽어줄 수 있습니다.

```
[HTML 문서 구조]

<html>
  ├── <head>
  │     ├── <title>페이지 제목</title>
  │     ├── <meta> (문서 정보)
  │     └── <link> (CSS 연결)
  │
  └── <body>
        ├── <header>
        │     ├── <h1>사이트 제목</h1>
        │     └── <nav>메뉴</nav>
        │
        ├── <main>
        │     ├── <section>
        │     │     ├── <h2>섹션 제목</h2>
        │     │     └── <p>내용</p>
        │     │
        │     └── <article>
        │           ├── <h3>글 제목</h3>
        │           └── <p>본문</p>
        │
        ├── <aside>
        │     └── 사이드바 내용
        │
        └── <footer>
              └── 저작권 정보
```

## 6. HTML 속성

### 6-1. 속성의 개념

HTML 태그에는 속성(attribute)을 추가해서 더 자세한 정보를 줄 수 있습니다. 속성은 태그 안에 `속성명="값"` 형태로 씁니다.

예를 들어 링크 태그에 href 속성을 써서 어디로 이동할지 주소를 지정하거나, 이미지 태그에 src 속성을 써서 어떤 이미지 파일을 보여줄지 알려줄 수 있습니다.

### 6-2. 자주 사용하는 속성들

모든 HTML 태그에서 사용할 수 있는 공통 속성들이 있습니다. id 속성은 그 요소에 고유한 이름을 주고, class 속성은 여러 요소를 같은 그룹으로 분류할 때 사용합니다. 이는 나중에 CSS로 디자인할 때 매우 중요해집니다.

링크는 정말 **신기한 마법**입니다. href 속성으로 어디로 갈지 주소를 지정하는 것입니다. `<a href="about.html">회사소개</a>`라고 쓰면 **같은 집 안의 다른 방**으로 이동하는 거고, `<a href="https://naver.com">네이버</a>`라고 쓰면 **완전히 다른 집(외부 사이트)**으로 순간이동하는 것입니다. 게다가 target="_blank"를 추가하면 새 창에서 링크가 열린답니다. **매우 유용한 기능**입니다.

이미지 태그에서는 src로 파일 위치를 지정하고, alt로 이미지 설명을 반드시 넣어야 합니다. `<img src="company-logo.png" alt="ABC회사 로고">`처럼 말이죠.

### 6-3. 속성 사용 시 주의사항

속성 값은 항상 따옴표로 감싸야 하고, 속성 이름은 소문자로 쓰는 것이 좋습니다. 또한 이미지의 alt 속성처럼 접근성을 위해 반드시 넣어야 하는 속성들도 있습니다.

## 7. 올바른 HTML 작성법

### 7-1. HTML의 관대함과 엄격함

HTML은 문법에 상당히 관대한 언어입니다. 태그를 안 닫거나 대소문자를 섞어 써도 대부분 잘 작동합니다. 하지만 좋은 HTML을 만들려면 올바른 규칙을 지키는 것이 중요합니다.

### 7-2. 좋은 HTML 작성 원칙

태그 이름은 소문자로 통일하고, 속성 값은 항상 따옴표로 감싸며, 들여쓰기를 해서 구조를 명확하게 보여주는 것이 좋습니다.

가장 중요한 것은 내용의 의미에 맞는 태그를 사용하는 것입니다. 제목이라면 h1이나 h2 태그를 써야 하고, 단순히 글씨를 크게 만들고 싶다고 해서 제목 태그를 남용하면 안 됩니다.

### 7-3. 접근성 고려사항

모든 사용자가 웹사이트를 이용할 수 있도록 접근성을 고려해야 합니다. 이미지에는 반드시 alt 속성을 넣어서 설명을 달아주고, 링크 텍스트는 "여기 클릭" 같은 애매한 표현 대신 "회사 소개 보기" 같이 구체적으로 써야 합니다.

제목 태그도 h1부터 순서대로 사용해서 논리적인 구조를 만들어야 합니다.

### 7-4. HTML 작성 시 주의사항

HTML은 구조와 내용에만 집중해야 합니다. 색깔이나 크기 같은 디자인 요소는 나중에 나오는 CSS에서 담당하므로, HTML에서는 "이건 제목이다", "이건 중요한 내용이다" 같은 의미만 표시하는 것이 좋습니다.

또한 웹표준을 준수하고 모든 브라우저에서 동일하게 작동하도록 검증된 태그와 속성만 사용하는 것이 중요합니다.
