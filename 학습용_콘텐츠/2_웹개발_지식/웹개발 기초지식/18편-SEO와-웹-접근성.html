18편 | SEO와 웹 접근성

17편에서 보안과 성능 최적화를 다루었다면, 이제 중요한 마지막 두 주제를 살펴볼 차례입니다.

웹사이트를 만들었는데 아무도 찾지 못한다면, 또는 일부 사용자들이 접근할 수 없다면 매우 아쉬운 일입니다.

이번 편에서는 구글과 네이버 검색 엔진에 최적화된 SEO 기법과, 모든 사용자가 차별 없이 이용할 수 있는 웹 접근성 구현 방법을 함께 살펴보겠습니다. 우수한 콘텐츠도 사람들이 찾지 못하면 의미가 없습니다.

1. SEO의 이해

1-1. SEO란 무엇인가

**SEO(Search Engine Optimization)**란 검색 엔진의 이해를 돕는 작업입니다. 구글이나 네이버 같은 검색 엔진이 웹사이트를 제대로 이해하고, 사용자들에게 적절히 추천할 수 있도록 최적화하는 과정입니다.

도서관에서 원하는 책을 찾는 과정에 비유할 수 있습니다. 큰 도서관에서 원하는 책을 찾으려면 제목이 명확하고, 분류가 체계적이며, 접근성이 좋아야 합니다. 검색 엔진도 마찬가지입니다. 웹사이트가 체계적으로 구성되어 있어야 사람들이 쉽게 찾을 수 있습니다.

SEO가 중요한 이유:

  • 검색 결과 1페이지가 전체 클릭의 90%를 차지합니다
  • 무료로 지속적인 방문자를 확보할 수 있습니다
  • 브랜드 신뢰도가 향상됩니다
  • 광고비 절감 효과를 얻을 수 있습니다

1-2. 검색 엔진의 작동 원리

검색 엔진의 작동 원리는 세 단계로 나눈어집니다:

1단계: 크롤링 (Crawling) 구글의 웹 크롤러(스파이더)가 인터넷을 돌아다니며 웹페이지를 발견하고 정보를 수집합니다. 도서관 사서가 새로운 책들을 수집하는 것과 유사합니다.

2단계: 인덱싱 (Indexing) 수집한 정보를 방대한 데이터베이스에 분류하고 저장합니다. 하나하나의 페이지를 주제와 내용에 따라 체계적으로 정리합니다.

3단계: 랭킹 (Ranking) 사용자가 검색하면 수많은 후보들 중에서 가장 적합한 결과를 찾아서 순위를 매깁니다. 이 모든 과정이 0.5초 만에 이루어집니다.

하지만 여기서 정말 흥미로운 사실은, 검색 엔진이 무려 200개 이상의 요소를 동시에 고려한다는 점입니다:

  • 🎯 콘텐츠의 품질과 관련성 - 사용자가 원하는 답이 정확히 있는가?
  • 🏅 웹사이트의 권위와 신뢰도 - 다른 사이트들이 이 사이트를 추천하는가?
  • ⚡ 사용자 경험 - 빠르게 로드되고, 모바일에서도 잘 보이는가?
  • 🔧 기술적 최적화 상태 - 검색 엔진이 읽기 쉽게 만들어졌는가?

이런 복잡한 계산을 인공지능이 순식간에 처리합니다.

2025년 현재 중요한 SEO 요소:

  • Core Web Vitals (LCP, INP, CLS)
  • 모바일 우선 인덱싱
  • E-E-A-T (구글의 품질 평가 기준)
  • 네이버 C-Rank 알고리즘

1-3. E-E-A-T 심화 이해

[SEO 최적화 구조]

┌─────────────────────────────────────┐
│         웹페이지 구조                │
├─────────────────────────────────────┤
│ <head>                              │
│   ├── <title>제목 (60자 이내)</title> │
│   ├── <meta description>           │

│   ├── Open Graph 태그              │
│   └── Schema.org 구조화 데이터      │
├─────────────────────────────────────┤
│ <body>                              │
│   ├── <header>                     │
│   │    └── <h1>메인 제목</h1>      │
│   ├── <nav>메뉴</nav>              │
│   ├── <main>                       │
│   │    ├── <article>               │
│   │    │    ├── <h2>소제목</h2>    │
│   │    │    └── 콘텐츠             │
│   │    └── 내부 링크               │
│   └── <footer>                     │
└─────────────────────────────────────┘

[검색엔진 크롤링 순서]
1. robots.txt 확인
2. sitemap.xml 읽기
3. 페이지 크롤링
4. 인덱싱
5. 순위 결정

E-E-A-T는 구글이 콘텐츠의 신뢰도를 판단하는 핵심 기준입니다. 2022년 12월부터 기존 E-A-T에 Experience(경험)가 추가되어 더욱 까다로워졌어요. 마치 엄격한 선생님이 과제를 채점하는 것처럼 꼼꼼히 살펴본다고 생각하시면 됩니다.

Experience (경험) - 실제 경험을 바탕으로 작성한 이야기인지 확인합니다. 단순히 인터넷에서 다른 글을 참고해서 쓴 내용이 아니어야 합니다.

  • 제품을 직접 구매해서 사용해본 후 리뷰를 작성해야 합니다.
  • 여행지 추천글이라면 실제로 방문한 경험이 있어야 설득력이 있습니다.

Expertise (전문성) 📚

  • 해당 분야의 전문가가 작성해야 합니다. 단순히 지식만 나열하는 것이 아니라, 심도 있고 정확한 정보를 제공해야 합니다.
  • 마치 해당 분야의 선생님이 설명해주는 듯한 신뢰감을 주어야 합니다.

Authoritativeness (권위성) 🏆

  • 다른 사람들이 인정하는 사이트여야 합니다. 마치 법학대 교수님의 논문이 인용되는 것처럼요
  • 다른 권위 있는 사이트들이 해당 사이트의 정보가 우수하다고 인정하며 링크를 걸어주어야 합니다.

Trustworthiness (신뢰성) ✨

  • 정보가 정확하고 최신이어야 합니다. 2020년 정보를 2025년에도 그대로 두면 안 됩니다.
  • 숨기는 것 없이 투명해야 합니다. 예를 들어 제품 리뷰에서 단점도 솔직하게 언급해야 합니다.
  • 전자상거래라면 안전한 결제 시스템을 갖추고 있어야 합니다.

E-E-A-T 적용 방법:

  • 저자 정보 페이지 만들기
  • 자격증, 경력 명시
  • 출처 명확히 표시
  • 정기적인 콘텐츠 업데이트
  • 사용자 리뷰와 평가 포함

1-4. 네이버 C-Rank 알고리즘

네이버는 한국 시장에 특화된 C-Rank 알고리즘을 사용합니다. Creator Rank의 약자로, 창작자와 콘텐츠의 품질을 종합적으로 평가합니다.

C-Rank 핵심 요소:

  • 콘텐츠 품질: 독창성과 전문성
  • 창작자 신뢰도: 지속적인 양질의 콘텐츠 생산
  • 사용자 반응: 체류 시간, 재방문율
  • 최신성: 정기적인 업데이트
  • 모바일 최적화: 모바일 환경 대응

2. 웹 접근성의 이해

2-1. 웹 접근성이란

웹 접근성을 **'디지털 남녀노소 차별 금지'**라고 생각하시면 되어요. 장애가 있든 없든, 나이가 많든 어리든, 모든 사람이 편리하게 이용할 수 있는 웹사이트를 만드는 것이죠.

아파트에 에스컬레이터만 있고 엘리베이터가 없다면 얼마나 불편할지 고려해보시기 바랍니다. 휘체어 이용자는 당연히, 무거운 짐을 든 사람도 곤란하겠고요. 웹사이트도 마찬가지에요!

우리 주변에는 생각보다 다양한 상황의 사용자들이 있습니다:

  • 👀 시각 장애 사용자: **'스크린 리더'**라는 음성 프로그램이 화면을 읽어줄니다
  • 👂 청각 장애 사용자: 동영상에 자막이 없으면 아무것도 모르겠죠?
  • 👍 마우스를 쓸 수 없는 사용자: 키보드만으로 모든 것을 조작해야 합니다
  • 🧐 인지 장애 사용자: 복잡한 메뉴나 화려한 디자인은 혹시 혼란스러워요
  • 🤕 일시적 제약 상황: 팔을 다쳐서 한 손으로만 조작, 아기를 안고 있어서 한 손이 바쁘거나, 지하철에서 소리를 들을 수 없는 상황 등

이런 분들도 여러분의 웹사이트를 이용하고 싶어 한다는 사실, 속으로 깨닫지 않으시기 바란니다.

웹 접근성의 혜택:

  • 법적 의무 준수 (한국: 장애인차별금지법)
  • 더 넓은 사용자층 확보
  • SEO 개선 (접근성과 SEO는 밀접한 관계)
  • 브랜드 이미지 향상

2-2. WCAG 가이드라인

WCAG는 W3C에서 제정한 웹 접근성 국제 표준입니다. Web Content Accessibility Guidelines의 약자입니다.

**WCAG 2.2 (2023년 10월)**의 4가지 원칙 (POUR):

인지가능 (Perceivable)

  • 이미지에 대체 텍스트 제공
  • 동영상에 자막 제공
  • 충분한 색상 대비 (일반 4.5:1, 큰 텍스트 3:1)

운용가능 (Operable)

  • 키보드로 모든 기능 사용 가능
  • 충분한 시간 제공
  • 발작을 일으킬 수 있는 콘텐츠 금지

이해가능 (Understandable)

  • 읽기 쉬운 텍스트
  • 예측 가능한 기능
  • 입력 도움말 제공

견고성 (Robust)

  • 유효한 HTML 코드
  • ARIA 속성 적절히 사용
  • 다양한 보조 기술 호환성

3. SEO 실전 적용

3-1. 온페이지 SEO

제목 태그 최적화

<title>홈페이지 제작 | 웹디자인 전문 - ABC 회사</title>
  • 60자 이내
  • 핵심 키워드 포함
  • 각 페이지마다 고유한 제목

메타 설명 작성

<meta name="description" content="10년 경력의 웹디자인 전문 회사. 반응형 홈페이지 제작, SEO 최적화, 합리적인 가격. 무료 상담 가능.">
  • 155자 이내
  • 클릭을 유도하는 문구
  • 페이지 내용 요약

URL 구조

  • 좋음: example.com/products/laptop
  • 나쁨: example.com/p?id=123
  • 한글보다 영문 권장
  • 하이픈으로 단어 구분

3-2. 콘텐츠 최적화

키워드 전략

  • 자연스러운 키워드 배치
  • 동의어와 관련어 활용
  • 키워드 밀도 1-3%
  • 롱테일 키워드 활용

콘텐츠 구조화

  • H1 태그는 페이지당 하나
  • H2-H6 태그로 계층 구조
  • 짧은 단락 (3-4줄)
  • 불렛 포인트 활용

3-3. 기술적 SEO

사이트맵 제작

  • XML 사이트맵: 검색 엔진용
  • HTML 사이트맵: 사용자용
  • 정기적인 업데이트

robots.txt 설정

User-agent: *
Disallow: /admin/
Allow: /
Sitemap: https://example.com/sitemap.xml

페이지 속도 최적화

  • 이미지 압축
  • CSS/JS 최소화
  • 캐싱 활용
  • CDN 사용

4. 접근성 실전 적용

4-1. 기본 접근성 구현

이미지 대체 텍스트

  • 모든 이미지에 alt 속성
  • 장식용 이미지는 alt=""
  • 의미 있는 설명 작성

폼 접근성

  • 모든 입력 필드에 레이블
  • 오류 메시지 명확히 표시
  • 필수 항목 표시

키보드 접근성

  • Tab 키로 모든 요소 접근
  • 포커스 표시 명확히
  • Skip to content 링크

4-2. ARIA 활용

ARIA(Accessible Rich Internet Applications)는 웹 접근성을 향상시키는 속성입니다:

  • role: 요소의 역할 정의
  • aria-label: 요소 설명
  • aria-live: 동적 콘텐츠 알림
  • aria-hidden: 스크린 리더에서 숨기기

4-3. 테스트 도구

접근성 테스트

  • WAVE: 웹 접근성 평가 도구
  • axe DevTools: 브라우저 확장 프로그램
  • NVDA/JAWS: 스크린 리더 테스트

SEO 테스트

  • Google Search Console
  • 네이버 서치어드바이저
  • Screaming Frog SEO Spider

5. 모바일 SEO와 접근성

5-1. 모바일 우선 인덱싱

2025년 현재 구글은 모바일 버전을 기준으로 사이트를 평가합니다:

  • 반응형 디자인 필수
  • 터치 타겟 최소 44×44 픽셀
  • 읽기 쉬운 폰트 크기
  • 뷰포트 설정 필수

5-2. AMP와 PWA

AMP (Accelerated Mobile Pages)

  • 초고속 모바일 페이지
  • 구글 검색 결과 우대
  • 제한된 JavaScript

PWA (Progressive Web App)

  • 앱처럼 작동하는 웹
  • 오프라인 작동
  • 푸시 알림 가능

6. 로컬 SEO

6-1. 지역 비즈니스 최적화

  • Google My Business 등록
  • 네이버 플레이스 등록
  • 일관된 NAP (이름, 주소, 전화번호)
  • 지역 키워드 활용
  • 고객 리뷰 관리

6-2. 구조화된 데이터

Schema.org 마크업으로 검색 엔진에 추가 정보 제공:

  • 영업 시간
  • 가격 정보
  • 리뷰 평점
  • 이벤트 정보
