3편 | 명령줄(CLI)와 개발 환경 설정

2편에서 웹사이트의 비밀스러운 작동 원리를 알아보았으니, 이제 진짜 실전 준비를 해보겠습니다. 마치 요리를 하기 전에 도구들을 준비하듯이, 웹개발의 세계에 들어가기 전에도 필수 도구들을 준비해야 합니다. 처음에는 조금 낯설어 보이지만, 생각보다 전혀 어렵지 않습니다. 새로운 스마트폰을 처음 사용할 때처럼, 하나씩 천천히 익혀보겠습니다.

Part A. 명령줄(CLI(명령줄 인터페이스; Command Line Interface)) 완벽 이해하기

1. 명령줄(CLI(명령줄 인터페이스)) 기본 개념과 활용

1-1. CLI(명령줄 인터페이스)가 무엇인지 이해하기

**CLI(명령줄 인터페이스; Command Line Interface)**는 많은 분들에게 생소한 개념일 수 있지만, 사실 엄청 단순한 개념입니다. 마우스 대신 키보드로 컴퓨터와 대화하는 방법이라고 생각하면 됩니다.

예를 들어보겠습니다. 평소에 "내 컴퓨터"를 열고 마우스로 클릭클릭 하면서 "문서" 폴더에 들어가는 방식이 GUI(그래픽 사용자 인터페이스; Graphical User Interface)입니다. 반대로 CLI(명령줄 인터페이스)는 검은 화면에 cd Documents라고 키보드로 입력해서 똑같은 작업을 하는 것입니다. 마치 컴퓨터와 텍스트로 대화하는 것 같습니다.

개발자들이 CLI(명령줄 인터페이스)를 사용하는 이유

여기서 재미있는 사실입니다 원래 컴퓨터는 CLI(명령줄 인터페이스)밖에 없었습니다. GUI(그래픽 사용자 인터페이스)는 나중에 사람들이 "이게 더 편하겠다" 하고 만들어진 편의 기능입니다. 그래서 지금도 많은 전문적인 개발 도구들이 CLI(명령줄 인터페이스)를 기반으로 만들어져 있습니다. 이런 강력한 도구들을 사용하려면 CLI(명령줄 인터페이스)를 알아야 한다는 것입니다.

그리고 생각해보면, 마우스로 100개의 파일 이름을 하나씩 클릭해서 바꿀니 터버달 버튼까지 눈이 아픈 경험 있을 것입니다? 하지만 CLI(명령줄 인터페이스)에서는 단 한 줄의 마법 명령어로 모든 파일을 순식간에 처리할 수 있습니다. 이것이 얼마나 효율적인지 생각해보세요.

1-2. 기본 명령어 익히기

처음 사용해볼 명령어들:

명령줄(CLI(명령줄 인터페이스))이 열리면 현재 위치가 표시됩니다. 예를 들어 C:\Users\사용자명 같은 경로가 나타납니다. 여기서 다음 명령어들을 입력해보수 있습니다:

  • dir 입력 후 엔터: 현재 폴더에 있는 파일과 폴더 목록을 보여줍니다
  • cd Desktop 입력 후 엔터: 바탕화면 폴더로 이동합니다
  • cd .. 입력 후 엔터: 한 단계 위 폴더로 올라갑니다
  • mkdir 폴더명 입력 후 엔터: 새로운 폴더를 만듭니다

이 명령어들은 탐색기에서 마우스로 하는 작업과 똑같지만, 키보드로만 수행하는 것입니다.

1-3. 명령줄(CLI(명령줄 인터페이스)) 사용 팁

시간을 절약하는 Tab 키 활용법:

긴 폴더 이름을 다 입력할 필요가 없습니다. 예를 들어 cd Des까지만 입력하고 Tab 키를 누르면 컴퓨터가 자동으로 cd Desktop으로 완성해줍니다. 만약 Des로 시작하는 폴더가 여러 개라면 Tab을 여러 번 누르면서 원하는 폴더를 선택할 수 있습니다.

명령어 히스토리 사용하기:

방금 입력했던 명령어를 다시 사용하고 싶다면 화살표 위(↑) 키를 누르세요. 계속 누르면 더 이전에 사용했던 명령어들이 나타납니다. 이 기능은 긴 명령어를 반복해서 사용할 때 매우 유용합니다.

탐색기와 명령줄(CLI(명령줄 인터페이스)) 연동하기:

특정 폴더에서 명령줄(CLI(명령줄 인터페이스))을 열고 싶다면 탐색기에서 해당 폴더로 이동한 후 다음 방법들을 사용할 수 있습니다:

  • 주소창에 cmd 입력 → CMD가 열림
  • 주소창에 powershell 입력 → PowerShell이 열림
  • 주소창에 wt 입력 → Windows Terminal이 열림 (설치된 경우)
  • 폴더에서 우클릭 → "Terminal에서 열기" (Windows 11)

2. 윈도우 명령줄(CLI(명령줄 인터페이스))의 종류와 차이점

Windows에는 3가지 명령줄(CLI)이 있습니다. 각각 다른 특징과 여는 방법을 가지고 있습니다.

2-1. CMD (Command Prompt) - 기본 명령줄(CLI)

특징:

  • Windows에 기본으로 제공
  • 가볍고 빠른 실행
  • 기본적인 파일 조작에 적합

여는 방법:

  • Windows 키 + R → cmd 입력
  • 시작 메뉴에서 "명령 프롬프트" 검색
  • 파일 탐색기 주소창에 cmd 입력

주요 사용 상황: 간단한 파일 이동, 폴더 생성 등 기본 작업

2-2. PowerShell - 고급 명령줄(CLI)

특징:

  • 더 강력한 스크립팅 기능
  • 객체 지향 명령어 체계
  • Windows 시스템 관리에 특화

여는 방법:

  • Windows 키 + R → powershell 입력
  • 시작 메뉴에서 "PowerShell" 검색
  • Windows 키 + X → "Windows PowerShell" 선택
  • 파일 탐색기 주소창에 powershell 입력

주요 사용 상황: 복잡한 시스템 관리, 자동화 스크립트 작성

2-3. Windows Terminal - 최신 터미널

특징:

  • 2019년 출시된 현대적인 터미널
  • 탭 기능으로 여러 세션 동시 관리
  • 색상 테마와 폰트 커스터마이징 가능
  • CMD, PowerShell을 모두 지원

설치 방법:

  • Microsoft Store에서 "Windows Terminal" 검색 후 설치
  • Windows 11에는 기본 설치되어 있음

여는 방법:

  • Windows 키 + R → wt 입력 (설치 후)
  • 시작 메뉴에서 "Terminal" 검색
  • Windows 키 + X → "Terminal" 선택 (Windows 11)
  • 파일 탐색기에서 우클릭 → "Terminal에서 열기"

개발자 권장 순서:

  1. Windows Terminal (가장 편리)
  2. PowerShell (고급 기능 필요시)
  3. CMD (간단한 작업용)

Part B. 개발 환경 설정하기

1. Node.js & npm - 모든 웹개발의 기반

1-1. Node.js의 개념과 중요성

Node.js는 웹브라우저가 아닌 컴퓨터에서 JavaScript를 실행할 수 있게 해주는 런타임 환경(Runtime Environment)입니다.

Node.js가 중요한 이유:

  • 최신 웹개발 도구들이 대부분 Node.js 기반으로 만들어짐
  • npm 패키지 관리자 사용 가능
  • 로컬 개발 서버 실행 가능
  • React, Vue, Angular 등 프레임워크 실행에 필수

1-2. Node.js 설치 과정

설치 단계:

  1. https://nodejs.org 접속
  2. LTS 버전 다운로드 (Long Term Support, 장기 지원 - 안정성 보장)
  3. 다운로드한 파일 실행
  4. 설치 옵션은 모두 기본값으로 설정

설치 확인: 명령줄(CLI)에서 node --version과 npm --version을 입력해보세요. 버전 번호가 정상으로 출력되면 설치가 성공적으로 완료된 것입니다.

1-3. Node.js 기본 활용

JavaScript 파일 실행: 터미널에서 node 파일명.js 명령어로 JavaScript 파일을 실행할 수 있습니다.

웹서버 생성: Node.js를 사용하면 간단한 코드로 웹서버를 만들 수 있습니다. 필요할 때 Claude Code가 자동으로 생성해드립니다.

1-4. npm - 패키지 관리 시스템

1.4.1 패키지 관리자란?

npm(\ub178\ub4dc \ud328\ud0a4\uc9c0 \uad00\ub9ac\uc790)은 Node Package Manager의 줄임말입니다. 이는 세계 최대의 소프트웨어 패키지 저장소입니다.

예를 들어보겠습니다. 웹사이트에 이미지 슬라이더를 만들고 싶다고 가정해보겠습니다. 처음부터 직접 코드를 작성할 수도 있습니다. 하지만 이미 누군가가 만들어 놓은 '슬라이더 패키지'를 npm을 통해 설치해서 사용할 수 있습니다. 마치 스마트폰에서 앱 스토어에서 앱을 다운로드하는 것과 비슷합니다.

npm(노드 패키지 관리자)이 하는 주요 일들:

  • 필요한 패키지를 설치하고 업데이트 관리
  • 패키지가 필요로 하는 다른 패키지들도 자동으로 함께 설치
  • 프로젝트에서 사용하는 패키지 목록 관리
  • 개발 서버 실행 같은 작업을 간단한 명령어로 실행
1.4.2 npm 필수 명령어

새 프로젝트 시작: npm init 명령어로 새 프로젝트를 시작할 수 있습니다. 이 명령어를 실행하면 프로젝트 정보를 입력받아 package.json 파일이 생성됩니다.

패키지 설치:

  • npm install 패키지명: 현재 프로젝트에 패키지 설치
  • npm install -g 패키지명: 컴퓨터 전체에 설치 (전역 설치)
  • 여러 패키지를 한 번에 설치하는 것도 가능합니다

패키지 제거: npm uninstall 패키지명 명령어로 설치된 패키지를 제거할 수 있습니다.

1.4.3 package.json의 역할

프로젝트 정보와 사용하는 패키지 목록을 저장하는 핵심 파일입니다. 이 파일에는 다음과 같은 정보가 포함됩니다. 프로젝트 이름, 버전, 의존성 패키지, 실행 스크립트 등입니다.

scripts 기능: package.json에 자주 사용하는 명령어를 scripts로 저장해두면 편리합니다. npm start, npm run dev 같은 간단한 명령어로 실행할 수 있습니다.

2. Git - 코드 버전 관리의 필수

2-1. Git의 개념과 중요성

Git은 코드의 변경사항을 추적하고 관리하는 버전 관리 시스템입니다.

문서 작업을 할 때를 생각해보세요. '최종.docx', '최종_진짜.docx', '최종_진짜_수정.docx' 같은 파일들을 만들어본 경험이 있으실 겁니다. Git은 이런 번거로움을 해결해줍니다. 하나의 파일을 유지하면서도 모든 수정 이력을 보관하고, 필요하면 언제든 원하는 시점으로 돌아갈 수 있습니다.

Git이 제공하는 편리한 기능들:

  • 코드를 잘못 수정해서 웹사이트가 작동하지 않을 때, 이전 상태로 되돌릴 수 있음
  • 여러 사람이 함께 작업할 때 각자의 수정사항을 합칠 수 있음
  • "어제 낮 2시쯤 코드가 잘 돌아갔는데..."라고 할 때 그 시점으로 정확히 되돌릴 수 있음
  • 새로운 기능을 실험하면서도 원래 코드는 안전하게 보호

2-2. Git 설치와 초기 설정

설치:

  1. https://git-scm.com 접속
  2. "Download for Windows" 클릭
  3. 설치 파일 실행 (모든 옵션 기본값 유지)

사용자 정보 설정 (최초 1회): git config --global user.name "이름"과 git config --global user.email "이메일"로 사용자 정보를 등록합니다.

설치 확인: git --version 명령어로 설치 여부를 확인할 수 있습니다.

2-3. Git 핵심 개념과 워크플로우

주요 개념:

  • 저장소(Repository): Git으로 관리하는 프로젝트 폴더
  • 커밋(Commit): 특정 시점의 코드 스냅샷 저장
  • 스테이징: 커밋할 파일들을 선택하는 준비 단계

기본 명령어:

  • git init: 새 저장소 초기화
  • git status: 현재 상태 확인
  • git add .: 모든 변경사항 스테이징
  • git commit -m "변경 내용 설명": 커밋 생성
  • git log: 커밋 히스토리 확인

기본 워크플로우:

  1. 프로젝트 폴더에서 git init으로 Git 저장소 초기화
  2. 파일 생성 또는 수정
  3. git add .로 변경사항 스테이징
  4. git commit -m "설명"으로 변경사항 저장
  5. 필요할 때 git log로 히스토리 확인

3. 로컬 서버 - 테스트 환경 구축

3-1. 서버와 로컬 서버의 차이

서버는 웹사이트 파일을 저장하고 사용자에게 제공하는 컴퓨터입니다.

예를 들어 네이버에 접속하면, 네이버의 서버 컴퓨터가 네이버 홈페이지를 여러분의 브라우저로 전송해주는 것입니다. 로컬 서버는 내 컴퓨터를 임시 서버로 만들어서, 내가 만든 웹사이트를 브라우저에서 테스트해보는 것입니다.

로컬 서버가 필요한 이유:

HTML 파일을 그냥 더블클릭해서 열어도 브라우저에 표시되지만, 이렇게 하면 많은 기능들이 제대로 작동하지 않습니다. 예를 들어 데이터베이스 연결, 사용자 로그인, 서버 쪽 코드 실행 등은 서버가 있어야만 가능합니다.

또한 코드를 수정할 때마다 브라우저가 자동으로 새로고침되는 편리한 기능도 로컬 서버를 통해 사용할 수 있습니다.

3-2. 로컬 서버 실행 방법들

live-server 사용 (추천): npm install -g live-server로 설치 후 live-server 명령어로 실행합니다.

  • 자동으로 브라우저가 열림
  • 파일 수정 시 자동 새로고침
  • 초보자에게 가장 편리

http-server 사용: npm install -g http-server로 설치 후 http-server 명령어로 실행합니다.

Node.js 기본 서버: Node.js 내장 모듈을 사용해서 직접 서버를 만들 수도 있습니다. 클로드 코드가 필요할 때 자동으로 생성해줍니다.

3-3. localhost와 포트 개념

localhost: 내 컴퓨터를 가리키는 주소 (127.0.0.1과 동일)

포트: 같은 컴퓨터에서 여러 서비스를 구분하는 번호

  • 3000번: http://localhost:3000
  • 8080번: http://localhost:8080
  • 5000번: http://localhost:5000
